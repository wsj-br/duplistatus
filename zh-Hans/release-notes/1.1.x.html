版本 1.1.x

概览​

版本 1.1.x 是一个小版本，解决了备份监控中的关键问题，提高了 Podman 容器的兼容性，并引入了容器架构的改进。这个版本专注于修复备份监控的准确性和解决 Podman 环境中的网络问题。

新功能​

用户界面增强​

  • 自动刷新按钮的右键菜单 - 在应用程序工具栏中右键单击自动刷新按钮，可以快速访问显示设置页面
  • 改进的收集备份模态 - 当选择服务器时，密码字段标签现在清楚地指示 "（仅在密码更改时填写）"，使得字段的需求更加明显

文档改进​

  • 文档托管迁移 - 文档已从 Docker 容器镜像迁移到 GitHub Pages https://wsj-br.github.io/duplistatus/user-guide/overview，以提高可访问性和更新方便
  • 应用程序头部的用户指南链接现在指向外部文档站点

🐛 错误修复​

关键：备份监控检测 - 完全重新实现​

  • 修复备份监控检测 - 完全重新实现了备份监控检测系统，采用新的算法来跟踪预期的备份日期:
    • 算法确保手动备份不会改变计划备份时间（例如，晚上 10 点的手动备份不会改变 2 点的计划）
    • 修复了 "下次运行" 列在备份监控设置中显示不正确的日期
    • 备份设置现在在通过 /api/upload 或 /api/backups/collect 接收备份后立即更新
    • 对于没有 Duplicati 同步的新服务器，使用第一个备份的时间作为计划参考
    • 有关更多详细信息，请参阅 逾期检测算法 文档

Podman 容器修复​

  • 修复 Podman pod 网络 - 解决了在 Podman pod 中运行应用程序时，外部无法访问应用程序的问题。服务器现在可以在 Podman pod 和独立容器模式下正确绑定到所有接口（0.0.0.0:9666）
  • 修复 Tailscale 主机名的 DNS 解析 - 解决了 Podman 容器中的 DNS 解析问题，防止连接到 Tailscale 主机名或自定义 DNS 服务器。更新了安装文档中的说明。

电子邮件配置修复​

  • 修复 SMTP 空用户名/密码处理 - 解决了清除电子邮件配置中的用户名或密码字段仍然将加密值存储在数据库中的问题。空字段现在可以正确地存储为空字符串并清除

系统改进​

  • 减少控制台日志 - AuditLogger 现在只将失败的登录尝试记录到控制台；所有其他审计条目都写入数据库

🚀 迁移说明​

从版本 1.0.x​

何时升级到版本 1.1.0 时：

  1. 文档位置更改 - 文档现在托管在 GitHub Pages 上，而不是包含在 Docker 容器中。应用程序头部的用户指南链接现在指向外部文档站点

  2. Podman 容器配置 - 如果使用 Podman 容器，请确保在容器级别使用 --dns 和 --dns-search 标志设置 DNS 配置。应用程序现在可以在 Podman 容器和独立容器中正确工作

  3. 逾期备份检测 - 逾期备份检测系统已完全重新实现，使用新的算法来保留备份时间。现有的备份监控配置将继续工作，但您可能会在备份监控设置中看到更准确的 "下次运行" 日期。新的算法确保手动备份不会更改计划备份时间

  4. 无需数据库迁移 - 本版本不需要任何数据库模式更改

API 端点​

本版本中没有添加新的 API 端点。所有现有的端点保持不变。

  • API 参考：API 文档

  • 社区: GitHub 讨论

报告错误​

报告错误时，请包括:

  • 版本：1.1.x（指定确切版本，例如 1.1.0）
  • 操作系统和版本
  • Docker/podman 版本
  • 容器类型（Docker 或 podman/Pod）
  • 错误消息和日志
  • 重现步骤
  • Podman 配置详细信息（如果在 podman/Pod 中运行）

更新日志​

版本 1.1.0 更改​

  • 添加：在自动刷新按钮上添加右键上下文菜单以打开显示设置
  • 添加：在收集备份模态中改进密码字段标签
  • 更改：从 Docker 容器图像中删除文档（现在托管在 GitHub Pages 上）
  • 更改：更新应用程序头部的用户指南链接以指向 GitHub Pages 文档
  • 更改：减少 AuditLogger 的控制台日志记录（仅记录失败的登录尝试到控制台）
  • 修复：通过正确实现预期备份日期跟踪来修复关键的逾期备份检测（请参阅 逾期检测算法 以获取详细信息）
  • 修复：Podman 容器网络问题，防止外部访问
  • 修复：Podman 容器中的 Tailscale 主机名的 DNS 解析
  • 修复：SMTP 配置存储空用户名和密码字段的加密值
  • 修复：备份监控设置中的 "下次运行" 列显示不正确的日期
