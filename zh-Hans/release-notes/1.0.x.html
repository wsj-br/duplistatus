版本 1.0.x

概览​

版本 1.0.x 是一个功能版本，引入了对电子邮件配置、服务器管理功能和数据库维护工具的重大增强。该版本专注于提高电子邮件通知的可靠性，扩展 SMTP 连接选项，并提供更好的工具来管理服务器和数据库完整性。

新功能​

服务器管理增强​

  • 服务器密码更改功能 - 用户现在可以直接从服务器详细信息页面更改服务器密码，方法是点击密钥图标
  • 重复服务器检测和合并 - 数据库维护表单中的新功能，可以检测重复服务器并将它们合并为一个服务器条目

审计日志记录增强​

  • 备份上传审计日志 - 所有通过 /api/upload 端点上传的备份日志现在都会自动记录在审计日志中。每个条目包括备份详细信息（服务器 ID、服务器名称、备份名称、状态、日期、持续时间、文件计数、大小、警告和错误）以及客户端 IP 地址和用户代理信息，用于跟踪和安全目的

电子邮件配置增强​

  • 配置不完整通知 - 当电子邮件设置不正确配置时显示的视觉警报
  • 服务状态指示器 - 当电子邮件服务未配置时，在通知复选框上显示“(已禁用)”标签
  • 可自定义的发件人信息:
    • Sender Name 字段，用于自定义在电子邮件通知中显示的发件人显示名称
    • From Address 字段，用于指定显示为发件人的电子邮件地址
    • 注意：一些电子邮件提供商将覆盖 From Address 使用 SMTP Server Username
  • SMTP 身份验证切换 - 启用或禁用 SMTP 身份验证的选项，支持不需要身份验证的服务器
  • 电子邮件格式验证 - 验证收件人和发件人地址字段，以确保正确的电子邮件地址格式
  • 增强的测试电子邮件 - 测试电子邮件现在显示包括 SMTP 服务器主机名、端口、连接类型、身份验证状态、用户名（如果适用）、收件人电子邮件、发件人地址、发件人名称和测试时间戳在内的综合连接详细信息

SMTP 连接类型支持​

  • 多个连接类型 - 支持三种连接类型:
    • Plain SMTP （无加密）在端口 25
    • STARTTLS （升级到 TLS）在端口 587
    • Direct SSL/TLS （从开始加密）在端口 465
  • 默认连接类型 - 对于新配置更改为 Direct SSL/TLS
  • 特定于连接的要求 - From Address 字段对于 Plain SMTP 连接是必需的，以确保正确的电子邮件发件人标识和 RFC 5322 合规性

🐛 错误修复​

电子邮件系统修复​

  • 修复缺少 From 标头 - 通过确保所有电子邮件消息都包含正确的 From 标头来解决 RFC 5322 合规性错误
  • 修复 STARTTLS 的电子邮件加密 - 更正了 STARTTLS 连接（端口 587）的加密处理
  • 修复普通 SMTP 连接 - 解决了导致 Plain SMTP 连接失败的 TLS 协商错误
  • 修复未保存的配置丢失 - 防止在更改密码时清除未保存的电子邮件配置
  • 修复过时的 SMTP 配置 - 解决了测试电子邮件端点使用过时的 SMTP 配置的问题

用户界面修复​

  • 修复了密码可见性切换 - 修正了密码更改表单中查看密码功能的行为；点击眼睛图标现在同时显示两个密码字段的密码

系统修复​

  • 修复了Docker入口日志记录 - 解决了Docker入口脚本丢失日志消息的问题
  • 修复了下次运行日期计算 - 添加了自动重新计算Next Run日期的功能，在收集备份日志后，确保日期在日志收集后正确更新

🚀 迁移说明​

从版本0.9.x​

此版本包括电子邮件配置和服务器管理的增强功能。当从版本0.9.x升级时:

  1. 无需数据库迁移 - 此版本不需要任何数据库模式更改
  2. 电子邮件配置更新 - 现有的电子邮件配置将继续工作，但您可能需要查看和更新设置以利用新功能:
    • 考虑设置自定义Sender Name以便于电子邮件识别
    • 确保您的From Address设置正确，特别是当使用Plain SMTP连接时
    • 使用增强的测试电子邮件功能测试电子邮件配置
  3. 连接类型默认 - 新的电子邮件配置将默认为Direct SSL/TLS连接类型
  4. 服务器密码管理 - 您现在可以直接从服务器详细信息页面更改服务器密码，而无需重新收集备份

电子邮件配置建议​

  • 用于生产环境: 使用STARTTLS或Direct SSL/TLS连接进行安全的电子邮件传递
  • 用于本地测试: Plain SMTP连接可用于受信任的本地网络
  • 发件人地址要求: 当使用Plain SMTP或身份验证被禁用时，确保From Address字段正确配置
  • 测试配置: 始终使用"发送测试电子邮件"按钮来验证电子邮件设置正确

API 端点​

电子邮件配置端点​

  • GET /api/email/config - 获取电子邮件配置
  • POST /api/email/config - 更新电子邮件配置
  • POST /api/email/test - 发送测试电子邮件
  • DELETE /api/email/config - 删除电子邮件配置

服务器管理端点​

  • PATCH /api/servers/{id}/password - 更新服务器密码
  • POST /api/database/maintenance/merge-duplicate-servers - 合并重复服务器

报告错误​

当报告错误时，请包括：

  • 版本：1.0.x（指定确切版本，例如1.0.3）
  • 操作系统和版本
  • Docker/podman 版本
  • 容器类型（Docker 或 podman/Pod）
  • 错误消息和日志
  • 重现步骤
  • 电子邮件配置详细信息（如果与电子邮件相关）
  • Podman 配置详细信息（如果在 podman/Pod 中运行）
