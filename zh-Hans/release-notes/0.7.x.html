版本 0.7.x

概览​

版本 0.7.x 是一个主要版本，它引入了对数据库模式、API 结构和用户界面的重大改进。该版本包括需要更新外部集成的破坏性变化。

重大变化​

数据库模式变化​

  • 表重命名: machines 表重命名为 servers
  • 新字段: 添加了 alias、notes、created_at 和 updated_at 字段
  • API 响应变化: 更新字段名称以保持一致性

向后不兼容的 API 变化​

  • 字段重命名: totalMachines → totalServers
  • 对象重命名: machine → server 在 API 响应中
  • 字段重命名: backup_types_count → backupJobsCount
警告

使用 API 的外部集成必须更新 以便与版本 0.7.x 兼容。请参阅 向后不兼容的 API 变化 指南以获取详细信息。

新功能​

提升的服务器管理​

  • 服务器别名: 为服务器赋予友好、易记的名称
  • 服务器备注: 添加描述和其他信息
  • 改进的状态跟踪: 更好的服务器状态监控
  • 增强的服务器详细信息: 更全面的服务器信息

改进的 API 结构​

  • 一致的响应格式: 标准化的 API 响应结构
  • 更好的错误处理: 改进的错误消息和代码
  • 增强的文档: 更好的 API 文档
  • 新端点: 额外的 API 端点以实现更好的集成

数据库改进​

  • 更好的性能: 优化的数据库查询和索引
  • 数据完整性: 增强的数据验证和约束
  • 迁移系统: 自动化的数据库迁移系统
  • 备份系统: 改进的数据库备份和恢复

增强​

用户界面​

  • 现代设计: 更新的 UI，具有更好的视觉设计
  • 响应式布局: 更好的移动设备和平板电脑支持
  • 改进的导航: 增强的导航和用户体验
  • 更好的图表: 改进的图表可视化和交互

通知系统​

  • 增强的模板: 更好的通知消息模板
  • 改进的传递: 更可靠的通知传递
  • 更好的错误处理: 改进的通知错误处理
  • 增强的日志记录: 更好的日志记录以用于通知调试

性能​

  • 更快的查询: 优化的数据库查询
  • 更好的缓存: 改进的缓存机制
  • 减少的内存使用: 更好的内存管理
  • 更快的 API 响应: 改进了 API 响应时间

错误修复​

数据库问题​

  • 修复了数据库损坏问题
  • 解决了升级过程中的迁移问题
  • 修复了特定场景下的数据丢失问题
  • 改进了数据库备份和恢复程序

API 问题​

  • 修复了 API 响应不一致问题
  • 解决了身份验证问题
  • 修复了数据验证问题
  • 改进了错误处理和消息

用户界面​

  • 修复了仪表板加载问题
  • 解决了图表渲染问题
  • 修复了移动布局问题
  • 改进了错误消息显示

通知系统​

  • 修复了通知投递失败问题
  • 解决了电子邮件配置问题
  • 修复了 NTFY 集成问题
  • 改进了通知模板渲染

迁移指南​

自动迁移​

从 0.6.x 到 0.7.x 的迁移过程是自动的:

  1. 备份创建: 自动创建数据库备份
  2. 模式更新: 自动更新数据库模式
  3. 数据迁移: 保留和迁移现有数据
  4. 验证: 验证和记录迁移

手动步骤​

  • 更新外部集成: 更新使用 API 的外部工具
  • 测试功能: 验证所有功能是否正常工作
  • 更新文档: 更新任何自定义文档

配置更改​

新的配置选项​

# Server management
servers:
  defaultAlias: true
  showNotes: true
  sortBy: "name" # name, status, lastBackup

# API settings
api:
  version: "1.0"
  rateLimit: 100
  timeout: 30000

# Database settings
database:
  backupRetention: 7
  autoVacuum: true
  optimizeInterval: 24

废弃的选项​

  • machines 配置（使用 servers 代替）
  • 旧的 API 字段名称（使用新的字段名称）

Docker 镜像​

可用的镜像​

  • Docker Hub: wsjbr/duplistatus:0.7.x
  • GitHub 容器注册表: ghcr.io/wsj-br/duplistatus:0.7.x
  • 最新标签: wsjbr/duplistatus:latest

镜像大小​

  • Alpine: 52MB
  • Debian: 85MB
  • 多架构: 支持 ARM64 和 AMD64

安装​

Docker Compose​

services:
  duplistatus:
    image: wsjbr/duplistatus:0.7.x
    container_name: duplistatus
    restart: unless-stopped
    ports:
      - "9666:9666"
    volumes:
      - duplistatus_data:/app/data
    environment:
      - TZ=UTC

Docker CLI​

docker run -d \
  --name duplistatus \
  --restart unless-stopped \
  -p 9666:9666 \
  -v duplistatus_data:/app/data \
  wsjbr/duplistatus:0.7.x

升级说明​

从版本 0.6.x 升级​

  1. 备份数据 (升级过程中自动创建备份)
  2. 更新 Docker 镜像: docker pull wsjbr/duplistatus:0.7.x
  3. 重启容器: docker-compose down && docker-compose up -d
  4. 监控日志: docker logs -f duplistatus
  5. 更新集成: 更新使用 API 的外部工具
  6. 验证升级: 检查所有功能是否正常工作

从版本 0.5.x 或更早版本升级​

  1. 首先升级到 0.6.x (如果可用)
  2. 然后升级到 0.7.x 使用上述步骤

已知问题​

迁移问题​

  • 大型数据库: 迁移可能需要更长时间
  • 磁盘空间: 确保有足够的磁盘空间用于迁移备份
  • 权限: 检查数据库访问的文件权限

API 兼容性​

  • 外部工具: 使用 API 的外部工具需要更新
  • 字段名称: 一些字段名称已更改
  • 响应格式: 一些响应格式已更改

解决方法​

  • 迁移: 确保有足够的磁盘空间和时间用于迁移
  • API: 更新外部集成以使用新的 API 格式
  • 测试: 升级后测试所有功能
