安装指南

应用程序可以使用 Docker、Portainer Stacks 或 Podman 进行部署。安装完成后，您可能需要配置 TIMEZONE，如 配置时间区 中所述，并需要配置 Duplicati 服务器以将备份日志发送到 duplistatus，如 Duplicati 配置 部分中所述。

先决条件​

确保您已安装以下内容:

  • Docker Engine - Debian 安装指南
  • Docker Compose - Linux 安装指南
  • Portainer（可选）- Docker 安装指南
  • Podman（可选）- 安装指南

身份验证​

duplistatus 从版本 0.9.x 开始需要用户身份验证。默认 admin 账户在安装应用程序或从早期版本升级时自动创建：

  • 用户名：admin
  • 密码：Duplistatus09

您可以在 设置 > 用户 中创建额外的用户账户，第一次登录后。

重要

系统强制执行最小密码长度和复杂性。这些要求可以使用 PWD_ENFORCE 和 PWD_MIN_LEN 环境变量 进行调整。使用不够复杂或长度较短的密码可能会损害安全性。请谨慎使用这些设置。

容器镜像​

您可以使用来自以下位置的镜像:

  • Docker Hub：docker.io/wsjbr/duplistatus:latest
  • GitHub 容器注册表：ghcr.io/wsj-br/duplistatus:latest

选项 1：使用 Docker Compose​

这是本地部署或自定义配置的推荐方法。它使用 docker compose 文件来定义和运行容器及其所有设置。

# download the compose file
wget https://github.com/wsj-br/duplistatus/raw/refs/heads/master/production.yml -O duplistatus.yml
# start the container
docker compose -f duplistatus.yml up -d

检查 时区 部分以获取有关如何调整时区和数字/日期/时间格式的更多详细信息。

选项 2：使用 Portainer Stacks（Docker Compose）​

  1. 前往您的 Portainer 服务器中的“栈”并点击“添加栈”。
  2. 为您的栈命名（例如，“duplistatus”）。
  3. 选择“构建方法”为“网页编辑器”。
  4. 复制并粘贴以下内容到网页编辑器中：
# duplistatus production compose.yml
services:
  duplistatus:
    image: ghcr.io/wsj-br/duplistatus:latest
    container_name: duplistatus
    restart: unless-stopped
    environment:
      - TZ=Europe/London
      - PWD_ENFORCE=true
      - PWD_MIN_LEN=8
    ports:
      - "9666:9666"
    volumes:
      - duplistatus_data:/app/data
    networks:
      - duplistatus_network

networks:
  duplistatus_network:
    driver: bridge

volumes:
  duplistatus_data:
    name: duplistatus_data
  1. 检查 时区 部分以获取有关如何调整时区和数字/日期/时间格式的更多详细信息。
  2. 单击“部署栈”。

选项 3：使用 Portainer Stacks（GitHub 存储库）​

  1. 在 Portainer 中，转到“栈”并单击“添加栈”。
  2. 为栈命名（例如“duplistatus”）。
  3. 选择“构建方法”作为“存储库”。
  4. 输入存储库 URL：https://github.com/wsj-br/duplistatus.git
  5. 在 "Compose path" 字段中，输入：production.yml
  6. （可选）设置 "Environment variables" 部分中的 TZ、LANG、PWD_ENFORCE 和 PWD_MIN_LEN 环境变量。检查 Timezone 部分以获取有关如何调整时区和数字/日期/时间格式的更多详细信息。
  7. 点击 "Deploy the stack"。

选项 4：使用 Docker CLI​

# Create the volume
docker volume create duplistatus_data

# Start the container
docker run -d \
  --name duplistatus \
  -p 9666:9666 \
  -e TZ=Europe/London \
  -e LANG=en_GB \
  -v duplistatus_data:/app/data \
  ghcr.io/wsj-br/duplistatus:latest
  • duplistatus_data 卷用于持久存储。容器镜像使用 Europe/London 作为默认时区和 en_GB 作为默认语言（语言）。

选项 5：使用 Podman（CLI）rootless​

对于基本设置，您可以在不配置 DNS的情况下启动容器：

mkdir -p ~/duplistatus_data
# Start the container (standalone)
podman run -d \
  --name duplistatus \
  --userns=keep-id \
  -e TZ=Europe/London \
  -e LANG=en_GB \
  -p 9666:9666 \
  -v ~/duplistatus_data:/app/data \
  ghcr.io/wsj-br/duplistatus:latest

为 Podman 容器配置 DNS​

如果您需要自定义 DNS 配置（例如，用于 Tailscale MagicDNS、企业网络或自定义 DNS 设置），您可以手动配置 DNS 服务器和搜索域。

查找您的 DNS 配置：

  1. 对于 systemd-resolved 系统（大多数现代 Linux 发行版）：

    # Get DNS servers
    resolvectl status | grep "DNS Servers:" | awk '{print "--dns " $3}'

    # Get DNS search domains
    resolvectl status | grep "DNS Domain:" | awk '{print "--dns-search " $3}'

  2. 对于非 systemd 系统 或作为后备：

    cat /run/systemd/resolve/resolv.conf 2>/dev/null || cat /etc/resolv.conf

查找以 nameserver 开头的行（用于 DNS 服务器）和 search 开头的行（用于搜索域）。如果您不确定 DNS 设置或网络搜索域，请咨询您的网络管理员以获取此信息。

带有 DNS 配置的示例：

mkdir -p ~/duplistatus_data
# Start the container with DNS configuration
podman run -d \
  --name duplistatus \
  --userns=keep-id \
  --dns 100.100.100.100 \
  --dns-search example.com \
  -e TZ=Europe/London \
  -e LANG=en_GB \
  -p 9666:9666 \
  -v ~/duplistatus_data:/app/data \
  ghcr.io/wsj-br/duplistatus:latest

您可以通过添加多个 --dns 标志来指定多个 DNS 服务器：

--dns 8.8.8.8 --dns 1.1.1.1

您可以通过添加多个 --dns-search 标志来指定多个搜索域：

--dns-search example.com --dns-search internal.local

注释：配置 DNS 服务器时，请跳过包含 : 的 IPv6 地址和类似 127.0.0.53 的 localhost 地址。

检查 Timezone 部分以获取有关如何调整时区和数字/日期/时间格式的更多详细信息。

选项 6：使用 Podman Pods​

Podman Pods 允许您在共享网络命名空间中运行多个容器。这对于测试或在您需要在其他容器旁边运行 duplistatus 时很有用。

基本 Pod 设置：

mkdir -p ~/duplistatus_data

# Create the pod
podman pod create --name duplistatus-pod --publish 9666:9666/tcp

# Create the container in the pod
podman create --name duplistatus \
  --pod duplistatus-pod \
  --user root \
  -e TZ=Europe/London \
  -e LANG=en_GB \
  -v ~/duplistatus_data:/app/data \
  ghcr.io/wsj-br/duplistatus:latest

# Start the pod
podman pod start duplistatus-pod

为 Podman Pods 配置 DNS​

使用容器组时，DNS 配置必须设置在容器组级别，而不是容器级别。 使用与选项 5 中描述的相同方法来找到您的 DNS 服务器和搜索域。

带有 DNS 配置的示例：

mkdir -p ~/duplistatus_data

# Create the pod with DNS configuration
podman pod create --name duplistatus-pod \
  --publish 9666:9666/tcp \
  --dns 100.100.100.100 \
  --dns-search example.com

# Create the container in the pod
podman create --name duplistatus \
  --pod duplistatus-pod \
  --user root \
  -e TZ=Europe/London \
  -e LANG=en_GB \
  -v ~/duplistatus_data:/app/data \
  ghcr.io/wsj-br/duplistatus:latest

# Start the pod
podman pod start duplistatus-pod

管理容器组：

# Stop the pod (stops all containers in the pod)
podman pod stop duplistatus-pod

# Start the pod
podman pod start duplistatus-pod

# Remove the pod and all containers
podman pod rm -f duplistatus-pod

基本配置​

  1. 配置您的 Duplicati 服务器 将备份日志消息发送到 duplistatus （必填）。
  2. 登录到 duplistatus - 参见 用户指南 中的说明。
  3. 收集初始备份日志 - 使用 收集备份日志 功能用所有 Duplicati 服务器的历史备份数据填充数据库。这也会根据每个服务器的配置自动更新备份监控间隔。
  4. 配置服务器设置 - 在 设置 → 服务器 中设置服务器别名和注释，以使您的仪表盘更具信息量。
  5. 配置 NTFY 设置 - 在 设置 → NTFY 中设置 NTFY 通知。
  6. 配置电子邮件设置 - 在 设置 → 电子邮件 中设置电子邮件通知。
  7. 配置备份通知 - 在 设置 → 备份通知 中设置每个备份或每个服务器的通知。

请参阅以下部分以配置诸如时区、数字格式和 HTTPS 之类的可选设置。
