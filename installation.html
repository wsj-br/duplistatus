Installation Guide

The application can be deployed using Docker, Portainer Stacks, or Podman. After the installation, you may want to configure the TIMEZONE and LANGUAGE, as described in the Configure Timezone and Language and need to configure the Duplicati servers to send backup logs to duplistatus, as outlined in the Duplicati Configuration section.

Prerequisites​

Ensure you have the following installed:

  • Docker Engine - Debian installation guide
  • Docker Compose - Linux installation guide
  • Portainer (optional) - Docker installation guide
  • Podman (optional) - Installation guide

Container Images​

You can use the images from:

  • Docker Hub: wsjbr/duplistatus:latest
  • GitHub Container Registry: ghcr.io/wsj-br/duplistatus:latest

Option 1: Using Docker Compose​

This is the recommended method for local deployments or when you want to customise the configuration. It uses a docker compose file to define and run the container with all its settings.

# create a folder for the compose file
mkdir duplistatus
cd duplistatus
# download the compose file
wget https://github.com/wsj-br/duplistatus/raw/refs/heads/master/production.yml -O compose.yml
# start the container
docker compose up -d

Check the Timezone and Locale section to more details on how to adjust the timezone and number/date/time format.

Option 2: Using Portainer Stacks (Docker Compose)​

  1. Go to "Stacks" in your Portainer server and click "Add stack".
  2. Name your stack (e.g., "duplistatus").
  3. Choose "Build method" as "Web editor".
  4. Copy and paste this in the web editor:
# duplistatus production compose.yml
services:
  duplistatus:
    image: wsjbr/duplistatus:latest
    container_name: duplistatus
    restart: unless-stopped
    ports:
      - "9666:9666"
    volumes:
      - duplistatus_data:/app/data
    networks:
      - duplistatus_network

networks:
  duplistatus_network:
    driver: bridge

volumes:
  duplistatus_data:
    name: duplistatus_data
  1. Check the Timezone and Locale section to more details on how to adjust the timezone and number/date/time format.
  2. Click "Deploy the stack".

Option 3: Using Portainer Stacks (GitHub Repository)​

  1. In Portainer, go to "Stacks" and click "Add stack".
  2. Name your stack (e.g., "duplistatus").
  3. Choose "Build method" as "Repository".
  4. Enter the repository URL: https://github.com/wsj-br/duplistatus.git
  5. In the "Compose path" field, enter: production.yml
  6. (optional) Set the TZ and LANG environment variables in the "Environment variables" section. Check the Timezone and Locale section to more details on how to adjust the timezone and number/date/time format.
  7. Click "Deploy the stack".

Option 4: Using Docker CLI​

# Create the volume
docker volume create duplistatus_data

# Start the container
docker run -d \
  --name duplistatus \
  -p 9666:9666 \
  -v duplistatus_data:/app/data \
  wsjbr/duplistatus:latest
  • The duplistatus_data volume is used for persistent storage.

Option 5: Using Podman with Pod (CLI)​

# Create a pod for the container
podman pod create --name Duplistatus --publish 9666:9666/tcp

# Create and start the container
podman create \
  --name duplistatus \
  --pod Duplistatus \
  --user root \
  -v /root/duplistatus_home/data:/app/data \
  ghcr.io/wsj-br/duplistatus:latest

# Start the pod (which starts the container)
podman pod start Duplistatus

Check the Timezone and Locale section to more details on how to adjust the timezone and number/date/time format.

Option 6: Using Podman Compose (CLI)​

# create a folder for the compose file
mkdir duplistatus
cd duplistatus
# download the compose file
wget https://github.com/wsj-br/duplistatus/raw/refs/heads/master/production.yml -O compose.yml
# start the container
podman-compose up -d

Check the Timezone and Locale section to more details on how to adjust the timezone and number/date/time format.
