Versão 0.9.x

Visão geral​

Versão 0.9.x é uma versão principal que introduz um Sistema Abrangente de Controle de Acesso de Usuários e Segurança. Esta versão transforma duplistatus de uma aplicação de acesso aberto em um sistema seguro e multi-usuário com controle de acesso baseado em funções, registro de auditoria abrangente e recursos de segurança de nível empresarial. Toda a funcionalidade existente agora é protegida por autenticação, garantindo que apenas usuários autorizados possam acessar e gerenciar o monitoramento de backup.

Mudanças Significativas​

Autenticação Obrigatória​

  • Todas as páginas agora exigem autenticação - Os usuários devem fazer login para acessar o aplicativo
  • Conta de administrador padrão é criada automaticamente:
    • Nome de usuário: admin
    • Senha: Duplistatus09 (deve ser alterada no primeiro login)
  • Sessões existentes são invalidadas - Todos os usuários devem fazer login após a atualização
  • Pontos de extremidade da API agora exigem autenticação - Integrações externas devem incluir cookies de sessão ou usar solicitações autenticadas
aviso

Todos os usuários devem entrar após atualizar para a versão 0.9.x. As credenciais padrão de admin são admin/Duplistatus09. Altere a senha imediatamente após o primeiro login.

Migração de Schema de Banco de Dados​

  • Migração automática de v3.1 para v4.0 para instalações existentes
  • Novas tabelas adicionadas: users, sessions, audit_log
  • Backup do banco de dados criado automaticamente antes da migração
  • Novas instalações iniciam diretamente com schema v4.0 (nenhuma migração necessária)

Novos Recursos​

Controle de Acesso do Usuário & Sistema de Segurança​

Sistema de Autenticação​

  • Funcionalidade de login/logout do usuário com gerenciamento seguro de sessão
  • Autenticação baseada em senha com hash bcrypt (fator de custo 12)
  • Mecanismo de bloqueio de conta (5 tentativas falhas, bloqueio de 15 minutos)
  • Alteração forçada de senha no primeiro login
  • Aplicação de política de senhas:
    • Mínimo de 8 caracteres
    • Pelo menos uma letra maiúscula
    • Pelo menos uma letra minúscula
    • Pelo menos um número
    • Caracteres especiais opcionais
  • Funcionalidade "Lembrar de mim" com persistência do nome de usuário no localStorage
  • Autenticação baseada em sessão com sessões armazenadas no banco de dados (substitui a memória)
  • Proteção CSRF integrada à autenticação
  • Cookies HTTP-only para segurança da sessão
  • Rastreamento de endereço IP e agente do usuário para monitoramento de segurança

Gerenciamento de usuários (Apenas Admin)​

  • Criação de usuário com geração automática de senha temporária
  • Lista de usuários com pesquisa, paginação e filtros
  • Edição de usuário (nome de usuário, status de administrador, exigência de alteração de senha)
  • Funcionalidade de redefinição de senha com exibição da senha temporária
  • Exclusão de usuário com proteções (impede a exclusão do último administrador)
  • Controle de acesso baseado em funções (funções Administrador/Usuário)
  • Indicadores de status (Ativo, Bloqueado, Deve alterar senha)
  • Rastreamento e exibição do último acesso

Sistema de Registro de Auditoria​

  • Trilha de auditoria abrangente para todas as alterações no sistema e ações do usuário
  • Visualizador de log de auditoria com filtragem avançada:
    • Filtro por intervalo de data
    • Filtro por usuário
    • Filtro por ação
    • Filtro por categoria
    • Filtro por status
  • Funcionalidade de exportação (formatos CSV e JSON)
  • Estatísticas e análise de log de auditoria
  • Período de retenção configurável (30-365 dias, padrão: 90 dias)
  • Tarefa agendada de limpeza automática (executa diariamente às 2h UTC)
  • API de limpeza manual com suporte a modo de teste
  • Registro de auditoria para:
    • Eventos de autenticação (login, logout, alterações de senha, bloqueios de conta)
    • Operações de gerenciamento de usuários (criação, atualização, exclusão, redefinições de senha)
    • Alterações de configuração (e-mail, NTFY, modelos, tolerância a atrasos, configurações de backup)
    • Operações de backup (coleta, exclusão, limpeza)
    • Gerenciamento de servidores (adição, atualização, exclusão)
    • Operações do sistema (migrações, limpezas, notificações)

Redesign da Página de Configurações​

  • Navegação lateral moderna com barra lateral recolhível
  • Persistência do estado da barra lateral por usuário (preferência de recolhida/expandida salva por usuário)
  • Seções de configurações agrupadas:
    • Notificações: Notificações de Backup, Monitoramento de Backup, Modelos
    • Integrações: NTFY, E-mail
    • Sistema: Servidores, Usuários (somente administrador), Registro de Auditoria
  • Barra lateral fixa que permanece visível durante a rolagem
  • Design responsivo com espaçamento otimizado
  • Ícone de configurações e título "Configurações do Sistema" no cabeçalho da barra lateral
  • Botão Voltar integrado ao cabeçalho do aplicativo

Melhorias da Interface do Usuário​

  • Página de login independente com design moderno
  • Caixa de seleção "Lembrar de mim" no formulário de login para manter o nome de usuário
  • Botões Mostrar/Ocultar senha nos formulários de login e alteração de senha
  • Modal de alteração de senha com lista de verificação de validação em tempo real
  • Indicador de usuário e botão Sair no cabeçalho do aplicativo
  • Visibilidade da interface baseada em função (recursos de administrador ocultos de usuários comuns)
  • Bandeiras e indicadores de status em toda a interface
  • Preferências específicas por usuário armazenadas no localStorage (por exemplo, estado recolhido da barra lateral)

Recursos de Segurança​

  • Hash de senhas com bcrypt (padrão da indústria)
  • Sanitização de dados sensíveis nos logs de auditoria (senhas, tokens e segredos nunca são registrados)
  • Limitação de taxa para tentativas de login
  • Expiração de sessão (24 horas)
  • Validação de token CSRF para todas as operações que alteram estado
  • Ferramenta CLI de recuperação de administrador para redefinição de senha em caso de bloqueio

Ferramentas do Desenvolvedor​

  • Script CLI de recuperação de administrador (admin-recovery) para execução fácil em contêineres Docker
  • Funciona localmente e no Docker com detecção automática de caminho
  • Interfaces TypeScript abrangentes (sem tipos any)
  • Middleware de autenticação para proteção de rotas
  • Classe utilitária de registro de auditoria com métodos de conveniência

🚀 Notas de Migração​

Da Versão 0.8.x​

Esta versão introduz autenticação e requer que todos os usuários entrem. Quando atualizar da versão 0.8.x:

  1. Migração Automática do Banco de Dados: O aplicativo migrará automaticamente seu esquema do banco de dados da v3.1 para a v4.0
  2. Backup do Banco de Dados: Um backup automático é criado antes da migração
  3. Conta de Administrador Padrão: Uma conta de administrador padrão é criada:
    • Nome de usuário: admin
    • Senha: Duplistatus09
    • Você deve alterar esta senha no primeiro login
  4. Invalidação de Sessões: Todas as sessões existentes são invalidadas
  5. Login de Usuário Obrigatório: Todos os usuários devem fazer login após a atualização
  6. Autenticação de API: Integrações externas devem ser atualizadas para incluir autenticação

Considerações de Segurança​

  • Alterar Senha Padrão: Altere imediatamente a senha do administrador padrão após o primeiro login
  • Criar Contas de Usuário: Crie contas individuais para cada pessoa que precisar de acesso
  • Verificar Registros de Auditoria: Consulte os registros de auditoria regularmente para monitoramento de segurança
  • Configurar Retenção: Defina o período apropriado de retenção do registro de auditoria com base em seus requisitos de conformidade
  • Fazer Backup da Chave Mestra: Se estiver atualizando da versão 0.8.x, certifique-se de fazer backup do arquivo .duplistatus.key

Primeiros Passos de Login​

  1. Entrar com as credenciais padrão (admin / Duplistatus09)
  2. Alterar senha quando solicitado (obrigatório no primeiro login)
  3. Criar contas de usuário para outros usuários (Configurações → Usuários)
  4. Configurar retenção de log de auditoria se necessário (Configurações → Registro de Auditoria)
  5. Revisar registros de auditoria para verificar se o sistema está funcionando corretamente

🐛 Correções de Bugs​

  • Corrigidos problemas de persistência de sessão ao reiniciar o servidor
  • Melhorado o tratamento de erros para falhas de autenticação
  • Aprimorados os feedbacks de validação de senha
  • Corrigidas inconsistências na interface da página de configurações
  • Melhorado o desempenho do filtro de registros de auditoria

Pontos de Extremidade da API​

Pontos de Extremidade de Autenticação​

  • POST /api/auth/login - Autenticação de usuário
  • POST /api/auth/logout - Encerramento de sessão
  • GET /api/auth/me - Informações do usuário atual
  • POST /api/auth/change-password - Alteração de senha

Endpoints de Gerenciamento de usuários (Apenas Admin)​

  • GET /api/users - Listar usuários
  • POST /api/users - Criar usuário
  • PATCH /api/users/{id} - Atualizar usuário
  • DELETE /api/users/{id} - Excluir usuário

Endpoints de Log de Auditoria​

  • GET /api/audit-log - Consultar registros de auditoria
  • GET /api/audit-log/download - Exportar registros de auditoria
  • GET /api/audit-log/stats - Estatísticas de auditoria
  • POST /api/audit-log/cleanup - Limpeza manual (somente administrador)
  • GET /api/audit-log/retention - Obter configuração de retenção
  • PATCH /api/audit-log/retention - Atualizar retenção (somente administrador)
nota

Todos os endpoints da API agora exigem autenticação. Consulte a Referência da API para detalhes sobre autenticação.
