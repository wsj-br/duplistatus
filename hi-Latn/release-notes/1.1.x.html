Sanskaran 1.1.x

Overview​

Sanskaran 1.1.x ek laghu release hai jo vilambit backup ka pata lagane mein aane wali gambhir samasyao ko hal karti hai, Podman container anukoolata ko behtar banati hai, aur container architecture mein sudhar lati hai. Yah release vilambit backup ka pata lagane ki shuddhta ko theek karne aur Podman parivesh mein networking samasyao ko hal karne par kendrit hai.

New Features​

Upyogkarta Interface Sudhar​

  • Auto-refresh button par Right-click context menu - Application toolbar mein auto-refresh button par right-click karne se display pasand ke liye display settings page khul jata hai
  • Behtar collect backups modal - Jab koi server chuna jata hai to password field label ab spasht roop se "(Sirf tab bharein jab password badla ho)" darshata hai, jisse yah spasht ho jata hai ki kab field ki avashyakta hai

Dastavejikaran Sudhar​

  • Dastavejikaran hosting migration - Dastavejikaran ko Docker container image se GitHub Pages par https://wsj-br.github.io/duplistatus/user-guide/overview par sthanantarit kar diya gaya hai taki pahunch aur aasan updates mein sudhar ho sake
  • App header mein User Guide link ab bahari dastavejikaran site ki or ishara karta hai

🐛 Bug Fixes​

Gambhir: Vilambit Backup Pata Lagana - Pura Punarnirman​

  • Vilambit backup ka pata lagana theek kiya gaya - Ek naye algorithm ke saath vilambit backup ka pata lagane wale system ko poori tarah se punarnirmit kiya gaya hai jo apekshit backup tithiyon ko sahi dhang se track karta hai:
    • Algorithm sunishchit karta hai ki manual backup scheduled backup samay ko na badle (udaharan, 10pm ka manual backup 2pm ke schedule ko nahi badlega)
    • Backup monitoring settings mein "Aage ke chalan" column mein galat tithiyon ko dikhane wali samasya ko theek kiya gaya
    • Backup settings ab /api/upload ya /api/backups/collect dwara backup prapt karne ke turant baad update ho jati hain
    • Duplicati sync ke bina naye servers ke liye, schedule reference ke roop mein pehle backup ke samay ka upyog karta hai
    • Adhik vivaran ke liye, Vilambit Pata Lagane ka Algorithm dastavejikaran dekhein

Podman Container Fixes​

  • Podman pod networking theek kiya gaya - Podman pods mein chalne par application tak bahari pahunch ko rokne wali samasya ko hal kiya gaya. Server ab Podman pod aur standalone container donon modes mein sabhi interfaces (0.0.0.0:9666) par sahi dhang se bind hota hai
  • Tailscale hostnames ke liye DNS resolution theek kiya gaya - Podman containers mein DNS resolution samasyao ko hal kiya gaya jisse Tailscale hostnames ya custom DNS servers se judne mein rukavat aati thi. Nirdeshon ke saath install dastavejikaran ko update kiya gaya hai.

Email Configuration Fixes​

  • SMTP khali username/password handling theek kiya gaya - Is samasya ko hal kiya gaya jahan email configuration mein username ya password fields ko khali karne par bhi encrypted values database mein store ho jati thin. Khali fields ab sahi dhang se khali strings ke roop mein store hote hain aur sahi dhang se khali kiye ja sakte hain

Pranali Sudhar​

  • Console logging kam kiya gaya - AuditLogger ab kewal asafal login prayason ko console mein log karta hai; anya sabhi audit entries kewal database mein likhi jati hain

🚀 Migration Notes​

Sanskaran 1.0.x se​

Sanskaran 1.1.0 mein upgrade karte samay:

  1. Dastavejikaran Sthan Parivartan - Dastavejikaran ab Docker container mein shamil hone ke bajaye GitHub Pages par host kiya gaya hai. App header mein User Guide link ab bahari dastavejikaran site ki or ishara karta hai

  2. Podman Container Configuration - Yadi Podman pods ka upyog kar rahe hain, to sunishchit karein ki DNS configuration pod level par --dns aur --dns-search flags ka upyog karke set ki gayi hai. Application ab Podman pods aur standalone containers donon mein sahi dhang se kaam karta hai

  3. Vilambit Backup Pata Lagana - Vilambit backup ka pata lagane wale system ko ek naye algorithm ke saath poori tarah se punarnirmit kiya gaya hai jo schedule samay ko banaye rakhta hai. Maujooda backup monitoring configurations kaam karti rahengi, lekin aapko backup monitoring settings mein adhik shuddh "Aage ke chalan" tithiyan dikh sakti hain. Naya algorithm sunishchit karta hai ki manual backup scheduled backup samay ko nahi badalte hain

  4. Database Migration Ki Avashyakta Nahi - Is release ke liye kisi bhi database schema parivartan ki avashyakta nahi hai

API Endpoints​

Is release mein koi naye API endpoints nahi jode gaye hain. Sabhi maujooda endpoints avam parivartit hain.
