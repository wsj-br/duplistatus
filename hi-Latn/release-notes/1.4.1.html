Sanskaran 1.4.1

Overview​

Sanskaran 1.4.1 dashboard anubhav mein mahatvapurn sudhaar laata hai jismein inline chart niyantran, behtar filtering kshamataen, pranali vastu shaili samarthan, aur sthaniya format vikalp shamil hain. Yah release chart rendering, backup vivaran load hone, aur server connectivity ka pata lagane ko prabhavit karne wali kai trutiyon ko bhi theek karti hai. Iske alawa, dependencies mein suraksha kamzoriyon ko hal kar diya gaya hai.

Naye features​

Inline chart niyantran​

  • Chart samay pariman selector: Chart panel headers mein stock-chart-shaili ke pill buttons (1W | 2W | 1M | 3M) jo Display Settings mein navagate kiye bina tezi se samay pariman badalne ke liye hain. Display Settings ke saath state share karta hai.
  • Chart shaili toggle (mudda #54): Chart panel header se seedhe line aur bar charts ke beech switch karne ke liye toggle button. Sabhi panels mein persist aur sync hota hai.
  • Saral samay pariman: 7 vikalpon se ghatakar 4 kendrit pariman (1 Saptah, 2 Saptah, 1 Maah, 3 Maah) kar diye gaye hain, jismein 1 Maah naya default hai.

Display aur format vikalp​

  • Pranali vastu shaili samarthan: Display Settings mein ab "Pranali (OS ka palan karen)" vikalp shamil hai jo aapke operating system ki ujjwal/andhakar pasand ko svatah mel karta hai.
  • Format sthaniya override (mudda #59): Upyogkarta Settings > Display Settings mein UI bhaasha se svatantr ek formatting sthaniya chun sakte hain (416 sthaniya samarthit hain). Taareekh, samay, aur sankhya formatting chune hue format sthaniya ka palan karti hai jismein live preview hota hai.

Server filtering​

  • Dashboard server filter: Khoj field app header mein auto-refresh aur toolbar kriyaon ke saath sthanantarit. Control khali hone par kewal khoj icon dikhata hai; hover/click karne par text field khulta hai.
  • Behtar filter mel (mudda #53): Filter ab server id, URL, aur kisi bhi backup job naam se mel karta hai. Filter badalne par overview card grid update na hone ki samasya theek ki gayi.
  • Settings filtering: Settings > Backup Monitoring aur Settings > Servers mein server list mein filter input joda gaya hai.

Sudhaar​

Chart data handling​

  • Rolling time windows: Samay pariman ab rolling din windows ka upyog karte hain: 1S = antim 7 din, 2S = antim 14 din, 1M = antim 30 din, 3M = antim 90 din (pehle calendar-aadhaarit).
  • Data consolidation: Sabhi chart data points calendar din ke anusar consolidated kiye jaate hain bucketing se pehle. Ek din mein kai backups ko uchit roop se aggregate kiya jaata hai.
  • Limited data points: Behtar performance aur readability ke liye charts ab adhiktam 30 data points enforce karte hain.
  • Tooltip saralikaran: Tooltip ab samay aur taareekh (jaise "5/12/26") ke bajay kewal taareekh dikhata hai.

Backup scheduling​

  • UTC-aadhaarit gananaen: Agla apekshit backup aur vilambit pata lagana ab UTC mein intervals aur saptah ke din ki jaanch ko aage badhata hai, na ki sthaniya samay mein, jisse host timezone dwara DST paar karne par ek-ghante ki UTC drift se bacha jaata hai.
  • Badhi hui vilambit tolerance: Naye installs aur defaults ab vilambit tolerance ke liye 1 ghante ke bajay 2 ghante ka upyog karte hain.

Connectivity aur reliability​

  • Connectivity loss detection: /api/ping endpoint aur 30-second header-aadhaarit connectivity probe joda gaya hai. Refresh aur auto-refresh mein vifalta ab raw error text ko expose kiye bina lost-connection modal kholti hai.
  • UI preferences persistence: Race conditions ko theek kiya gaya hai jiske karan user UI preferences (overview side panel, chart samay pariman, chart shaili) page refresh hone par persist nahi hote the.

Dastaavez​

  • Duplicati remote access spashtikaran (mudda #40): Documentation ab spasht karti hai ki Duplicati remote access vaikalpik hai aur kewal tabhi avashyak hai jab Duplicati UI ke liye seedhe links ki avashyakta ho.

Bug fixes​

  • Overview charts panel samay pariman filter ko nazarandaaz kar raha hai: Panel dwara chune hue samay pariman ke bawajood sabhi data points dikhane ki samasya theek ki gayi. Panel ab uchit samay pariman parameters ke saath API se apna data fetch karta hai.
  • Backup details par OOM crash (mudda #62): Queries aur UI rendering ko optimize karke JavaScript heap out of memory ko rok diya gaya hai. Backup logs ab ek saath sabhi ko load karne ke bajay on-demand fetch kiye jaate hain, jismein adhiktam 1000 lines hote hain.
  • Backup version parsing (mudda #65): convertTimestampToISO() ab Duplicati dwara vibhinn system locales mein upyog ki jaane wali sabhi samanya locale-vishesh timestamp formats ko handle karta hai.

Suraksha​

Dependency vulnerability fixes​

  • npm audit vulnerabilities: 7 vulnerabilities resolved (1 uchch, 6 madhyam):

    • next ko 16.3.0-canary.19 mein update kiya gaya postcss@8.4.31 (XSS) ko theek karne ke liye
    • .pnpmfile.cjs ko serialize-javascript@^7.0.5 (RCE, DoS) ko force karne ke liye add kiya gaya
    • mermaid override ko >=11.14.1 mein update kiya gaya (4 madhyam vulnerabilities)
    • Spasht postcss@^8.5.14 aur serialize-javascript@^7.0.5 dependencies jodi gayin

  • pnpm overrides: Transitive uuid >=14.0.0 aur postcss >=8.5.10 ke liye root pnpm.overrides add kiya gaya.

  • Uchch-severity vulnerabilities: minimatch >=3.1.4 aur serialize-javascript >=7.0.3 ke liye pnpm overrides ke madhyam se 4 uchch-severity vulnerabilities theek ki gayin.

Migration notes​

Sanskaran 1.3.2 {#from-version-132} se​

Sanskaran 1.4.1 mein upgrade karte samay:

  1. Database migration ki avashyakta nahin: Is release ke liye kisi bhi database schema parivartan ki avashyakta nahin hai.
  2. Naya default vilambit tolerance: Naye installations mein 1 ghante ke bajay 2 ghante ka default vilambit tolerance hoga. Maujooda settings mein koi parivartan nahin hoga.
  3. Chart pasandeeda: Chart samay pariman aur shaili pasandeeda ab alag tarah se store kiye jate hain aur upgrade ke baad default (1 Mahina, Rekha chart) par reset ho jayenge.
  4. Dependencies: Updated lockfile aur overrides ko laagoo karne ke liye pnpm install run karen.
