Sanskaran 0.7.x

Overview​

Sanskaran 0.7.x ek pramukh release hai jo database schema, API structure, aur user interface mein mahatvapurna sudhaar laati hai. Is sanskaran mein breaking changes shaamil hain jinhein bahari integrations ko update karne ki zaroorat hai.

Breaking Changes​

Database Schema Changes​

  • Table Renaming: machines table ka naam badalkar servers kar diya gaya hai
  • New Fields: alias, notes, created_at, aur updated_at fields jode gaye hain
  • API Response Changes: Ekroopata ke liye field names update kiye gaye hain

Backward-incompatible API changes​

  • Field Renaming: totalMachines → totalServers
  • Object Renaming: API responses mein machine → server
  • Field Renaming: backup_types_count → backupJobsCount
warning

API ka istemaal karne wali bahari integrations ko sanskaran 0.7.x ke saath kaam karne ke liye update kiya jaana chahiye. Vivaran ke liye Backward-incompatible API changes guide dekhein.

New Features​

Enhanced Server Management​

  • Server Aliases: Servers ko anukool, yaad rakhne yogya naam dein
  • Server Notes: Vivaran aur atirikt jaankari jodein
  • Improved Status Tracking: Behtar server stithi monitoring
  • Enhanced Server Details: Adhik vyapak server jaankari

Improved API Structure​

  • Consistent Response Format: Standardised API response structure
  • Better Error Handling: Sudhare hue error sandesh aur codes
  • Enhanced Documentation: Behtar API documentation
  • New Endpoints: Behtar integration ke liye atirikt API endpoints

Database Improvements​

  • Better Performance: Optimised database queries aur indexing
  • Data Integrity: Enhanced data validation aur constraints
  • Migration System: Automated database migration system
  • Backup System: Sudhara hua database backup aur restore

Enhancements​

User Interface​

  • Modern Design: Behtar visual design ke saath updated UI
  • Responsive Layout: Behtar mobile aur tablet support
  • Improved Navigation: Enhanced navigation aur user experience
  • Better Charts: Sudhari hui chart visualisation aur interactivity

Notification System​

  • Enhanced Templates: Behtar notification sandesh templates
  • Improved Delivery: Adhik vishwasniya notification delivery
  • Better Error Handling: Notifications ke liye sudhara hua error handling
  • Enhanced Logging: Notification debugging ke liye behtar logging

Performance​

  • Faster Queries: Optimised database queries
  • Better Caching: Sudhare hue caching mechanisms
  • Reduced Memory Usage: Behtar memory management
  • Faster API Responses: Sudhare hue API response times

Trutiyon ka Sudhar​

Database Samasyaen​

  • Kuch kam mamlon mein database corruption ki samasyaon ko theek kiya gaya
  • Upgrades ke dauran migration ki samasyaon ko hal kiya gaya
  • Vishesh paristhitiyon mein data nuksan ki samasyaon ko theek kiya gaya
  • Database backup aur restore prakriyaon mein sudhar kiya gaya

API Samasyaen​

  • API response mein asangatiyon ko theek kiya gaya
  • Authentication ki samasyaon ko hal kiya gaya
  • Data validation ki samasyaon ko theek kiya gaya
  • Truti handling aur sandeshon mein sudhar kiya gaya

Upyogkarta Interface​

  • Dashboard loading ki samasyaon ko theek kiya gaya
  • Chart rendering ki samasyaon ko hal kiya gaya
  • Mobile layout ki samasyaon ko theek kiya gaya
  • Truti sandesh display mein sudhar kiya gaya

Sandesh Pranali​

  • Sandesh delivery mein vifaltaon ko theek kiya gaya
  • Email configuration ki samasyaon ko hal kiya gaya
  • NTFY integration ki samasyaon ko theek kiya gaya

Migration Guide​

Swachalit Migration​

0.6.x se 0.7.x tak migration swachalit roop se hoti hai:

  1. Backup Banaya Jana: Swachalit database backup banaya jata hai
  2. Schema Update: Database schema swachalit roop se update kiya jata hai
  3. Data Migration: Maujooda data surakshit rakha jata hai aur migrate kiya jata hai
  4. Pushti: Migration ki pushti ki jati hai aur log kiya jata hai

Anivarya Manual Kadam​

  • Bahari Integrations Update Karen: API ka upyog karne wale kisi bhi bahari tool ko update karen
  • Karyakshamata ka Parikshan Karen: Sabhi features sahi se kaam kar rahe hain, iski pushti karen
  • Documentation Update Karen: Kisi bhi anukoolit documentation ko update karen

Configuration Badlav​

Naye Configuration Vikalp​

# Server management
servers:
  defaultAlias: true
  showNotes: true
  sortBy: "name" # name, status, lastBackup

# API settings
api:
  version: "1.0"
  rateLimit: 100
  timeout: 30000

# Database settings
database:
  backupRetention: 7
  autoVacuum: true
  optimizeInterval: 24

Band Kiye Gaye Vikalp​

  • machines configuration (iske bajay servers ka upyog karen)
  • Purane API field naam (naye field naam ka upyog karen)

Upgrade Nirdesh​

Sanskaran 0.6.x se​

  1. Apne data ka backup lein (upgrade ke dauraan swachalit backup banaya jaata hai)
  2. Docker image update karein: docker pull wsjbr/duplistatus:0.7.x
  3. Container ko restart karein: docker-compose down && docker-compose up -d
  4. Logs ki nigrani karein: docker logs -f duplistatus
  5. Integrations update karein: API ka upyog karne wale kisi bhi bahari upkaran ko update karein
  6. Upgrade ki pushti karein: Jaanch karein ki sabhi features sahi dhang se kaam kar rahe hain

Sanskaran 0.5.x ya Usse Pehle se​

  1. Pehle 0.6.x mein upgrade karein (yadi uplabdh ho)
  2. Phir upar diye gaye steps ka upyog karke 0.7.x mein upgrade karein
