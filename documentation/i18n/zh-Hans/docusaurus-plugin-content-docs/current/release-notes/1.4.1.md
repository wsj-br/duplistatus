# 版本 1.4.1 {#version-141}

## 概览 {#overview}

版本 1.4.1 为仪表板体验带来了显著的改进，包括内联图表控制、增强的过滤功能、系统主题支持和区域设置格式选项。此版本还解决了多个影响图表渲染、备份详细信息加载和服务器连接检测的错误。此外，依赖项中的安全漏洞已得到解决。

---

## 新功能 {#new-features}

### 内联图表控制 {#inline-chart-controls}

- **图表时间范围选择器**：图表面板标题中的股票图表样式按钮（`1W | 2W | 1M | 3M`）可快速在不导航到显示设置的情况下切换时间范围。与显示设置共享状态。
- **图表样式切换（问题 #54）**：图表面板标题中的切换按钮可直接在图表面板标题中切换线图和柱状图。状态在所有面板中保持一致。
- **简化时间范围**：从 7 个选项减少到 4 个集中范围（1 周、2 周、1 个月、3 个月），默认为 1 个月。

### 显示和格式化选项 {#display-and-formatting-options}

- **系统主题支持**：显示设置现在包括 "系统（跟随操作系统）" 选项，以自动匹配操作系统的浅色/深色首选项。
- **格式区域设置覆盖（问题 #59）**：用户可以在设置 > 显示设置中独立于 UI 语言选择格式区域设置（支持 416 个区域设置）。日期、时间和数字格式化尊重所选的格式区域设置，并提供实时预览。

### 服务器过滤 {#server-filtering}

- **仪表板服务器过滤器**：搜索字段已移动到应用程序标题，并具有自动刷新和工具栏操作。当为空时，控件仅显示搜索图标；悬停/单击展开文本字段。
- **改进的过滤匹配（问题 #53）**：过滤器现在匹配服务器 ID、URL 和任何备份作业名称。修复了概览卡片网格在更改过滤器时未更新的问题。
- **设置过滤**：备份监控和服务器设置中的服务器列表添加了过滤输入。

## 改进 {#improvements}

### 图表数据处理 {#chart-data-handling}

- **滚动时间窗口**：时间范围现在使用滚动日窗口：1W = 最后 7 天，2W = 最后 14 天，1M = 最后 30 天，3M = 最后 90 天（以前基于日历）。
- **数据整合**：所有图表数据点在分配之前按日历天整合。同一天的多个备份适当聚合。
- **限制数据点**：图表现在强制执行最多 30 个数据点，以提高性能和可读性。
- **工具提示简化**：工具提示现在仅显示日期（例如 "5/12/26"），而不是日期和时间。

### 备份调度 {#backup-scheduling}

- **基于 UTC 的计算**：下一个预期备份和过期检测现在使用 UTC 而不是本地时间进行间隔和星期检查，避免了主机时区跨越夏令时时的 1 小时漂移。
- **增加过期容忍度**：新安装和默认值现在使用 2 小时而不是 1 小时作为过期容忍度。

### 连接性和可靠性 {#connectivity-and-reliability}

- **连接丢失检测**：添加了 `/api/ping` 端点和 30 秒的基于标题的连接探测。刷新和自动刷新失败现在打开丢失连接模态而不暴露原始错误文本。
- **UI 首选项持久性**：修复了导致用户 UI 首选项（概览侧面板、图表时间范围、图表样式）在页面刷新时未持久化的竞争条件。

### 文档 {#documentation}

- **Duplicati 远程访问说明（问题 #40）**：文档现在澄清 Duplicati 远程访问是可选的，只有在需要直接链接到 Duplicati UI 时才需要。

## 错误修复 {#bug-fixes}

- **概览图表面板忽略时间范围过滤器**: 修复了面板显示所有数据点而不考虑所选时间范围的问题。面板现在从 API 中获取自己的数据，并带有适当的时间范围参数。
- **备份详细信息的 OOM 崩溃（问题 #62）**: 通过优化查询和 UI 渲染来防止 `JavaScript heap out of memory`。备份日志现在按需获取，而不是一次性加载所有日志，最大限制为 1000 行。
- **备份版本解析（问题 #65）**: `convertTimestampToISO()` 现在可以处理 Duplicati 在不同系统区域设置中使用的所有常见的本地化时间戳格式。

---

## 安全 {#security}

### 依赖性漏洞修复 {#dependency-vulnerability-fixes}

- **npm 审计漏洞**: 解决了 7 个漏洞（1 个高危，6 个中等危险）:
  - 更新 `next` 到 `16.3.0-canary.19` 以修复 `postcss@8.4.31`（XSS）
  - 添加 `.pnpmfile.cjs` 以强制 `serialize-javascript@^7.0.5`（RCE，DoS）
  - 更新 `mermaid` 重写为 `>=11.14.1`（4 个中等危险漏洞）
  - 添加显式 `postcss@^8.5.14` 和 `serialize-javascript@^7.0.5` 依赖

- **pnpm 重写**: 添加根 `pnpm.overrides` 以用于传递 `uuid >=14.0.0` 和 `postcss >=8.5.10`。

- **高危险漏洞**: 通过 pnpm 重写修复了 `minimatch >=3.1.4` 和 `serialize-javascript >=7.0.3` 的 4 个高危险漏洞。

## 迁移说明 {#migration-notes}

### 从版本 1.3.2 {#from-version-132}

升级到版本 1.4.1 时:

1. **无需数据库迁移**: 本版本不需要任何数据库模式更改。
2. **新的默认过期容忍度**: 新安装将默认使用 2 小时过期容忍度，而不是 1 小时。现有的设置保持不变。
3. **图表首选项**: 图表时间范围和样式首选项现在存储方式不同，并将在升级后重置为默认值（1 个月，折线图）。
4. **依赖**: 运行 `pnpm install` 以应用更新的锁定文件和重写。
