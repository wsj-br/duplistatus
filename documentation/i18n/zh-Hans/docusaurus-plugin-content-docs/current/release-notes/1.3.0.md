# 版本 1.3.0 {#version-130}

## 概览 {#overview}

版本 1.3.0 引入了可配置的密码策略、新的应用程序日志查看器、基于区域设置的星期显示以及几个用户体验改进。该版本专注于增强的管理工具和改进的国际化支持。

## 新功能 {#new-features}

### 可配置的密码策略 {#configurable-password-policy}

- **密码策略环境变量**：通过环境变量配置密码验证要求：
  - `PWD_ENFORCE`：设置为 `false` 以禁用密码复杂性要求（大写、 小写、数字）。默认情况下强制执行所有规则。当禁用时，只执行最小长度检查。
  - `PWD_MIN_LEN`：设置最小密码长度（字符）。默认为 8 个字符。
  - 默认密码检查（防止使用默认管理员密码）始终被强制执行，无论 `PWD_ENFORCE` 设置如何。

### 应用程序日志查看器 {#application-logs-viewer}

- **新管理员日志查看器**：直接从设置页面查看和下载应用程序日志：
  - 访问综合 `application.log` 文件和轮换日志版本
  - 可配置的行数选择器（100、500、1000、5000、10000 行）
  - 实时搜索过滤器，支持大小写不敏感的文本匹配
  - 自动滚动切换，用于实时日志监控，具有自动轮询
  - 复制到剪贴板并导出过滤日志作为文本文件
  - 文件元数据显示，显示行数、文件大小和最后修改时间戳
  - 容器启动时的日志轮换（通过 `LOG_ROTATION_VERSIONS` 配置，默认：5）

### 上下文感知用户指南 {#context-aware-user-guide}

- **动态帮助按钮**：应用程序头部的用户指南按钮现在打开与当前页面相关的文档。工具提示更新为显示 "`Page Name` 的帮助" 以指示将要打开的文档。

### 服务器配置增强 {#server-configuration-enhancement}

- **快速访问 Duplicati UI**：右键单击服务器配置按钮或服务器选择弹出窗口中的任何服务器项，会在新标签页中打开 Duplicati 旧 UI（`/ngax`）。

## 🔧 改进 {#-improvements}

### 基于区域设置的星期显示 {#locale-aware-weekday-display}

- **国际化星期顺序**：备份监控表单中的星期显示现在尊重浏览器的区域设置。星期几根据区域设置以正确的顺序显示（英国/欧洲为星期一至星期日，美国/巴西为星期日至星期六）。

### 增强的 SMTP 错误处理 {#enhanced-smtp-error-handling}

- **改进的错误消息**：SMTP 错误消息现在提供更准确的指导：
  - 身份验证错误清晰地指示用户名/密码问题
  - SSL/TLS 版本错误被正确识别
  - 连接类型建议仅在真正相关时出现

### Toast 通知行为 {#toast-notification-behaviour}

- **持久错误通知**：错误 toast 现在会持续存在，直到手动关闭，确保您不会错过重要的错误消息。成功和信息 toast 会在 3 秒后自动消失。

### 通知设置表 {#notification-settings-table}

- **简化布局**: 通知设置表已简化，合并了「服务器 / 备份」列，并使用上下文图标指示其他目标配置状态。

## 🚀 迁移说明 {#-migration-notes}

### 从版本 1.2.x {#from-version-12x}

升级到版本 1.3.0 时:

1. **密码策略**: 新的密码策略默认强制执行所有复杂性要求。如果您需要更简单的密码策略，请配置 `PWD_ENFORCE` 和 `PWD_MIN_LEN` 环境变量。

2. **应用程序日志**: 所有容器输出（服务器和 cron 服务）现在已合并到一个单独的 `data/logs/application.log` 文件中。

3. **日志轮换**: 日志轮换可以通过 `LOG_ROTATION_VERSIONS` 环境变量配置（默认：5 个版本）。

4. **无需数据库迁移**: 此版本不需要任何数据库模式更改。

## API 端点 {#api-endpoints}

以下 API 端点已在此版本中引入:

### 密码策略端点 {#password-policy-endpoint}

- `GET /api/auth/password-policy` - 返回当前密码策略配置

### 应用程序日志端点 {#application-logs-endpoints}

- `GET /api/application-logs` - 查看带有文件选择和尾部功能的日志
- `GET /api/application-logs/export` - 以文本文件形式导出过滤后的日志

### 删除的端点 {#removed-endpoints}

以下未使用的 API 端点已在此版本中删除:

- `DELETE /api/backups/[backupId]` - 从未使用的开发端点
- `GET /api/audit-log/stats` - 审计日志查看器从未调用过的端点
