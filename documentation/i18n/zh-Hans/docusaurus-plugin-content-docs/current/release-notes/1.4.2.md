# 版本 1.4.2 {#version-142}

## 概览 {#overview}

版本 1.4.2 是一个维护版本，专注于更广泛的语言支持、可靠性和依赖项维护。它引入了完全本地化的简体中文和印地语（罗马文）界面，将文档扩展到七种语言，并解决了一个重要的 Docker 启动崩溃问题，以及多个仪表板和服务器合并问题。依赖项也已更新，包括对 `nodemailer` 的重大升级。

---

## 新功能 {#new-features}

### 语言支持 {#language-support}

- **简体中文和印地语（罗马文）界面**：应用程序界面现已完全提供简体中文和印地语（罗马文），加入了英语、法语、德语、西班牙语和巴西葡萄牙语的行列。请从设置中的语言菜单选择它们。
- **扩展的文档语言**：在线文档现以七种语言发布，在现有翻译的基础上增加了印地语（罗马文）和简体中文。

---

## 改进 {#improvements}

### 备份收集 {#backup-collection}

- **收集日志时的稳定服务器标识**：在收集备份日志时，duplistatus 现在会将 Duplicati 计算机名称与现有服务器进行匹配并重用该服务器的记录，而不是依赖 Duplicati 的计算机 ID。这会将备份历史保留在同一服务器上，即使 Duplicati 的计算机 ID 发生变化（例如，重新安装后），从而减少重复服务器条目。

### 文档 {#documentation}

- **最新的语言列表**：README 和文档现在描述了所有七种支持的语言，并链接到每个翻译版本。
- **文档语言菜单标志**：为新的印地语和简体中文文档添加了标志图标，为了保持一致性，将葡萄牙语切换为葡萄牙国旗。标志现在可以在 Windows、Linux 和 Chromium 上可靠渲染，无需下载外部字体。
- **扩展的重复服务器指南**：用户指南现在解释了服务器重复的两个原因（Duplicati 计算机 ID 已更改，以及 Duplicati API 混淆了身份 ID 和计算机 ID 的错误），并记录了 Duplicati 端的变通方法以及 duplistatus 合并步骤。

### 仪表板性能 {#dashboard-performance}

- **加载时减少登录检查**：仪表板现在获取一次当前用户并在页面（页眉、设置守卫、主题和小部件）之间共享，而不是由每个组件单独请求，从而减少了页面加载时的冗余请求。

---

## 错误修复 {#bug-fixes}

- **Docker 启动崩溃**：修复了容器启动时报告 `Cannot find module '.../@swc/helpers/...'` 的崩溃问题，该问题可能导致应用程序无法启动。容器现在可以可靠启动。
- **重复的仪表板条目**：当多条记录共享相同的最新备份日期时，仪表板不再为同一备份显示多行。
- **合并重复服务器**：合并服务器现在还会合并共享相同备份名称的重复 Duplicati 备份 ID，并保留最新备份的 ID。合并操作现在记录在审计日志中。

---

## 安全 {#security}

### 依赖项更新 {#dependency-updates}

- **`nodemailer` 升级到 9.x**：将电子邮件库更新到最新的主版本。v9 中唯一的重大更改（在获取远程附件、OAuth2 或代理内容时进行更严格的 TLS 验证）不会影响 duplistatus，因为 duplistatus 仅发送内联文本和 HTML 电子邮件。
- **其他依赖项更新**：将多个构建和类型依赖项刷新到其最新的兼容版本。

---

## 迁移说明 {#migration-notes}

### 从版本 1.4.1 {#from-version-141}

升级到版本 1.4.2 时:

1. **无需数据库迁移**：此版本不需要任何数据库架构更改。
2. **依赖项**：运行 `pnpm install` 以应用更新的锁定文件和覆盖设置。
3. **无需更改配置**：现有设置（包括您选择的语言）将被保留。

### 版本 1.4.2 更改 {#version-142-changes}

- **已添加**：完整的简体中文和印地语（罗马化）用户界面。
