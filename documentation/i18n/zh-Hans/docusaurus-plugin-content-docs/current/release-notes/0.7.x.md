# 版本 0.7.x {#version-07x}

## 概览 {#overview}

版本 0.7.x 是一个主要版本，它引入了对数据库模式、API 结构和用户界面的重大改进。该版本包括需要更新外部集成的破坏性变化。

## 重大变化 {#breaking-changes}

### 数据库模式变化 {#database-schema-changes}
- **表重命名**: `machines` 表重命名为 `servers`
- **新字段**: 添加了 `alias`、`notes`、`created_at` 和 `updated_at` 字段
- **API 响应变化**: 更新字段名称以保持一致性

### 向后不兼容的 API 变化 {#backward-incompatible-api-changes}
- **字段重命名**: `totalMachines` → `totalServers`
- **对象重命名**: `machine` → `server` 在 API 响应中
- **字段重命名**: `backup_types_count` → `backupJobsCount`

:::warning
**使用 API 的外部集成必须更新** 以便与版本 0.7.x 兼容。请参阅 [向后不兼容的 API 变化](../migration/api-changes) 指南以获取详细信息。
:::

## 配置更改 {#configuration-changes}

### 新的配置选项 {#new-configuration-options}

```yaml
# Server management
servers:
  defaultAlias: true
  showNotes: true
  sortBy: "name" # name, status, lastBackup

# API settings
api:
  version: "1.0"
  rateLimit: 100
  timeout: 30000

# Database settings
database:
  backupRetention: 7
  autoVacuum: true
  optimizeInterval: 24
```

### 废弃的选项 {#deprecated-options}
- `machines` 配置（使用 `servers` 代替）
- 旧的 API 字段名称（使用新的字段名称）

## 安装 {#installation}

### Docker Compose {#docker-compose}

```yaml
services:
  duplistatus:
    image: wsjbr/duplistatus:0.7.x
    container_name: duplistatus
    restart: unless-stopped
    ports:
      - "9666:9666"
    volumes:
      - duplistatus_data:/app/data
    environment:
      - TZ=UTC
```

### Docker CLI {#docker-cli}

```bash
docker run -d \
  --name duplistatus \
  --restart unless-stopped \
  -p 9666:9666 \
  -v duplistatus_data:/app/data \
  wsjbr/duplistatus:0.7.x
```

## 升级说明 {#upgrade-instructions}

### 从版本 0.6.x 升级 {#from-version-06x}
1. **备份数据** (升级过程中自动创建备份)
2. **更新 Docker 镜像**: `docker pull wsjbr/duplistatus:0.7.x`
3. **重启容器**: `docker-compose down && docker-compose up -d`
4. **监控日志**: `docker logs -f duplistatus`
5. **更新集成**: 更新使用 API 的外部工具
6. **验证升级**: 检查所有功能是否正常工作

### 从版本 0.5.x 或更早版本升级 {#from-version-05x-or-earlier}
1. **首先升级到 0.6.x** (如果可用)
2. **然后升级到 0.7.x** 使用上述步骤
