# 版本 1.3.1 {#version-131}

## 概览 {#overview}

版本 1.3.1 为应用程序界面引入了完整的多语言支持，通过在更多端点上要求身份验证来加强 API 安全性，并修复了与主密钥检测、区域设置格式和通知设置相关的若干错误。此版本侧重于国际化改进和安全加固。

## 新功能 {#new-features}

### 多语言界面支持 {#multi-language-interface-support}

- **应用程序标题栏中的语言选择器**：从用户菜单的下拉列表中选择您首选的界面语言。支持的语言：英语、德语、法语、西班牙语和葡萄牙语（巴西）。您的选择将被保存，并在会话之间保持。
- **区域感知导航**：所有页面和链接现在都支持带区域前缀的 URL，以便在您选择的语言中进行一致的导航。

### 通知模板国际化 {#notification-templates-internationalization}

- **模板语言选择器**：通知模板现在通过专用的语言选择器支持多种语言。所有 5 种支持的语言均提供预翻译的默认模板。
- **重置选项**：“将此模板重置为默认值”和“全部重置为默认值”按钮带有确认对话框，以便在更改语言时轻松管理模板。
- **区域感知通知内容**：通知中的日期、时间、数字和备份状态值现在根据模板语言进行格式化。

### 日期和数字本地化 {#date-and-number-localization}

- **区域感知日期和时间格式**：整个应用程序中的日期和时间（仪表板、服务器详细信息、图表、工具提示）现在以适合您区域设置的格式显示（例如，欧洲区域设置为 DD/MM/YYYY，英语为 MM/DD/YYYY）。
- **区域感知数字格式**：数字、文件大小和统计信息使用适合区域设置的格式（例如，德语为 1.234,56，英语为 1,234.56）。

### RTL 支持准备 {#rtl-support-preparation}

- **从右到左语言的基础**：已添加基础设施以支持未来的 RTL 语言（阿拉伯语、希伯来语）。当前语言保持从左到右。

## 安全 {#security}

### API 身份验证 {#api-authentication}

- **在更多端点上要求身份验证**：以下端点现在需要有效的会话和 CSRF 令牌（以前身份验证是可选的）：
  - `/api/backups/collect`
  - `/api/backups/sync-schedule`
  - `/api/cron-config` (POST)
  - `/api/notifications/check-overdue`
  - `/api/notifications/clear-overdue-timestamps`
- **依赖项安全更新**：升级了易受攻击的依赖项（fast-xml-parser、brace-expansion）以解决已知的安全问题。

---

## 错误修复 {#bug-fixes}

- **主密钥更改检测**：修复了同时还原数据库和 `.duplistatus.key` 文件时（密钥相同）错误检测到“密钥已更改”的问题。系统现在使用密钥指纹进行可靠检测，而不是使用可能将数据损坏误解为密钥更改的基于解密的检查。
- **详细信息页面初始加载**：服务器详细信息页面现在在首次加载时正确显示逾期消息和数据，而不是仅在第一次自动刷新后才显示。

## 改进 {#improvements}

### 国际化一致性 {#internationalisation-consistency}

- **一致的标签**：时间单位、间隔、状态标签和常用术语现在在整个应用程序中保持一致。
- **区域设置感知的星期顺序**：备份监控中的星期显示尊重您的语言/区域设置，可以在显示设置中覆盖。

### 身份验证 API 错误和成功代码（i18n） {#auth-api-error-and-success-codes-i18n}

- **稳定的错误和成功代码**：身份验证端点（`/api/auth/login`、`/api/auth/logout`、`/api/auth/me`、`/api/auth/change-password`、`/api/auth/password-policy`）现在除了英文 `error` 或 `message` 字段外，还返回一个 `errorCode`（以及，在成功时，一个 `successCode`）。客户端使用这些代码在用户的语言中显示本地化消息。
- **登录页面**：所有登录错误消息（例如 "用户名和密码是必需的"、"用户名或密码无效"、"账户已锁定"）现在在登录页面上针对所有五个区域设置进行了翻译。
- **更改密码模态**：所有更改密码 API 错误和成功消息现在在更改密码模态中进行了翻译。
- **API 参考**：[身份验证和安全](../api-reference/authentication-security.md#auth-api-error-and-success-codes-i18n) API 文档列出了所有错误和成功代码，并解释了如何使用它们进行 i18n。

### UI 改进 {#ui-improvements}

- **服务器过滤器宽度**：在备份通知表单中增加了服务器过滤器宽度（260px → 360px），以提高可用性。
- **批量选择**：增强了批量选择 UI，支持备份计数的正确复数形式。
- **帮助工具提示**：改进了帮助工具提示文本，添加了页面名称上下文。

### 文档 {#documentation}

- **配置页面重命名**：时区和语言配置页面已从 `configure-tz-lang` 重命名为 `configure-tz`。
- **通知模板指南**：更新了用户指南，以记录新的模板语言选择器、重置按钮和语言行为。
- **多语言文档**：用户指南现在记录了如何通过用户菜单中的语言选择器更改界面语言。

## 迁移说明 {#migration-notes}

### 从版本 1.3.0 {#from-version-130}

升级到版本 1.3.1 时：

1. **API 身份验证**：调用备份收集、同步计划、cron 配置或通知检查/清除端点的脚本或工具现在必须包含有效的会话身份验证和 CSRF 令牌。

2. **无需数据库迁移**：此版本不需要任何数据库模式更改。

3. **区域设置支持**：环境变量 `LANG` 不再使用。您可以在 `Settings > Templates` 页面上定义通知语言。
