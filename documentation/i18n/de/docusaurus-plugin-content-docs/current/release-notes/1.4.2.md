# Version 1.4.2 {#version-142}

## Übersicht {#overview}

Version 1.4.2 ist ein Wartungsrelease, das auf Stabilität und Zuverlässigkeit fokussiert ist. Es reduziert redundante Authentifizierungsanfragen beim Laden des Dashboards, verbessert das Tool „Doppelte Server zusammenführen“, sodass nun auch doppelte Duplicati-Backup-IDs konsolidiert werden, und entfernt doppelte Zeilen aus dem Dashboard. Zudem werden Server-Zusammenführungen im Audit-Protokoll aufgezeichnet und Next.js auf eine stabile Version fixiert.

---

## Fehlerbehebungen {#bug-fixes}

- **Doppelte Einträge auf dem Dashboard**: Das Dashboard zeigt keine mehrfachen Zeilen für denselben Backup-Namen mehr an, wenn mehrere Datensätze dasselbe aktuellste Backup-Datum teilen.
- **Zusammenführen doppelter Server**: Das Zusammenführen von Servern konsolidiert nun auch doppelte Duplicati-Backup-IDs, die auf dem zusammengeführten Server denselben Backup-Namen haben; dabei wird die ID des aktuellsten Backups beibehalten, damit die Historie konsistent bleibt.

---

## Verbesserungen {#improvements}

### Performance {#performance}

- **Weniger Authentifizierungsanfragen beim Laden**: Der angemeldete Benutzer wird nun einmal abgerufen und auf der Seite (Header, Passwort-Schutz, Design-/Konfigurations-Provider und Dashboard-Widgets) geteilt, anstatt dass jede Komponente dies separat anfordert. Dies reduziert redundante Netzwerkaufrufe beim Öffnen des Dashboards.

### Auditierung {#auditing}

- **Server-Zusammenführungen im Audit-Protokoll aufgezeichnet**: Das Zusammenführen von Servern wird nun als `servers_merged`-Eintrag im Audit-Protokoll aufgezeichnet, um eine bessere Rückverfolgbarkeit zu gewährleisten.

### Benutzeroberfläche {#interface}

- **Layout von Doppelte Server zusammenführen**: Unter Einstellungen > Datenbankverwaltung > Doppelte Server zusammenführen werden die Bezeichnungen „Zielserver (neueste)“ und „Alte Server-ID“ nun als ordnungsgemäße Tabellenkopfzellen (designabhängig) anstelle von Pillen-/Button-Formen gerendert, was für ein klareres und konsistenteres Erscheinungsbild sorgt.

---

## Geändert {#changed}

- **Next.js stabiles Release**: Next.js ist nun auf das stabile `16.2.9`-Release fixiert (zuvor ein Canary-Build), um die Zuverlässigkeit zu verbessern.

---

## Migrationsnoten {#migration-notes}

### Ab Version 1.4.1 {#from-version-141}

Beim Upgrade auf Version 1.4.2:

1. **Keine Datenbankmigration erforderlich**: Dieses Release erfordert keine Änderungen am Datenbankschema.
2. **Docker-Benutzer**: Laden Sie das neueste Image herunter; die Behebung des Absturzes beim Start ist im aktualisierten Container-Build enthalten.

## Änderungsprotokoll {#changelog}

### Änderungen in Version 1.4.2 {#version-142-changes}

- **Behoben**: Startabsturz des Docker-Standalone-Containers durch eine fehlende `@swc/helpers` ESM-Hilfsdatei.
- **Behoben**: Doppelte Dashboard-Zeilen, wenn mehrere Datensätze dasselbe aktuellste Backup-Datum teilten.
- **Behoben**: Server-Zusammenführung konsolidiert nun doppelte Duplicati-Backup-IDs, die denselben Backup-Namen teilen.
- **Geändert**: Der angemeldete Benutzer wird einmal über einen gemeinsamen Provider abgerufen, was redundante `/api/auth/me`-Anfragen beim Laden reduziert.
- **Geändert**: Server-Zusammenführungsoperationen werden nun im Audit-Protokoll als `servers_merged` aufgezeichnet.
- **Geändert**: Bezeichnungen für Doppelte Server zusammenführen werden als Tabellenkopfzellen anstelle von Pillen-/Button-Formen gerendert.
- **Geändert**: Next.js auf das stabile `16.2.9`-Release fixiert.
