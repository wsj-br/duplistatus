# Version 1.4.2 {#version-142}

## Übersicht {#overview}

Version 1.4.2 ist eine Wartungsversion, die sich auf erweiterte Sprachunterstützung, Zuverlässigkeit und die Bereinigung von Abhängigkeiten konzentriert. Sie führt eine vollständig lokalisierte Benutzeroberfläche für vereinfachtes Chinesisch und Hindi (Latein) ein, erweitert die Dokumentation auf sieben Sprachen und behebt einen wichtigen Absturz beim Docker-Start sowie mehrere Probleme beim Zusammenführen von Dashboard und Server. Die Abhängigkeiten wurden ebenfalls aktualisiert, einschließlich eines großen Upgrades für `nodemailer`.

---

## Neue Funktionen {#new-features}

### Sprachunterstützung {#language-support}

- **Benutzeroberfläche auf vereinfachtem Chinesisch und Hindi (Latein)**: Die Anwendungsoberfläche ist nun vollständig auf vereinfachtem Chinesisch und Hindi (Latein) verfügbar, zusätzlich zu Englisch, Französisch, Deutsch, Spanisch und Brasilianischem Portugiesisch. Wählen Sie diese im Sprachmenü unter Einstellungen aus.
- **Erweiterte Dokumentationssprachen**: Die Online-Dokumentation wird nun in sieben Sprachen veröffentlicht, wobei Hindi (Latein) und vereinfachtes Chinesisch zu den bestehenden Übersetzungen hinzugefügt wurden.

---

## Verbesserungen {#improvements}

### Sicherungserfassung {#backup-collection}

- **Stabile Serveridentität bei der Protokollerfassung**: Beim Erfassen von Sicherungsprotokollen gleicht duplistatus nun den Duplicati-Maschinennamen mit Ihren vorhandenen Servern ab und verwendet den Datensatz dieses Servers erneut, anstatt sich auf die Duplicati-Maschinen-ID zu verlassen. Dadurch bleibt der Sicherungsverlauf mit demselben Server verknüpft, selbst wenn sich die Duplicati-Maschinen-ID ändert (z. B. nach einer Neuinstallation), wodurch doppelte Servereinträge reduziert werden.

### Dokumentation {#documentation}

- **Aktuelle Sprachliste**: Die README-Datei und die Dokumentation beschreiben nun alle sieben unterstützten Sprachen und verlinken auf jede übersetzte Version.
- **Flaggen im Sprachmenü der Dokumentation**: Flaggensymbole für die neue Hindi- und die vereinfachte Chinesisch-Dokumentation hinzugefügt und Portugiesisch aus Gründen der Konsistenz auf die Flagge von Portugal umgestellt. Die Flaggen werden nun zuverlässig unter Windows, Linux und Chromium dargestellt, ohne externe Schriftart-Downloads.
- **Erweiterte Anleitung zu doppelten Servern**: Das Benutzerhandbuch erklärt nun beide Ursachen für doppelte Server (eine geänderte Duplicati-Maschinen-ID und einen Duplicati-API-Fehler, der die Identitäts-ID und die Maschinen-ID vermischt) und dokumentiert die Problemumgehung auf Duplicati-Seite zusammen mit dem duplistatus-Zusammenführungsschritt.

### Dashboard-Leistung {#dashboard-performance}

- **Weniger Anmeldeprüfungen beim Laden**: Das Dashboard ruft jetzt Ihren aktuellen Benutzer einmal ab und teilt ihn auf der Seite (Header, Einstellungs-Guard, Design und Widgets) statt jedes Komponente fordert ihn separat an, was redundante Anfragen beim Laden einer Seite reduziert.

## Fehlerbehebungen {#bug-fixes}

- **Docker-Startabsturz**: Ein Absturz beim Containerstart, der `Cannot find module '.../@swc/helpers/...'` meldete, wurde behoben, wodurch das Starten der Anwendung verhindert werden konnte. Container starten jetzt zuverlässig.
- **Doppelte Dashboard-Einträge**: Das Dashboard zeigt jetzt nicht mehr mehrere Zeilen für dasselbe Backup an, wenn mehrere Datensätze dasselbe aktuelle Backup-Datum teilen.
- **Zusammenführen doppelter Server**: Beim Zusammenführen von Servern werden jetzt auch doppelte Duplicati-Backup-IDs konsolidiert, die denselben Backup-Namen teilen, wobei die ID aus dem jüngsten Backup beibehalten wird. Zusammenführungsoperationen werden jetzt im Audit-Protokoll aufgezeichnet.

---

## Sicherheit {#security}

### Abhängigkeitsaktualisierungen {#dependency-updates}

- **`nodemailer` auf 9.x aktualisiert**: Die E-Mail-Bibliothek wurde auf die neueste Hauptversion aktualisiert. Die einzige Breaking Change in v9 (strengere TLS-Validierung beim Abrufen von Remote-Anhängen, OAuth2 oder Proxy-Inhalten) betrifft duplistatus nicht, da dieser nur Inline-Text und HTML-E-Mails sendet.
- **Andere Abhängigkeitsaktualisierungen**: Mehrere Build- und Typabhängigkeiten wurden auf die neuesten kompatiblen Versionen aktualisiert.

## Migrationsnoten {#migration-notes}

### Ab Version 1.4.1 {#from-version-141}

Beim Upgrade auf Version 1.4.2:

1. **Keine Datenbankmigration erforderlich**: Dieses Release erfordert keine Änderungen am Datenbankschema.
2. **Abhängigkeiten**: Führen Sie `pnpm install` aus, um das aktualisierte Lockfile und die Überschreibungen anzuwenden.
3. **Keine Konfigurationsänderungen erforderlich**: Bestehende Einstellungen, einschließlich Ihrer gewählten Sprache, werden beibehalten.

## Änderungsprotokoll {#changelog}

### Änderungen in Version 1.4.2 {#version-142-changes}

- **Hinzugefügt**: Vollständige Benutzeroberfläche auf vereinfachtem Chinesisch und Hindi (Roman).
- **Hinzugefügt**: Dokumentation jetzt in sieben Sprachen verfügbar, einschließlich Hindi (Roman) und vereinfachtem Chinesisch.
- **Geändert**: Die Erfassung von Sicherungsprotokollen identifiziert Server nun anhand des Duplicati-Maschinennamens (Wiederverwendung vorhandener Datensätze), wodurch der Sicherungsverlauf auch bei Änderung der Maschinen-ID verknüpft bleibt.
- **Geändert**: Das Sprachmenü der Dokumentation fügt Flaggen für die neuen Gebietsschemata hinzu und verwendet die Flagge von Portugal für Portugiesisch.
- **Geändert**: Dokumentation und README aktualisiert, um den vollständigen Satz der unterstützten Sprachen zu beschreiben, und die Fehlerbehebung für doppelte Server erweitert.
- **Geändert**: `nodemailer` auf 9.x aktualisiert und andere Build-/Typ-Abhängigkeiten erneuert.
- **Behoben**: Docker-Startabsturz verursacht durch ein fehlendes `@swc/helpers`-Modul.
- **Behoben**: Doppelte Dashboard-Einträge, wenn mehrere Sicherungen dasselbe neueste Datum haben.
- **Behoben**: Das Zusammenführen doppelter Server konsolidiert nun doppelte Sicherungs-IDs und wird im Audit-Protokoll protokolliert.
- **Behoben**: Redundante Anmeldeprüfungen beim Laden des Dashboards reduziert.
