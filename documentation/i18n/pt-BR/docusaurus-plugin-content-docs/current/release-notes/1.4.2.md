# Versão 1.4.2 {#version-142}

## Visão geral {#overview}

Versão 1.4.2 é uma versão de manutenção focada em suporte mais amplo de idiomas, confiabilidade e higiene de dependências. Ela introduz uma interface totalmente localizada em Chinês Simplificado e Hindi (Romano), expande a documentação para sete idiomas e resolve uma falha importante de inicialização do Docker, além de vários problemas de mesclagem do dashboard e do servidor. As dependências também foram atualizadas, incluindo uma atualização importante para `nodemailer`.

---

## Novos recursos {#new-features}

### Suporte a idiomas {#language-support}

- **Interface em Chinês Simplificado e Hindi (Romano)**: A interface do aplicativo agora está totalmente disponível em Chinês Simplificado e Hindi (Romano), juntando-se ao inglês, francês, alemão, espanhol e português do Brasil. Selecione-os no menu de idioma em Configurações.
- **Idiomas de documentação expandidos**: A documentação online agora é publicada em sete idiomas, adicionando Hindi (Romano) e Chinês Simplificado às traduções existentes.

---

## Melhorias {#improvements}

### Coleção de backup {#backup-collection}

- **Identidade estável do servidor durante a coleta de logs**: Ao coletar logs de backup, o duplistatus agora compara o nome da máquina Duplicati com seus servidores existentes e reutiliza o registro desse servidor, em vez de depender do ID da máquina Duplicati. Isso mantém o histórico de backup vinculado ao mesmo servidor, mesmo que o ID da máquina Duplicati mude (por exemplo, após uma reinstalação), reduzindo entradas de servidores duplicados.

### Documentação {#documentation}

- **Lista atualizada de idiomas**: O README e a documentação agora descrevem todos os sete idiomas suportados e fornecem links para cada versão traduzida.
- **Bandeiras de menu de idioma da documentação**: Foram adicionados ícones de bandeira para a nova documentação em hindi e chinês simplificado, e a bandeira de Portugal foi usada para o português para manter a consistência. As bandeiras agora são renderizadas de forma confiável no Windows, Linux e Chromium, sem necessidade de downloads de fontes externas.
- **Orientação ampliada sobre servidores duplicados**: O guia do usuário agora explica ambas as causas de servidores duplicados (uma alteração no ID da máquina Duplicati e um bug na API do Duplicati que mistura o ID de identidade e o ID da máquina) e documenta o workaround do lado do Duplicati ao lado da etapa de mesclagem do duplistatus.

### Desempenho do dashboard {#dashboard-performance}

- **Menos verificações de login ao carregar**: O dashboard agora busca seu usuário atual uma vez e o compartilha pela página (cabeçalho, proteção de configurações, tema e widgets) em vez de cada componente solicitá-lo separadamente, reduzindo solicitações redundantes quando uma página carrega.

---

## Correções de bugs {#bug-fixes}

- **Falha na inicialização do Docker**: Corrigida uma falha ao iniciar o contêiner que relatava `Cannot find module '.../@swc/helpers/...'`, o que poderia impedir o aplicativo de iniciar. Os contêineres agora iniciam de forma confiável.
- **Entradas duplicadas no dashboard**: O dashboard não mostra mais várias linhas para o mesmo backup quando vários registros compartilham a mesma Data do Backup mais recente.
- **Mesclando Servidores Duplicados**: A mesclagem de servidores agora também consolida IDs de backup Duplicati duplicados que compartilham o mesmo Nome do Backup, mantendo o ID do backup mais recente. As operações de mesclagem agora são registradas no Log de Auditoria.

---

## Segurança {#security}

### Atualizações de dependências {#dependency-updates}

- **`nodemailer` atualizado para 9.x**: Atualizada a biblioteca de E-mail para a versão principal mais recente. A única mudança incompatível na v9 (validação TLS mais estrita ao buscar anexo remoto, conteúdo OAuth2 ou proxy) não afeta o duplistatus, que envia apenas texto em linha e E-mail HTML.
- **Outras atualizações de dependências**: Atualizadas várias dependências de compilação e tipo para suas versões compatíveis mais recentes.

---

## Notas de migração {#migration-notes}

### A partir da versão 1.4.1 {#from-version-141}

Ao atualizar para a versão 1.4.2:

1. **Nenhuma migração de banco de dados necessária**: Esta versão não requer nenhuma alteração no esquema do banco de dados.
2. **Dependências**: Execute `pnpm install` para aplicar o lockfile e as substituições atualizados.
3. **Nenhuma alteração de configuração necessária**: As configurações existentes, incluindo seu idioma escolhido, são preservadas.
