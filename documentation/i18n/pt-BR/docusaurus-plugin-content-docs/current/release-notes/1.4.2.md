# Versão 1.4.2 {#version-142}

## Visão geral {#overview}

A Versão 1.4.2 é um lançamento de manutenção focado em estabilidade e confiabilidade. Ela corrige um container Docker que podia travar na inicialização, reduz requisições de autenticação redundantes quando o dashboard é carregado, melhora a ferramenta "Mesclar Servidores Duplicados" para que também consolide IDs de backup do Duplicati duplicados e remove linhas duplicadas do dashboard. Além disso, registra a mesclagem de servidores no log de auditoria e fixa o Next.js em uma versão estável.

---

## Correções de bugs {#bug-fixes}

- **Travamento na inicialização do Docker**: O container podia travar imediatamente na inicialização com um erro `Cannot find module … @swc/helpers/esm/_interop_require_default.js`. O build autônomo agora agrupa os arquivos auxiliares necessários corretamente, para que o container inicie de forma confiável.
- **Entradas duplicadas no dashboard**: O dashboard não exibe mais múltiplas linhas para o mesmo nome de backup quando vários registros compartilham a mesma data de backup mais recente.
- **Mesclagem de servidores duplicados**: A mesclagem de servidores agora também consolida IDs de backup do Duplicati duplicados que compartilham o mesmo nome de backup no servidor mesclado, mantendo o ID do backup mais recente para que o histórico permaneça consistente.

---

## Melhorias {#improvements}

### Desempenho {#performance}

- **Menos requisições de autenticação no carregamento**: O usuário autenticado agora é buscado uma única vez e compartilhado por toda a página (cabeçalho, guarda de senha, provedores de tema/configuração e widgets do dashboard) em vez de cada componente solicitá-lo separadamente, reduzindo chamadas de rede redundantes quando o dashboard abre.

### Auditoria {#auditing}

- **Mesclagens de servidores registradas no log de auditoria**: A mesclagem de servidores agora é registrada como uma entrada `servers_merged` no Log de Auditoria para melhor rastreabilidade.

### Interface {#interface}

- **Layout de Mesclar Servidores Duplicados**: Em Configurações > Manutenção do Banco de Dados > Mesclar Servidores Duplicados, os rótulos "Servidor de Destino (mais recente)" e "ID do Servidor Antigo" agora são renderizados como células de cabeçalho de tabela adequadas (compatíveis com o tema) em vez de formatos de pílula/botão, para um visual mais claro e consistente.

---

## Alterado {#changed}

- **Versão estável do Next.js**: O Next.js agora está fixado na versão estável `16.2.9` (anteriormente um build canary) para maior confiabilidade.

---

## Notas de migração {#migration-notes}

### A partir da versão 1.4.1 {#from-version-141}

Ao atualizar para a versão 1.4.2:

1. **Nenhuma migração de banco de dados necessária**: Este lançamento não requer nenhuma alteração no esquema do banco de dados.
2. **Usuários de Docker**: Baixe a imagem mais recente; a correção do travamento na inicialização está incluída no build atualizado do container.
3. **Dependências**: Execute `pnpm install` para aplicar o lockfile e as substituições atualizadas.

## Changelog {#changelog}

### Alterações da Versão 1.4.2 {#version-142-changes}

- **Corrigido**: Travamento na inicialização do container Docker autônomo causado por um arquivo auxiliar ESM `@swc/helpers` ausente.
- **Corrigido**: Linhas duplicadas no dashboard quando vários registros compartilhavam a mesma data de backup mais recente.
- **Corrigido**: A mesclagem de servidores agora consolida IDs de backup do Duplicati duplicados que compartilham o mesmo nome de backup.
- **Alterado**: O usuário autenticado é buscado uma única vez via um provedor compartilhado, reduzindo requisições `/api/auth/me` redundantes no carregamento.
- **Alterado**: Operações de mesclagem de servidores agora são registradas no log de auditoria como `servers_merged`.
- **Alterado**: Rótulos de Mesclar Servidores Duplicados são renderizados como células de cabeçalho de tabela em vez de formatos de pílula/botão.
- **Alterado**: Next.js fixado na versão estável `16.2.9`.
