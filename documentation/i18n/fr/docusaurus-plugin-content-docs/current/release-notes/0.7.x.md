# Version 0.7.x {#version-07x}

## Vue d'ensemble {#overview}

La Version 0.7.x est une version majeure qui introduit des améliorations significatives au schéma de base de données, à la structure de l'API et à l'interface utilisateur. Cette version inclut des modifications majeures qui nécessitent des mises à jour des Intégrations externes.

## Modifications incompatibles {#breaking-changes}

### Modifications du schéma de base de données {#database-schema-changes}
- **Renommage de table** : la table `machines` a été renommée en `servers`
- **Nouveaux champs** : ajout des champs `alias`, `notes`, `created_at` et `updated_at`
- **Modifications de réponse API** : noms de champs mis à jour pour la cohérence

### Modifications incompatibles de l'API {#api-breaking-changes}
- **Renommage de champ** : `totalMachines` → `totalServers`
- **Renommage d'objet** : `machine` → `server` dans les réponses de l'API
- **Renommage de champ** : `backup_types_count` → `backupJobsCount`

:::warning
**Les intégrations externes utilisant l'API doivent être mises à jour** pour fonctionner avec la version 0.7.x. Consultez le guide [Modifications majeures de l'API](../migration/api-changes) pour plus de détails.
:::

## Guide de Migration {#migration-guide}

### Migration Automatique {#automatic-migration}
La migration de 0.6.x vers 0.7.x s'effectue automatiquement :

1. **Création de sauvegarde** : Une sauvegarde de la base de données automatique est créée
2. **Mise à jour du schéma** : Le schéma de la base de données est mis à jour automatiquement
3. **Migration des données** : Les données existantes sont préservées et migrées
4. **Vérification** : La migration est vérifiée et enregistrée

### Étapes manuelles requises {#manual-steps-required}
- **Mettre à jour les Intégrations externes** : Mettez à jour tous les outils externes utilisant l'API
- **Tester la fonctionnalité** : Vérifiez que toutes les fonctionnalités fonctionnent correctement
- **Mettre à jour la documentation** : Mettez à jour toute documentation personnalisée

## Modifications de configuration {#configuration-changes}

### Nouvelles Options de Configuration {#new-configuration-options}

```yaml

  defaultAlias: true
  showNotes: true
  sortBy: "name" # name, status, lastBackup

# API settings
api:
  version: "1.0"
  rateLimit: 100
  timeout: 30000

# Database settings
database:
  backupRetention: 7
  autoVacuum: true
  optimizeInterval: 24
```

### Options dépréciées {#deprecated-options}
- Configuration `machines` (utilisez `servers` à la place)
- Anciens noms de champs API (utilisez les nouveaux noms de champs)

## Instructions de mise à niveau {#upgrade-instructions}

### À partir de la Version 0.6.x {#from-version-06x}
1. **Sauvegardez vos données** (sauvegarde créée automatiquement lors de la mise à niveau)
2. **Mettez à jour l'image Docker** : `docker pull wsjbr/duplistatus:0.7.x`
3. **Redémarrez le conteneur** : `docker-compose down && docker-compose up -d`
4. **Surveillez les journaux** : `docker logs -f duplistatus`
5. **Mettez à jour les intégrations** : Mettez à jour tous les outils externes utilisant l'API
6. **Vérifiez la mise à niveau** : Vérifiez que toutes les fonctionnalités fonctionnent correctement

### À partir de la Version 0.5.x ou antérieure {#from-version-05x-or-earlier}
1. **Commencez par mettre à jour vers la version 0.6.x** (si disponible)
2. **Puis mettez à jour vers la version 0.7.x** en suivant les étapes ci-dessus
