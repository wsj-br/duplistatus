# Version 0.7.x

## vue d'ensemble

Version 0.7.x est une version majeure qui introduit des améliorations significatives au schéma de la base de données, à la structure de l'API et à l'interface Utilisateur. Cette Version inclut des modifications majeures qui nécessitent des mises à jour des Intégrations externes.

## Modifications majeures

### Modifications du schéma de la base de données

- **Renommage de table** : table `machines` renommée en `servers`
- **Nouveaux champs** : Ajout des champs `alias`, `notes`, `created_at` et `updated_at`
- **Modifications des réponses API** : Noms de champs mis à jour pour la cohérence

### Modifications majeures de l'API

- **Renommage de champ** : `totalMachines` → `totalServers`
- **Renommage d'objet** : `machine` → `server` dans les réponses API
- **Renommage de champ** : `backup_types_count` → `backupJobsCount`

:::warning
**Les Intégrations externes utilisant l'API doivent être mises à jour** pour fonctionner avec la Version 0.7.x. Consultez le guide [Modifications majeures de l'API](../migration/api-changes) pour plus de Détails.
:::

## Guide de migration

### Migration automatique

La migration de 0.6.x vers 0.7.x se fait automatiquement :

1. **Création de sauvegarde** : Sauvegarde de base de données automatique Créée
2. **Mise à jour du schéma** : Le schéma de la base de données est mis à jour automatiquement
3. **Migration des données** : Les données existantes sont préservées et migrées
4. **Vérification** : La migration est vérifiée et enregistrée

### Étapes manuelles requises

- **Mettre à jour les Intégrations externes** : Mettre à jour tous les outils externes utilisant l'API
- **Tester les fonctionnalités** : Vérifier que Tous les fonctionnalités fonctionnent correctement
- **Mettre à jour la documentation** : Mettre à jour toute documentation Personnalisée

## Modifications de configuration

### Nouvelles options de configuration

```yaml
# Gestion des serveurs
servers:
  defaultAlias: true
  showNotes: true
  sortBy: "name" # name, status, lastBackup

# Paramètres API
api:
  version: "1.0"
  rateLimit: 100
  timeout: 30000

# Paramètres de base de données
database:
  backupRetention: 7
  autoVacuum: true
  optimizeInterval: 24
```

### Options obsolètes

- `machines` configuration (utilisez `servers` à la place)
- Anciens noms de champs API (utilisez les nouveaux noms de champs)

## Installation

### Docker Compose

```yaml
services:
  duplistatus:
    image: wsjbr/duplistatus:0.7.x
    container_name: duplistatus
    restart: unless-stopped
    ports:
      - "9666:9666"
    volumes:
      - duplistatus_data:/app/data
    environment:
      - TZ=UTC
```

### Docker CLI

```bash
docker run -d \
  --name duplistatus \
  --restart unless-stopped \
  -p 9666:9666 \
  -v duplistatus_data:/app/data \
  wsjbr/duplistatus:0.7.x
```

## Instructions de mise à niveau

### À partir de la Version 0.6.x

1. **Sauvegardez vos données** (sauvegarde automatique Créée lors de la mise à niveau)
2. **Mettre à jour l'image Docker** : `docker pull wsjbr/duplistatus:0.7.x`
3. **Redémarrer le conteneur** : `docker-compose down && docker-compose up -d`
4. **Surveiller les journaux** : `docker logs -f duplistatus`
5. **Mettre à jour les Intégrations** : Mettre à jour tous les outils externes utilisant l'API
6. **Vérifier la mise à niveau** : Vérifier que Tous les fonctionnalités fonctionnent correctement

### À partir de la Version 0.5.x ou antérieure

1. **Mettez d'abord à niveau vers 0.6.x** (si disponible)
2. **Ensuite, mettez à niveau vers 0.7.x** en utilisant les étapes ci-dessus

## Problèmes connus

### Problèmes de migration

- **Grandes bases de données** : La migration peut prendre plus de temps pour les grandes bases de données
- **Espace disque** : Assurez-vous d'avoir suffisamment d'espace disque pour la sauvegarde de migration
- **Autorisations** : Vérifier les autorisations de fichiers pour l'accès à la base de données

### Compatibilité API

- **Outils externes** : Les outils externes utilisant l'API ont besoin de mises à jour
- **Noms de champs** : Certains noms de champs ont changé
- **Format de réponse** : Certains formats de réponse ont changé

### Solutions de contournement

- **Migration** : Assurez-vous d'avoir suffisamment d'espace disque et de temps pour la migration
- **API** : Mettre à jour les Intégrations externes pour utiliser le nouveau format API
- **Test** : Tout tester après la mise à niveau
