# Version 0.9.x {#version-09x}

## Vue d'ensemble {#overview}

Version 0.9.x est une version majeure qui introduit un **Système complet de Contrôle d'Accès Utilisateur et de Sécurité**. Cette version transforme **duplistatus** d'une application en accès libre en un système sécurisé multi-utilisateurs avec contrôle d'accès basé sur les rôles, journalisation d'audit complète et fonctionnalités de sécurité de niveau entreprise. Toutes les fonctionnalités existantes sont maintenant protégées par authentification, garantissant que seuls les utilisateurs autorisés peuvent accéder et gérer la surveillance des sauvegardes.

## Modifications Incompatibles {#breaking-changes}

### Authentification requise {#authentication-required}

- **Toutes les pages nécessitent désormais une authentification** - Les utilisateurs doivent se connecter pour accéder à l'application
- **Compte administrateur par défaut** créé automatiquement :
  - Nom d'utilisateur : `admin`
  - Mot de passe : `Duplistatus09` (doit être modifié lors de la première connexion)
- **Les sessions existantes sont invalidées** - Tous les utilisateurs doivent se reconnecter après la mise à jour
- **Les points de terminaison API nécessitent désormais une authentification** - Les intégrations externes doivent inclure les cookies de session ou utiliser des requêtes authentifiées

:::warning
**Tous les utilisateurs doivent se connecter après la mise à niveau vers la version 0.9.x.** Les identifiants admin par défaut sont admin/Duplistatus09. Changez le mot de passe immédiatement après la première connexion.
:::

### Migration du Schéma de Base de Données {#database-schema-migration}

- **Migration automatique de v3.1 vers v4.0** pour les installations existantes
- **Nouvelles tables ajoutées** : `users`, `sessions`, `audit_log`
- **Sauvegarde de la base de données créée automatiquement** avant la migration
- **Les nouvelles installations** démarrent directement avec le schéma v4.0 (aucune migration nécessaire)

## Nouvelles fonctionnalités {#new-features}

### Contrôle d'accès Utilisateur & Système de Sécurité {#user-access-control--security-system}

#### Système d'authentification {#authentication-system}

- **Fonctionnalité de connexion/déconnexion utilisateur** avec gestion sécurisée des sessions
- **Authentification par mot de passe** avec hachage bcrypt (facteur de coût 12)
- **Mécanisme de verrouillage de compte** (5 tentatives échouées, verrouillage de 15 minutes)
- **Changement de mot de passe forcé** lors de la première connexion
- **Application de la politique de mot de passe** :
  - Minimum 8 caractères
  - Au moins une lettre majuscule
  - Au moins une lettre minuscule
  - Au moins un chiffre
  - Caractères spéciaux facultatifs
- **Fonctionnalité « Se souvenir de moi »** avec persistance du nom d'utilisateur dans localStorage
- **Authentification basée sur les sessions** avec sessions stockées en base de données (remplace le stockage en mémoire)
- **Protection CSRF** intégrée au système d'authentification
- **Cookies HTTP-only** pour la sécurité des sessions
- **Suivi de l'adresse IP et de l'agent utilisateur** pour la surveillance de sécurité

#### Gestion des utilisateurs (Admin uniquement) {#user-management-admin-only}

- **Création d'utilisateurs** avec génération automatique d'un mot de passe temporaire
- **Liste des utilisateurs** avec recherche, pagination et filtres
- **Modification des utilisateurs** (nom d'utilisateur, statut administrateur, obligation de changement de mot de passe)
- **Réinitialisation de mot de passe** avec affichage du mot de passe temporaire
- **Suppression d'utilisateurs** avec protections (empêche la suppression du dernier administrateur)
- **Contrôle d'accès basé sur les rôles** (rôles Administrateur/Utilisateur)
- **Indicateurs d'état** (Actif, Verrouillé, Doit changer le mot de passe)
- **Suivi et affichage de la dernière connexion**

#### Système de journalisation d'audit {#audit-logging-system}

- **Journal d'audit complet** pour toutes les modifications système et actions utilisateur
- **Visionneuse des journaux d'audit** avec filtres avancés :
  - Filtre par plage de dates
  - Filtre par utilisateur
  - Filtre par action
  - Filtre par catégorie
  - Filtre par statut
- **Fonction d'exportation** (formats CSV et JSON)
- **Statistiques et analyses des journaux d'audit**
- **Période de rétention configurable** (30 à 365 jours, par défaut : 90 jours)
- **Tâche planifiée de nettoyage automatique** (exécutée quotidiennement à 2h UTC)
- **API de nettoyage manuel** avec prise en charge du mode simulation (dry-run)
- **Journalisation d'audit pour** :
  - Événements d'authentification (connexion, déconnexion, changements de mot de passe, verrouillages de compte)
  - Opérations de gestion des utilisateurs (création, mise à jour, suppression, réinitialisations de mot de passe)
  - Modifications de configuration (e-mail, NTFY, modèles, tolérance pour les retards, paramètres de sauvegarde)
  - Opérations de sauvegarde (collecte, suppression, nettoyage)
  - Gestion des serveurs (ajout, mise à jour, suppression)
  - Opérations système (migrations, nettoyage, notifications)

### Refonte de la page Paramètres {#settings-page-redesign}

- **Navigation latérale moderne** avec barre latérale repliable
- **Persistance de l'état de la barre latérale par utilisateur** (préférence repliée/dépliée sauvegardée pour chaque utilisateur)
- **Sections de paramètres regroupées** :
  - **Notifications** : Notifications de sauvegarde, Surveillance des sauvegardes, Modèles
  - **Intégrations** : NTFY, E-mail
  - **Système** : Serveurs, Utilisateurs (administrateurs uniquement), Journal d'audit

- **Conception responsive** avec espacement optimisé
- **Icône des paramètres et titre "Paramètres système"** dans l'en-tête de la barre latérale
- **Bouton Retour** intégré à l'en-tête de l'application

### Améliorations de l'Interface Utilisateur {#user-interface-enhancements}

- **Page de connexion autonome** au design moderne
- **Case à cocher "Se souvenir de moi"** sur le formulaire de connexion pour la persistance du nom d'utilisateur
- **Boutons Afficher/Masquer le mot de passe** sur les formulaires de connexion et de changement de mot de passe
- **Modale de changement de mot de passe** avec liste de contrôle de validation en temps réel
- **Indicateur utilisateur et bouton de déconnexion** dans l'en-tête de l'application
- **Visibilité de l'interface selon le rôle** (fonctionnalités administrateur masquées aux utilisateurs standard)

### Fonctionnalités de Sécurité {#security-features}

- **Hachage des mots de passe** avec bcrypt (standard de l'industrie)
- **Assainissement des données sensibles** dans les journaux d'audit (les mots de passe, jetons et secrets ne sont jamais enregistrés)
- **Limitation de débit** des tentatives de connexion
- **Expiration de session** (24 heures)
- **Validation du jeton CSRF** pour toutes les opérations modifiant l'état
- **Outil CLI de récupération administrateur** pour réinitialiser le mot de passe en cas de verrouillage

- **Interfaces TypeScript complètes** (aucun type `any`)
- **Middleware d'authentification** pour la protection des routes
- **Classe utilitaire de journalisation d'audit** avec méthodes pratiques

## 🚀 Notes de migration {#-migration-notes}

### À partir de la Version 0.8.x {#from-version-08x}

Cette version introduit l'authentification et nécessite que tous les utilisateurs se connectent. Quand vous effectuez une mise à niveau à partir de la version 0.8.x :

1. **Migration automatique de la base de données** : L'application migrera automatiquement votre schéma de base de données de la v3.1 à la v4.0
2. **Sauvegarde de la base de données** : Une sauvegarde automatique est créée avant la migration
3. **Compte administrateur par défaut** : Un compte administrateur par défaut est créé :
   - Nom d'utilisateur : `admin`
   - Mot de passe : `Duplistatus09`
   - **Vous devez modifier ce mot de passe lors de la première connexion**
4. **Invalidation des sessions existantes** : Toutes les sessions existantes sont invalidées
5. **Connexion requise pour tous les utilisateurs** : Tous les utilisateurs doivent se connecter après la mise à jour
6. **Authentification API** : Les intégrations externes doivent être mises à jour pour inclure l'authentification

### Considérations de sécurité {#security-considerations}

- **Changer le mot de passe par défaut** : Changez immédiatement le mot de passe administrateur par défaut après la première connexion
- **Créer des comptes utilisateurs** : Créez un compte utilisateur individuel pour chaque personne nécessitant un accès
- **Vérifier les journaux d'audit** : Consultez régulièrement les journaux d'audit pour la surveillance de sécurité
- **Configurer la rétention** : Définissez une période de rétention des journaux d'audit adaptée à vos exigences de conformité
- **Sauvegarder la clé maîtresse** : Si vous mettez à jour depuis la version 0.8.x, assurez-vous de sauvegarder le fichier `.duplistatus.key`

### Premières étapes de connexion {#first-login-steps}

1. **Se connecter** avec les identifiants par défaut (`admin` / `Duplistatus09`)
2. **Changer le mot de passe** lorsque demandé (obligatoire à la première connexion)
3. **Créer des comptes utilisateurs** pour les autres utilisateurs (Paramètres → Utilisateurs)
4. **Configurer la rétention des journaux d'audit** si nécessaire (Paramètres → Journal d'audit)
5. **Vérifier les journaux d'audit** pour s'assurer que le système fonctionne correctement

## Points de terminaison API {#api-endpoints}

### Points de terminaison d'authentification {#authentication-endpoints}

- `POST /api/auth/login` - Authentification de l'utilisateur
- `POST /api/auth/logout` - Terminaison de session
- `GET /api/auth/me` - Informations de l'utilisateur actuel
- `POST /api/auth/change-password` - Modification du mot de passe

### Points de terminaison de gestion des utilisateurs (Admin uniquement) {#user-management-endpoints-admin-only}

- `GET /api/users` - Lister les utilisateurs
- `POST /api/users` - Créer l'utilisateur
- `PATCH /api/users/{id}` - Mettre à jour l'utilisateur
- `DELETE /api/users/{id}` - Supprimer l'utilisateur

### Points de Terminaison du Journal d'Audit {#audit-log-endpoints}

- `GET /api/audit-log` - Interroger les journaux d'audit
- `GET /api/audit-log/download` - Exporter les journaux d'audit
- `GET /api/audit-log/stats` - Statistiques d'audit
- `POST /api/audit-log/cleanup` - Nettoyage manuel (administrateur uniquement)
- `GET /api/audit-log/retention` - Obtenir le paramètre de rétention
- `PATCH /api/audit-log/retention` - Mettre à jour la rétention (administrateur uniquement)

:::note
Tous les points de terminaison API nécessitent maintenant une authentification. Consultez la [Référence API](../api-reference/authentication-security.md) pour plus de détails sur l'authentification.
:::
