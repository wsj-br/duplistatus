# Version 1.4.2 {#version-142}

## Vue d'ensemble {#overview}

La version 1.4.2 est une version de maintenance axée sur la stabilité et la fiabilité. Elle corrige un plantage du conteneur Docker au démarrage, réduit les requêtes d'authentification redondantes lors du chargement du tableau de bord, améliore l'outil « Fusionner les serveurs en double » pour qu'il consolide également les ID de sauvegarde Duplicati en double, et supprime les lignes redondantes du tableau de bord. Elle enregistre également les fusions de serveurs dans le journal d'audit et fige Next.js sur une version stable.

---

## Corrections de bogues {#bug-fixes}

- **Plantage au démarrage de Docker** : Le conteneur pouvait planter immédiatement au démarrage avec une erreur `Cannot find module … @swc/helpers/esm/_interop_require_default.js`. Le build autonome inclut désormais correctement les fichiers d'aide requis, permettant au conteneur de démarrer de manière fiable.
- **Entrées en double sur le tableau de bord** : Le tableau de bord n'affiche plus plusieurs lignes pour le même nom de sauvegarde lorsque plusieurs enregistrements partagent la même date de sauvegarde la plus récente.
- **Fusion de serveurs en double** : La fusion de serveurs consolide désormais également les ID de sauvegarde Duplicati en double qui partagent le même nom de sauvegarde sur le serveur fusionné, en conservant l'ID de la sauvegarde la plus récente afin de maintenir la cohérence de l'historique.

---

## Améliorations {#improvements}

### Performance {#performance}

- **Moins de requêtes d'authentification au chargement** : L'utilisateur connecté est désormais récupéré une seule fois et partagé sur toute la page (en-tête, protection du mot de passe, fournisseurs de thème/configuration et widgets du tableau de bord) au lieu que chaque composant ne le demande séparément, réduisant ainsi les appels réseau redondants lors de l'ouverture du tableau de bord.

### Audit {#auditing}

- **Fusions de serveurs enregistrées dans le journal d'audit** : La fusion de serveurs est désormais enregistrée comme une entrée `servers_merged` dans le journal d'audit pour une meilleure traçabilité.

### Interface {#interface}

- **Mise en page de Fusionner les serveurs en double** : Dans Paramètres > Maintenance de la base de données > Fusionner les serveurs en double, les libellés « Serveur cible (le plus récent) » et « Ancien ID de serveur » s'affichent désormais comme des cellules d'en-tête de tableau (compatibles avec le thème) au lieu de formes de pilules/boutons, pour un aspect plus clair et plus cohérent.

---

## Changements {#changed}

- **Version stable de Next.js** : Next.js est désormais figé sur la version stable `16.2.9` (auparavant un build canary) pour une fiabilité accrue.

---

## Notes de migration {#migration-notes}

### Depuis la version 1.4.1 {#from-version-141}

Lors de la mise à jour vers la version 1.4.2 :

1. **Aucune migration de base de données requise** : Cette version ne nécessite aucun changement de schéma de base de données.
2. **Utilisateurs Docker** : Téléchargez la dernière image ; le correctif du plantage au démarrage est inclus dans le build du conteneur mis à jour.
3. **Dépendances** : Exécutez `pnpm install` pour appliquer le fichier de verrouillage et les substitutions mis à jour.

## Journal des modifications {#changelog}

### Changements de la version 1.4.2 {#version-142-changes}

- **Corrigé** : Plantage au démarrage du conteneur Docker autonome causé par un fichier d'aide ESM `@swc/helpers` manquant.
- **Corrigé** : Lignes en double dans le tableau de bord lorsque plusieurs enregistrements partageaient la même date de sauvegarde la plus récente.
- **Corrigé** : La fusion de serveurs consolide désormais les ID de sauvegarde Duplicati en double partageant le même nom de sauvegarde.
- **Changé** : L'utilisateur connecté est récupéré une seule fois via un fournisseur partagé, réduisant les requêtes `/api/auth/me` redondantes au chargement.
- **Changé** : Les opérations de fusion de serveurs sont désormais enregistrées dans le journal d'audit sous la forme `servers_merged`.
- **Changé** : Les libellés de Fusionner les serveurs en double s'affichent comme des cellules d'en-tête de tableau au lieu de formes de pilules/boutons.
- **Changé** : Next.js figé sur la version stable `16.2.9`.
