# Sanskaran 1.2.1 {#version-121}

## Overview {#overview}

Sanskaran 1.2.1 ek chhoti release hai jo backup suchna prabandhan mein mahatvapurna sudhaar laati hai, audit logging kshamataon ko behtar banati hai, edge cases par agle chalan/vilambit ka pata lagane mein sudhaar karti hai, aur database backup aur restore functionality jodti hai. Yah release suchna samman ke saath upyogkarta anubhav ko behtar banane par kendrit hai.

## New Features {#new-features}

### Behtar Backup Suchna Prabandhan {#enhanced-backup-notifications-management}

- **Server-level default aur inheritance system**: Server-level par aadhit gantavya (suchna ghatnaayein, email, aur NTFY topics) configure karein jinhein sabhi backups svachaalit roop se inherit karte hain. Jab zaroorat ho to backups in defaults ko override kar sakte hain, inheritance sthiti ko darshane wale spasht drishya sanketon ke saath.
- **Behtar drishya hierarchy**: Backups ab server dwara group kiye jaate hain jismein spasht server header rows hote hain. Inherited maan link icons ke saath spasht roop se chihnit hote hain, aur overridden maanon ko aasani se inheritance par revert kiya ja sakta hai.
- **Bulk operations**: "Sabhi ke saath sankalit karein" button sabhi backup overrides ko saaf karta hai taaki sabhi backups server defaults se inherit kar sakein. "Sabhi saaf karein" button sabhi aadhitheeya gantavya ko saaf karta hai jabki inheritance structure ko banaye rakhta hai.
- **Parikshan functionality**: Backup Suchna Sammaan page mein seedhe test email aur test suchna buttons jode gaye hain. Test buttons vyaktigat backup samman aur bulk sampadan operations donon ke liye uplabdh hain, jisse aap parivartanon ko laagoo karne se pehle suchna pahunch ki pushti kar sakte hain.
- **QR code support**: Mobile devices ke saath suchna topics ko aasani se share karne ke liye NTFY topics ke liye QR code button joda gaya hai.

### Suchna Template Sudhaar {#notification-template-enhancements}

- **Naya `{log_text}` variable**: Suchna templates mein `{log_text}` variable joda gaya hai, jo suchna sandeshon mein istemaal ke liye log text sandeshon tak pahunch pradaan karta hai. Variable mein chetaavaniyon aur trutiyon ka poora text shaamil hai, jo plain text ke roop mein prati line ek item ke saath format kiya gaya hai.
- **Svachaalit template migration**: System mein ek lazy upgrade mechanism shaamil hai jo upgrade ke baad pehli baar padhe jaane par purane default templates ka svachaalit roop se pata lagata hai aur unhein update karta hai. Anukoolit templates ko bina badle rakha jaata hai.

### Database Prabandhan {#database-management}

- **Database backup functionality**: Administrator ab Database Maintenance samman page se poore database ka backup bana sakte hain. Lacheelepan ke liye binary (.db) aur SQL text (.sql) donon formats ka samarthan karta hai.
- **Database restore functionality**: Svachaalit suraksha backup banane, integrity validation, aur failure par rollback ke saath pehle banaye gaye backup files se database ko restore karein. Suraksha ke liye restore ke baad sabhi sessions saaf kar diye jaate hain.

### Master Key Detection {#master-key-detection}

- **Svachaalit master key parivartan detection**: Login ke baad, system svachaalit roop se jaanchta hai ki `.duplistatus.key` file mein parivartan hua hai ya nahin. Yadi pata chalta hai, to sabhi encrypted passwords saaf kar diye jaate hain aur ek modal popup upyogkartaon ko suchit karta hai ki passwords ko phir se configure karne ki zaroorat hai. Yah backups se restore karte samay ya naye system mein migrate karte samay trutiyon ko rokta hai.

### Backup monitoring Sudhaar {#backup-monitoring-improvements}

- **Behtar CSV export**: Backup monitoring samman mein "CSV download karein" button dwara generate ki gayi CSV file mein "Antim Backup Timestamp (DB)" column joda gaya hai. Yah upyogkartaon ko database se vastavik antim backup timestamp ki tulna configure ki gayi "Antim Backup" timestamp se karne ki anumati deta hai.
- **Aage ke chalan tooltip**: Prati row mein "Aage ke chalan" maan par hover karne par ab database se antim backup timestamp dikhane wala ek tooltip pradarshit hota hai, jo "Aage ke chalan" display ke saman roop mein format kiya gaya hai.

### Audit Logging Sudhaar {#audit-logging-enhancements}

- **Vyaktigat suchana audit pravishtiyan**: Bheji gayi har suchana aur email ab channel, configuration, aur parinaam (safalta ya asafalta) ke vivaran ke saath ek alag audit log pravishti banati hai. Ismein parikshan suchnayein shamil hain, jo suchana pahunchane ke liye behtar traceability pradan karti hain.
- **Majboot asafalta logging**: Suchana asafaltaon ke liye sabhi audit logging operations ab try-catch blocks mein lapete gaye hain taki audit logging trutiyon ko suchana pravah ko todne se roka ja sake. Pranali SMTP configuration prapt karne ka prayas karti hai yadi email asafaltaon ko log karte samay vah turant uplabdh na ho.
- **Configuration updates ke liye behtar audit logging**: `email_config_updated` aur `ntfy_config_updated` dono audit log pravishtiyan ab kewal tabhi record karti hain jab configuration mein vastavik parivartan hote hain. Jab parivartan pata chalte hain, to audit log mein pratyek sanshodhit kshetra ke liye purane aur naye moolya shamil hote hain, jisse yah track karna aasan ho jata hai ki vishesh roop se kya badla hai.

### Dastavejikaran sudhaar {#documentation-improvements}

- Dastavejikaran shaili ko application ke look aur feel se mel khane ke liye update kiya gaya hai. Sidebar ko bhi drishyata sudharne aur navigation ko aasan banane ke liye sanshodhit kiya gaya hai.

## 🐛 Bug Fixes {#-bug-fixes}

- **Audit log timestamps theek kiye gaye**: Audit log viewer mein timestamps ab GMT ke bajaye upyogkarta ke browser timezone mein sahi dhang se pradarshit hote hain. SQLite timestamps (UTC mein store kiye gaye) ab sahi dhang se UTC ke roop mein parse kiye jate hain aur pradarshan ke liye browser ke sthaniya timezone mein parivartit kiye jate hain.
- **"Aage ke chalan" calculation aur vilambit detection ko durust kiya gaya**: Backup monitoring settings mein "Aage ke chalan" column ab backup schedule aur antim backup timestamp dono ko dhyan mein rakhte hue, agle apekshit backup tareekh ki sahi dhang se calculation karta hai aur pradarshit karta hai. Yah scheduled aur manually run kiye gaye backup jobs dono par lagu hota hai, jiske parinaam swaroop adhik vishwasniya vilambit detection hota hai.

## 🚀 Migration Notes {#-migration-notes}

### Sanskaran 1.1.x se {#from-version-11x}

Sanskaran 1.2.1 mein upgrade karte samay:

1. **Suchana Templates**: Pranali pahli baar access karne par default notification templates ko naye `{log_text}` variable ko shamil karne ke liye swachalit roop se update karegi. Anukoolit templates aprivartit rahengi. Yadi aap apne templates mein naye `{log_text}` variable ka upyog karna chahte hain, to aapko unhein update karna hoga ya default templates ko manually restore karna hoga.

2. **Master Key Parivartan**: Yadi aap backup se restore karte hain ya alag `.duplistatus.key` file wale naye system mein migrate karte hain, to pranali ise swachalit roop se detect karegi aur encrypted passwords (SMTP passwords aur Duplicati server passwords) ko configure karne ke liye prompt karegi.

3. **Database Backup/Restore**: Database Maintenance settings page mein naye database backup aur restore functionality uplabdh hai. Upgrade karne se pehle backup banana anushansit hai, halanki is release ke liye kisi database schema parivartan ki avashyakta nahi hai.

4. **Database Migration Anivarya Nahi**: Is release ke liye kisi database schema parivartan ki avashyakta nahi hai.

## API Endpoints {#api-endpoints}

Is sanskaran mein nimnalikhit API endpoints prastut kiye gaye hain.

### Authentication & Security Endpoints {#authentication--security-endpoints}

- `GET /api/auth/admin-must-change-password` - Jaanch karein ki admin upyogkarta ko password badalna hai ya nahi

### Database Management Endpoints {#database-management-endpoints}

- `GET /api/database/backup` - Database backup banayein (binary ya SQL format)
- `POST /api/database/restore` - Backup file se database restore karein

### Backup Management Endpoints {#backup-management-endpoints}

- `GET /api/backups/last-timestamps` - Sabhi server-backup combination ke liye antim backup timestamps prapt karein
