# Sanskaran 1.3.0 {#version-130}

## Overview {#overview}

Sanskaran 1.3.0 mein configure karna yogy password nitiyaan, ek naya application log viewer prabandhakon ke liye, sthaniya anusaar saptaah ke din ka pradarshan, aur kai upyogkarta anubhav sudhaar shamil hain. Is release mein vishisht prabandhan upkarnon aur sudharit antarrashtriyaikaran samarthan par dhyan kendrit kiya gaya hai.

## New Features {#new-features}

### Configure Karna Yogy Password Niti {#configurable-password-policy}

- **Password niti paryavaran vikalp**: Paryavaran vikalpon ke madhyam se password validation aavashyaktaon ko configure karein:
  - `PWD_ENFORCE`: Nishkriya karne ke liye `false` par set karein password jatilta aavashyaktaon (uppercase, lowercase, sankhyayein). Default roop se sabhi niyamon ka bal hai. Jab nishkriya hota hai, keval nyunatam lambai ka bal hota hai.
  - `PWD_MIN_LEN`: Password ki nyunatam lambai character mein nirdharit karta hai. Default roop se 8 characters hota hai.
  - Default password janch (prabandhak password ka upyog rokne wala) hamesha `PWD_ENFORCE` setting ke bavjood bal hota hai.

### Application Log Viewer {#application-logs-viewer}

- **Naya prabandhak-only log viewer**: Seedhe Settings page se application logs ka avlokan aur download karein:
  - Samarthit `application.log` file aur ghoomti hui log sanskaranon ka avlokan
  - Rekhank ka chayan karne wala (100, 500, 1000, 5000, 10000 rekhayein)
  - Vaastavik samay mein khoj filter ke sath case-insensitive text matching
  - Auto-scroll toggle vaastavik samay mein log monitoring ke liye svaichalit polling ke sath
  - Clipboard mein copy karein aur filtered logs ko text file ke roop mein export karein
  - File metadata pradarshan jo rekhank gina, file aakar, aur antim parivartit samay chinh dikhata hai
  - Container startup par log rotation (`LOG_ROTATION_VERSIONS` ke madhyam se configure kiya ja sakta hai, default: 5)

### Sandarbh-Aware Upyogkarta Margdarshika {#context-aware-user-guide}

- **Dynamic sahayata button**: App header mein Upyogkarta Margdarshika button ab vartaman page se sambandhit dastavez khulta hai. Tooltip update hota hai "`Page Name` ke liye sahayata" dikhane ke liye ki kaun sa dastavez khulega.

### Server Sammanvaya Sudhaar {#server-configuration-enhancement}

- **Duplicati UI tak jaldi pahunch**: Server sammanvaya button ya server chayan popover mein kisi bhi server item par right-click karne se Duplicati legacy UI (`/ngax`) naye tab mein khulta hai.

## 🔧 Sudhaar {#-improvements}

### Sthaniya Anusaar Saptaah Ke Din Ka Pradarshan {#locale-aware-weekday-display}

- **Antarrashtriya saptaah ke din ka kram**: Backup monitoring form mein saptaah ke din ka pradarshan ab aapke browser ke sthaniya setting ko sammanit karta hai. Saptaah ke din sahi kram mein pradarshit hote hain (Somvaar-Ravivaar ke liye UK/Yurope, Ravivaar-Shanivaar ke liye US/Brazil).

### Sudharit SMTP Truti Handaling {#enhanced-smtp-error-handling}

- **Sudharit truti sandesh**: SMTP truti sandesh ab adhik satik margdarshan pradan karte hain:
  - Pramanikaran truti spasht roop se username/password samasyaon ko darshate hain
  - SSL/TLS sanskaran truti sahi roop se pahchani jaati hain
  - Connection type sujhav keval tabhi pradarshit hote hain jab ve vaastavik roop se prasangik hote hain

### Toast Suchna Pravritti {#toast-notification-behaviour}

- **Sthayi truti suchnaayein**: Truti toasts ab manav roop se band kiye jaane tak sthayi rehte hain, isse aapko mahatvapoorn truti sandesh na chhute. Safalta aur suchna toasts 3 second ke baad svaichalit roop se avagat hote hain.

### Suchna Sammanvaya Table {#notification-settings-table}

- **Saralit layout**: Suchna sammanvaya table ko ek sahit "Server / Backup" column aur sandarbhik chinhon ke sath sudharit kiya gaya hai jo additional destination sammanvaya sthiti ko darshate hain.

## 🚀 Migration Notes {#-migration-notes}

### Version 1.2.x se {#from-version-12x}

Jab upgrade version 1.3.0 mein karein:

1. **Password Niti**: Nayi password niti sabhi jatilata avashyaktaon ko lagu karne ke liye default hai. Yadi aapko ek saral password niti ki avashyakta hai, to `PWD_ENFORCE` aur `PWD_MIN_LEN` environment variables ko configure karein.

2. **Application Logs**: Sabhi container output (server aur cron service) ab ek `data/logs/application.log` file mein sammilit hain.

3. **Log Rotation**: Log rotation `LOG_ROTATION_VERSIONS` environment variable dwara configure kiya ja sakta hai (default: 5 versions).

4. **Database Migration Anivarya Nahi**: Is release ke liye kisi database schema parivartan ki avashyakta nahi hai.

## API Endpoints {#api-endpoints}

Is version mein nimnalikhit API endpoint shuru kiya gaya hai:

### Password Niti Endpoint {#password-policy-endpoint}

- `GET /api/auth/password-policy` - Vartaman password niti configuration vapas karta hai

### Application Logs Endpoints {#application-logs-endpoints}

- `GET /api/application-logs` - File selection aur tail functionality ke saath logs dekhein
- `GET /api/application-logs/export` - Filtered logs ko text file ke roop mein export karein

### Hataye gaye Endpoints {#removed-endpoints}

Nimnalikhit anupyogi API endpoints is version mein hataye gaye hain:

- `DELETE /api/backups/[backupId]` - Development-only endpoint jiska kabhi upyog nahi kiya gaya
- `GET /api/audit-log/stats` - Audit log viewer dwara endpoint ko call nahi kiya gaya tha
