# Sanskaran 1.0.x {#version-10x}

## Overview {#overview}

Sanskaran 1.0.x ek feature release hai jo email configuration, server prabandhan kshamataon, aur database maintenance tools mein mahatvapurn sudhaar laata hai. Yah release email suchna vishvasniyata mein sudhaar karne, SMTP connection vikalpon ko vistarit karne, aur serveron aur database integrity ko prabandhit karne ke liye behtar tools pradaan karne par kendrit hai.

## New Features {#new-features}

### Server Prabandhan Badhotari {#server-management-enhancements}

- **Server password badalne ki suvidha** - Upyogkarta ab server detail prushth se seedhe server password badal sakte hain, key icon par click karke
- **Duplicate server ka pata lagana aur merge karna** - Database maintenance form mein duplicate serveron ka pata lagane aur unhein ek server entry mein merge karne ke liye naya function

### Audit Logging Sudhaar {#audit-logging-enhancements}

- **Backup upload audit logging** - `/api/upload` endpoint ke maadhyam se sabhi backup log upload ab audit log mein svachaalit roop se logged hote hain. Har entry mein backup vivaran (server ID, server naam, backup naam, stithi, taareekh, avadhi, file sankhya, aakaar, chetaavaniyaan, aur trutiyan) client IP pata aur upyogkarta agent jaankari ke saath tracking aur suraksha ke uddeshya se shaamil hain

### Email Configuration Sudhaar {#email-configuration-enhancements}

- **Configuration adhoori suchna** - Jab email settings theek se configure nahin ki jaati hain to dikhai dene wala alert
- **Service stithi indicator** - Jab email sevaayein configure nahin ki jaati hain to notification checkbox par "(nishkriya)" labels dikhaaye jaate hain
- **Customizable bhejane wale ki jaankari**:
  - `Sender Name` field custom karne ke liye display naam jo email suchnaon mein bhejane wale ke roop mein dikhta hai
  - `From Address` field email address specify karne ke liye jo bhejane wale ke roop mein dikhta hai
  - Note: Kuch email provider `From Address` ko `SMTP Server Username` se override kar denge
- **SMTP authentication toggle** - SMTP authentication ko enable ya disable karne ka vikalp, aise serveron ka samarthan karta hai jinhein authentication ki zaroorat nahin hai
- **Email format validation** - Praaptak aur se address fields ke liye validation taaki email address ka sahi format sunishchit ho sake
- **Sudhaara hua test email** - Test email ab sampoorn connection vivaran dikhaata hai jismein SMTP server hostname, port, connection type, authentication stithi, username (yadi laagoo ho), praaptak email, se address, bhejane wale ka naam, aur test samay chinh shaamil hain

### SMTP Connection Type Support {#smtp-connection-type-support}

- **Bahut saare connection types** - Teen connection types ka samarthan:
  - `Plain SMTP` (koi encryption nahin) port 25 par
  - `STARTTLS` (TLS mein upgrade) port 587 par
  - `Direct SSL/TLS` (shuruaat se encrypted) port 465 par
- **Default connection type** - Nayi configurations ke liye `Direct SSL/TLS` mein badla gaya
- **Connection-vishesh aavashyaktaayein** - `From Address` field `Plain SMTP` connections ke liye zaroori hai taaki email bhejane wale ki pehchaan aur RFC 5322 compliance sunishchit ho sake

## 🐛 Bug Fixes {#-bug-fixes}

### Email Pranali Sudhaar {#email-system-fixes}

- **Ghayab `From` header theek kiya gaya** - RFC 5322 compliance trutiyon ko theek kiya gaya hai, yeh sunishchit karte hue ki sabhi email sandeshon mein `From` headers shaamil hon
- **STARTTLS ke liye email encryption theek ki gayi** - `STARTTLS` connections (port 587) ke liye encryption handling ko theek kiya gaya
- **Plain SMTP connections theek kiye gaye** - TLS negotiation trutiyon ko theek kiya gaya tha jiske kaaran `Plain SMTP` connections fail ho rahe the
- **Save na ki gayi configuration ka nuksaan theek kiya gaya** - Password badalte samay save na ki gayi email configuration ko clear hone se roka gaya
- **Stale SMTP configuration theek ki gayi** - Aisi samasya ko theek kiya gaya jahan test email endpoint purani SMTP configuration ka upyog kar raha tha

### User Interface Sudhaar {#user-interface-fixes}

- **Password visibility toggle theek kiya gaya** - Password badalne wale form mein view password feature ke vyavhaar ko theek kiya gaya; eye icon par click karne se ab dono password fields ke liye password ek saath dikhta hai

### Pranali Sudhaar {#system-fixes}

- **Docker entrypoint logging theek kiya gaya** - Aisi samasya ko theek kiya gaya jahan Docker entrypoint script log sandeshon ko chhod raha tha
- **Next Run taareekh calculation theek ki gayi** - Backup logs collect karne ke baad `Next Run` taareekh ki svachaalit punarganana jodi gayi hai, yeh sunishchit karte hue ki log collection ke baad taareekh theek se update ho

## 🚀 Migration Notes {#-migration-notes}

### Sanskaran 0.9.x se {#from-version-09x}

Yah release mein email configuration aur server management ke liye sudhaar shamil hain. Sanskaran 0.9.x se upgrade karte samay:

1. **Database Migration Ki Zaroorat Nahin** - Is release ke liye kisi bhi database schema badlav ki zaroorat nahin hai
2. **Email Configuration Updates** - Maujooda email configurations kaam karti rahengi, lekin aap nayi suvidhaon ka fayda uthane ke liye apni settings ki jaanch aur update karna chah sakte hain:
   - Behtar email pehchan ke liye ek anukoolit `Sender Name` set karne par vichar karein
   - Apne `From Address` ki jaanch karein ki vah sahi dhang se set hai, khaas taur par jab `Plain SMTP` connections ka istemal kar rahe hon
   - Behtar test email suvidha ka upyog karke apni email configuration ka parikshan karein
3. **Connection Type Default** - Nayi email configurations default roop se `Direct SSL/TLS` connection type par set hongi
4. **Server Password Management** - Ab aap server password ko seedhe server detail page se badal sakte hain bina backups ko dobara collect kiye

### Email Configuration Recommendations {#email-configuration-recommendations}

- **Production use ke liye**: Surakshit email delivery ke liye `STARTTLS` ya `Direct SSL/TLS` connections ka upyog karein
- **Local testing ke liye**: Bharosemand local networks ke liye `Plain SMTP` connections uplabdh hain
- **From Address ki avashyakta**: `Plain SMTP` ka upyog karte samay ya jab authentication disable ho, to sunishchit karein ki `From Address` field sahi dhang se configure kiya gaya hai
- **Apni configuration ka parikshan karein**: Apne email setup ke sahi kaam karne ki pushti ke liye hamesha "Send Test Email" button ka upyog karein

## API Endpoints {#api-endpoints}

### Email Configuration Endpoints {#email-configuration-endpoints}

- `GET /api/email/config` - Email configuration prapt karein
- `POST /api/email/config` - Email configuration update karein
- `POST /api/email/test` - Test email bhejein
- `DELETE /api/email/config` - Email configuration delete karein

### Server Management Endpoints {#server-management-endpoints}

- `PATCH /api/servers/{id}/password` - Server password update karein
- `POST /api/database/maintenance/merge-duplicate-servers` - Duplicate servers merge karein
