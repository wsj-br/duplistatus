# Versión 1.4.2 {#version-142}

## Resumen {#overview}

La Versión 1.4.2 es una versión de mantenimiento centrada en la estabilidad y la fiabilidad. Reduce las solicitudes de autenticación redundantes cuando se carga el panel de control, mejora la herramienta "Combinar servidores duplicados" para que también consolide los ID de copia de seguridad de Duplicati duplicados y elimina las filas duplicadas del panel de control. También registra las combinaciones de servidores en el Registro de auditoría y fija Next.js en una versión estable.

---

## Correcciones de errores {#bug-fixes}

- **Entradas duplicadas en el panel**: El panel ya no muestra varias filas para el mismo nombre de copia de seguridad cuando varios registros comparten la misma fecha de copia de seguridad más reciente.
- **Fusión de servidores duplicados**: La fusión de servidores ahora también consolida los ID de copia de seguridad duplicados de Duplicati que comparten el mismo nombre de copia de seguridad en el servidor fusionado, conservando el ID de la copia de seguridad más reciente para que el historial permanezca coherente.

---

## Mejoras {#improvements}

### Rendimiento {#performance}

- **Menos solicitudes de autenticación al cargar**: El Usuario autenticado ahora se obtiene una sola vez y se comparte en toda la Página (encabezado, protector de Contraseña, proveedores de Tema/Configuración y widgets del panel de control) en lugar de que cada componente lo solicite por separado, reduciendo las llamadas de red redundantes cuando se abre el panel de control.

### Auditoría {#auditing}

- **Combinaciones de servidores registradas en el Registro de auditoría**: La combinación de servidores ahora se registra como una entrada `servers_merged` en el Registro de auditoría para una mejor trazabilidad.

### Interfaz {#interface}

- **Diseño de Combinar servidores duplicados**: En Configuración > Mantenimiento de base de datos > Combinar servidores duplicados, las etiquetas "Servidor Destino (más reciente)" e "ID de servidor antiguo" ahora se renderizan como celdas de encabezado de tabla adecuadas (compatibles con el tema) en lugar de formas de píldora/botón, para una apariencia más clara y coherente.

---

## Cambios {#changed}

- **Versión estable de Next.js**: Next.js ahora está fijado a la versión estable `16.2.9` (anteriormente una compilación canary) para mejorar la fiabilidad.

---

## Notas de migración {#migration-notes}

### Desde la versión 1.4.1 {#from-version-141}

Al actualizar a la versión 1.4.2:

1. **No se requiere migración de base de datos**: Esta versión no requiere ningún cambio en el esquema de la base de datos.
2. **Usuarios de Docker**: Descargue la imagen más reciente; la corrección del error de cierre inesperado al iniciar se incluye en la compilación actualizada del contenedor.

## Registro de cambios {#changelog}

### Cambios de la Versión 1.4.2 {#version-142-changes}

- **Corregido**: Fallo de inicio del contenedor independiente de Docker causado por la falta de un archivo auxiliar ESM `@swc/helpers`.
- **Corregido**: Filas duplicadas en el panel de control cuando varios registros compartían la misma fecha de copia de seguridad más reciente.
- **Corregido**: La combinación de servidores ahora consolida los ID de copia de seguridad de Duplicati duplicados que comparten el mismo Nombre de copia de seguridad.
- **Cambiado**: El Usuario autenticado se obtiene una sola vez a través de un proveedor compartido, reduciendo las solicitudes `/api/auth/me` redundantes al cargar.
- **Cambiado**: Las operaciones de combinación de servidores ahora se registran en el Registro de auditoría como `servers_merged`.
- **Cambiado**: Las etiquetas de Combinar servidores duplicados se renderizan como celdas de encabezado de tabla en lugar de formas de píldora/botón.
- **Cambiado**: Next.js fijado a la versión estable `16.2.9`.
