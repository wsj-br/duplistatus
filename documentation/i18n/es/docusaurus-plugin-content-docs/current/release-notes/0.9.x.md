# Versión 0.9.x {#version-09x}

## Resumen {#overview}

Versión 0.9.x es una versión importante que introduce un **Sistema integral de Control de Acceso de Usuarios y Seguridad**. Esta versión transforma **duplistatus** de una aplicación de acceso abierto a un sistema seguro y multiusuario con control de acceso basado en roles, registro de auditoría integral y características de seguridad de nivel empresarial. Toda la funcionalidad existente ahora está protegida detrás de autenticación, garantizando que solo los usuarios autorizados puedan acceder y gestionar la supervisión de backups.

## Cambios Importantes {#breaking-changes}

### Autenticación Requerida {#authentication-required}

- **Todas las páginas ahora requieren autenticación** - Los usuarios deben iniciar sesión para acceder a la aplicación
- Se crea automáticamente una **cuenta de administrador por defecto**:
  - Nombre de usuario: `admin`
  - Contraseña: `Duplistatus09` (debe cambiarse en el primer inicio de sesión)
- **Las sesiones existentes se invalidan** - Todos los usuarios deben iniciar sesión después de la actualización
- **Los puntos de conexión de la API ahora requieren autenticación** - Las integraciones externas deben incluir cookies de sesión o usar solicitudes autenticadas

:::warning
**Todos los usuarios deben iniciar sesión después de actualizar a la versión 0.9.x.** Las credenciales de admin por defecto son admin/Duplistatus09. Cambie la contraseña inmediatamente después del primer inicio de sesión.
:::

### Migración del Esquema de Base de Datos {#database-schema-migration}

- **Migración automática de v3.1 a v4.0** para instalaciones existentes
- **Nuevas tablas agregadas**: `users`, `sessions`, `audit_log`
- **Backup de base de datos creado automáticamente** antes de la migración
- **Nuevas instalaciones** inician directamente con esquema v4.0 (no se requiere migración)

## Nuevas Características {#new-features}

### Control de acceso de usuario y sistema de seguridad {#user-access-control--security-system}

#### Sistema de Autenticación {#authentication-system}

- **Funcionalidad de inicio y cierre de sesión de usuario** con gestión segura de sesiones
- **Autenticación por contraseña** con hash bcrypt (factor de costo 12)
- **Mecanismo de bloqueo de cuenta** (5 intentos fallidos, bloqueo de 15 minutos)
- **Cambio de contraseña forzado** en el primer inicio de sesión
- **Aplicación de políticas de contraseñas**:
  - Mínimo 8 caracteres
  - Al menos una letra mayúscula
  - Al menos una letra minúscula
  - Al menos un número
  - Caracteres especiales opcionales
- **Funcionalidad "Recordarme"** con persistencia del nombre de usuario en localStorage
- **Autenticación basada en sesiones** con sesiones respaldadas por base de datos (reemplaza las sesiones en memoria)
- **Protección CSRF** integrada con la autenticación
- **Cookies solo HTTP** para mayor seguridad de sesión
- **Seguimiento de dirección IP y agente de usuario** para monitoreo de seguridad

#### Gestión de usuarios (Solo Admin) {#user-management-admin-only}

- **Creación de usuarios** con generación automática de contraseñas temporales
- **Lista de usuarios** con búsqueda, paginación y filtros
- **Edición de usuarios** (nombre de usuario, estado de administrador, requisito de cambio de contraseña)
- **Funcionalidad de restablecimiento de contraseña** con visualización de contraseña temporal
- **Eliminación de usuarios** con protecciones (impide la eliminación del último administrador)
- **Control de acceso basado en roles** (roles Administrador/Usuario)
- **Indicadores de estado** (Activo, Bloqueado, Debe cambiar la contraseña)
- **Seguimiento y visualización del último inicio de sesión**

#### Sistema de Registro de Auditoría {#audit-logging-system}

- **Registro de auditoría completo** para todos los cambios del sistema y acciones de usuario
- **Visor de registros de auditoría** con filtros avanzados:
  - Filtro por rango de fechas
  - Filtro por usuario
  - Filtro por acción
  - Filtro por categoría
  - Filtro por estado
- **Funcionalidad de exportación** (formatos CSV y JSON)
- **Estadísticas y análisis del registro de auditoría**
- **Período de retención configurable** (30-365 días, por defecto: 90 días)
- **Tarea programada de limpieza automática** (se ejecuta diariamente a las 2 AM UTC)
- **API de limpieza manual** con soporte para simulación (dry-run)
- **Registro de auditoría para**:
  - Eventos de autenticación (inicio de sesión, cierre de sesión, cambios de contraseña, bloqueos de cuenta)
  - Operaciones de gestión de usuarios (crear, actualizar, eliminar, restablecimientos de contraseña)
  - Cambios de configuración (correo electrónico, NTFY, plantillas, tolerancia por retraso, configuración de copias de seguridad)
  - Operaciones de copia de seguridad (recopilación, eliminación, limpieza)
  - Gestión de servidores (agregar, actualizar, eliminar)
  - Operaciones del sistema (migraciones, limpieza, notificaciones)

### Rediseño de la Página de Configuración {#settings-page-redesign}

- **Navegación moderna en barra lateral** con barra lateral plegable
- **Persistencia del estado de la barra lateral por usuario** (preferencia de plegado/expansión guardada por usuario)
- **Secciones de configuración agrupadas**:
  - **Notificaciones**: Notificaciones de copia de seguridad, Monitoreo de Copias de Seguridad, Plantillas
  - **Integraciones**: NTFY, Correo electrónico
  - **Sistema**: Servidores, Usuarios (solo administrador), Registro de auditoría
- **Barra lateral fija** que permanece visible al desplazarse
- **Diseño responsivo** con espaciado optimizado
- **Icono de configuración y título "Configuración del Sistema"** en el encabezado de la barra lateral
- **Botón Atrás** integrado en el encabezado de la aplicación

### Mejoras de la Interfaz de Usuario {#user-interface-enhancements}

- **Página de inicio de sesión independiente** con diseño moderno
- **Casilla "Recordarme"** en el formulario de inicio de sesión para mantener el nombre de usuario
- **Botones Mostrar/Ocultar contraseña** en los formularios de inicio de sesión y cambio de contraseña
- **Modal de cambio de contraseña** con lista de verificación de validación en tiempo real
- **Indicador de usuario y botón de cierre de sesión** en el encabezado de la aplicación
- **Visibilidad de la interfaz según el rol** (funciones solo para administradores ocultas a usuarios regulares)
- **Insignias e indicadores de estado** en toda la interfaz
- **Preferencias específicas por usuario** almacenadas en localStorage (por ejemplo, estado plegado de la barra lateral)

### Características de Seguridad {#security-features}

- **Hash de contraseñas** con bcrypt (estándar de la industria)
- **Sanitización de datos sensibles** en registros de auditoría (nunca se registran contraseñas, tokens ni secretos)
- **Límite de intentos** para inicios de sesión
- **Expiración de sesión** (24 horas)
- **Validación de token CSRF** para todas las operaciones que cambian el estado
- **Herramienta CLI de recuperación para administrador** para restablecer la contraseña si se queda bloqueado

### Herramientas para Desarrolladores {#developer-tools}

- **Script CLI de recuperación para administrador** (`admin-recovery`) para ejecución sencilla en contenedores Docker
- **Funciona tanto localmente como en Docker** con detección automática de rutas
- **Interfaces de TypeScript completas** (sin tipos `any`)
- **Middleware de autenticación** para protección de rutas
- **Clase de utilidad para registro de auditoría** con métodos prácticos

## 🚀 Notas de Migración {#-migration-notes}

### Desde la Versión 0.8.x {#from-version-08x}

Esta versión introduce autenticación y requiere que todos los usuarios inicien sesión. Cuándo actualiza desde la versión 0.8.x:

1. **Migración automática de la base de datos**: La aplicación migrará automáticamente el esquema de su base de datos de la v3.1 a la v4.0
2. **Copia de seguridad de la base de datos**: Se crea una copia de seguridad automática antes de la migración
3. **Cuenta de administrador por defecto**: Se crea una cuenta de administrador por defecto:
   - Nombre de usuario: `admin`
   - Contraseña: `Duplistatus09`
   - **Debe cambiar esta contraseña en el primer inicio de sesión**
4. **Invalidación de sesiones**: Todas las sesiones existentes se invalidan
5. **Inicio de sesión requerido para todos los usuarios**: Todos los usuarios deben iniciar sesión tras la actualización
6. **Autenticación de API**: Las integraciones externas deben actualizarse para incluir autenticación

### Consideraciones de Seguridad {#security-considerations}

- **Cambiar la contraseña por defecto**: Cambie inmediatamente la contraseña del administrador por defecto tras el primer inicio de sesión
- **Crear cuentas de usuario**: Cree cuentas de usuario individuales para cada persona que necesite acceso
- **Revisar registros de auditoría**: Revise regularmente los registros de auditoría para monitoreo de seguridad
- **Configurar retención**: Establezca un período de retención adecuado para los registros de auditoría según sus requisitos de cumplimiento
- **Copia de seguridad de la clave maestra**: Si actualiza desde la versión 0.8.x, asegúrese de hacer una copia de seguridad del archivo `.duplistatus.key`

### Pasos del Primer Inicio de Sesión {#first-login-steps}

1. **Iniciar sesión** con credenciales por defecto (`admin` / `Duplistatus09`)
2. **Cambiar contraseña** cuando se solicite (obligatorio en el primer inicio de sesión)
3. **Crear cuentas de usuario** para otros usuarios (Configuración → Usuarios)
4. **Configurar la retención del registro de auditoría** si es necesario (Configuración → Registro de auditoría)
5. **Revisar los registros de auditoría** para verificar que el sistema funcione correctamente

## Puntos finales de API {#api-endpoints}

### Puntos finales de autenticación {#authentication-endpoints}

- `POST /api/auth/login` - Autenticación de usuario
- `POST /api/auth/logout` - Terminación de sesión
- `GET /api/auth/me` - Información del usuario actual
- `POST /api/auth/change-password` - Cambio de contraseña

### Puntos finales de Gestión de usuarios (Solo Admin) {#user-management-endpoints-admin-only}

- `GET /api/users` - Listar usuarios
- `POST /api/users` - Crear usuario
- `PATCH /api/users/{id}` - Actualizar usuario
- `DELETE /api/users/{id}` - Eliminar usuario

### Puntos finales del Log de Auditoría {#audit-log-endpoints}

- `GET /api/audit-log` - Consultar registros de auditoría
- `GET /api/audit-log/download` - Exportar registros de auditoría
- `GET /api/audit-log/stats` - Estadísticas de auditoría
- `POST /api/audit-log/cleanup` - Limpieza manual (solo administrador)
- `GET /api/audit-log/retention` - Obtener configuración de retención
- `PATCH /api/audit-log/retention` - Actualizar retención (solo administrador)

:::note
Todos los puntos finales de API ahora requieren autenticación. Consulte la [Referencia de API](../api-reference/authentication-security.md) para obtener detalles sobre autenticación.
:::
