# Versión 1.0.x {#version-10x}

## Resumen {#overview}

Versión 1.0.x es una versión de características que introduce mejoras significativas en la Configuración de correo electrónico, las capacidades de Gestión de servidores y las herramientas de Mantenimiento de base de datos. Esta versión se enfoca en mejorar la confiabilidad de las notificaciones de correo electrónico, expandir las opciones de conexión SMTP y proporcionar mejores herramientas para gestionar la integridad de los servidores y la base de datos.

## Nuevas Características {#new-features}

### Mejoras en la Gestión de servidores {#server-management-enhancements}

- **Funcionalidad de cambio de contraseña del servidor** - Los usuarios ahora pueden cambiar las contraseñas del servidor directamente desde la página de detalle del servidor haciendo clic en el icono de clave
- **Detección y fusión de servidores duplicados** - Nueva función en el formulario de Mantenimiento de base de datos para detectar servidores duplicados y fusionarlos en una única entrada de servidor

### Mejoras en el Registro de Auditoría {#audit-logging-enhancements}

- **Registro de auditoría de subida de backup** - Todas las subidas de log de backup a través del endpoint `/api/upload` ahora se registran automáticamente en el Log de Auditoría. Cada entrada incluye Detalles del backup (ID del servidor, Nombre del servidor, Nombre de backup, Estado, Fecha, Duración, recuentos de archivos, tamaños, Advertencias y Errores) junto con la Dirección IP del cliente e información del Agente de usuario para fines de seguimiento y Seguridad

### Mejoras en la configuración de correo electrónico {#email-configuration-enhancements}

- **Aviso de configuración incompleta** - Alerta visual mostrada cuando la configuración de correo electrónico no está correctamente establecida
- **Indicadores de estado del servicio** - Etiquetas "(deshabilitado)" mostradas en las casillas de notificación cuando los servicios de correo no están configurados
- **Información del remitente personalizable**:
  - Campo `Sender Name` para personalizar el nombre mostrado como remitente en las notificaciones por correo
  - Campo `From Address` para especificar la dirección de correo mostrada como remitente
  - Nota: Algunos proveedores de correo sobrescribirán el `From Address` con el `SMTP Server Username`
- **Alternar autenticación SMTP** - Opción para habilitar o deshabilitar la autenticación SMTP, compatible con servidores que no requieren autenticación
- **Validación de formato de correo** - Validación de los campos de destinatario y dirección de origen para asegurar el formato correcto de direcciones de correo electrónico
- **Correo de prueba mejorado** - El correo de prueba ahora muestra detalles completos de la conexión, incluyendo el nombre de host del servidor SMTP, puerto, tipo de conexión, estado de autenticación, nombre de usuario (si corresponde), correo del destinatario, dirección de origen, nombre del remitente y marca de tiempo de la prueba

### Compatibilidad de tipo de conexión SMTP {#smtp-connection-type-support}

- **Múltiples tipos de conexión** - Soporte para tres tipos de conexión:
  - `Plain SMTP` (sin cifrado) en el puerto 25
  - `STARTTLS` (actualización a TLS) en el puerto 587
  - `Direct SSL/TLS` (cifrado desde el inicio) en el puerto 465
- **Tipo de conexión predeterminado** - Cambiado a `Direct SSL/TLS` para nuevas configuraciones
- **Requisitos específicos por conexión** - El campo `From Address` es obligatorio para conexiones `Plain SMTP` para asegurar una identificación adecuada del remitente del correo y cumplir con RFC 5322

## 🐛 Correcciones de Errores {#-bug-fixes}

### Correcciones del Sistema de Correo Electrónico {#email-system-fixes}

- **Corregido encabezado `From` faltante** - Solucionados errores de cumplimiento con RFC 5322 al asegurar que todos los mensajes de correo incluyan encabezados `From` adecuados
- **Corrección del cifrado de correo para STARTTLS** - Corregido el manejo del cifrado para conexiones `STARTTLS` (puerto 587)
- **Corrección de conexiones SMTP plano** - Solucionados errores de negociación TLS que hacían que las conexiones `Plain SMTP` fallaran
- **Corrección de pérdida de configuración no guardada** - Evitado que la configuración de correo no guardada se borre al cambiar la contraseña
- **Corrección de configuración SMTP obsoleta** - Solucionado el problema por el cual el punto final del correo de prueba usaba una configuración SMTP desactualizada

### Correcciones de la Interfaz de Usuario {#user-interface-fixes}

- **Corrección del toggle de visibilidad de contraseña** - Se corrigió el comportamiento de la función ver contraseña en el formulario de cambio de contraseña; al hacer clic en el icono de ojo ahora se muestra la contraseña en ambos campos de contraseña simultáneamente

### Correcciones del Sistema {#system-fixes}

- **Corrección del registro de entrada de Docker** - Se resolvió el problema por el cual el script de entrada de Docker eliminaba mensajes de registro
- **Corrección del cálculo de la Fecha de Próxima Ejecución** - Se agregó el recálculo automático de la fecha `Next Run` después de recopilar los registros de respaldo, asegurando que la fecha se actualice correctamente tras la recopilación de registros

## 🚀 Notas de Migración {#-migration-notes}

### Desde la Versión 0.9.x {#from-version-09x}

Esta versión incluye mejoras en la Configuración de correo electrónico y la Gestión de servidores. Cuándo actualiza desde la Versión 0.9.x:

1. **No se requiere migración de base de datos** - Esta versión no requiere cambios en el esquema de la base de datos
2. **Actualizaciones de configuración de correo electrónico** - Las configuraciones existentes seguirán funcionando, pero puede que desee revisar y actualizar sus ajustes para aprovechar las nuevas funciones:
   - Considere establecer un `Sender Name` personalizado para una mejor identificación del correo
   - Verifique que su `From Address` esté configurado correctamente, especialmente si utiliza conexiones `Plain SMTP`
   - Pruebe su configuración de correo usando la función mejorada de correo de prueba
3. **Tipo de conexión predeterminado** - Las nuevas configuraciones de correo usarán por defecto el tipo de conexión `Direct SSL/TLS`
4. **Gestión de contraseñas de servidor** - Ahora puede cambiar las contraseñas de los servidores directamente desde la página de detalles del servidor sin necesidad de recopilar copias de seguridad nuevamente

### Recomendaciones de configuración de correo electrónico {#email-configuration-recommendations}

- **Para uso en producción**: Utilice conexiones `STARTTLS` o `Direct SSL/TLS` para la entrega segura de correo electrónico
- **Para pruebas locales**: Las conexiones `Plain SMTP` están disponibles para redes locales de confianza
- **Requisito de Dirección de remitente**: Cuando utilice `Plain SMTP` o cuándo la autenticación está deshabilitada, asegúrese de que el campo `Dirección de remitente` esté correctamente configurado
- **Pruebe su configuración**: Siempre utilice el botón "Enviar correo de prueba" para verificar que su configuración de correo electrónico funciona correctamente

## Puntos finales de API {#api-endpoints}

### Puntos finales de configuración de correo electrónico {#email-configuration-endpoints}

- `GET /api/email/config` - Obtener configuración de correo electrónico
- `POST /api/email/config` - Actualizar configuración de correo electrónico
- `POST /api/email/test` - Enviar correo de prueba
- `DELETE /api/email/config` - Eliminar configuración de correo electrónico

### Puntos finales de gestión de servidores {#server-management-endpoints}

- `PATCH /api/servers/{id}/password` - Actualizar contraseña del servidor
- `POST /api/database/maintenance/merge-duplicate-servers` - Fusionar servidores duplicados
