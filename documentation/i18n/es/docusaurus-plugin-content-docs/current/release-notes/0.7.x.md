# Versión 0.7.x {#version-07x}

## Resumen {#overview}

Versión 0.7.x es una versión importante que introduce mejoras significativas en el esquema de base de datos, la estructura de API y la interfaz de Usuario. Esta Versión incluye cambios importantes que requieren actualizaciones en las Integraciones externas.

## Cambios importantes {#breaking-changes}

### Cambios de esquema de base de datos {#database-schema-changes}

- **Cambio de nombre de tabla**: tabla `machines` renombrada a `servers`
- **Nuevos campos**: Añadidos campos `alias`, `notes`, `created_at` y `updated_at`
- **Cambios de respuesta de API**: Nombres de campo actualizados para consistencia

### Cambios importantes de API {#api-breaking-changes}

- **Cambio de nombre de campo**: `totalMachines` → `totalServers`
- **Cambio de nombre de objeto**: `machine` → `server` en respuestas de API
- **Cambio de nombre de campo**: `backup_types_count` → `backupJobsCount`

:::warning
**Las Integraciones externas que utilizan la API deben actualizarse** para funcionar con Versión 0.7.x. Consulte la guía [Cambios importantes de API](../migration/api-changes) para obtener Detalles.
:::

## Guía de migración {#migration-guide}

### Migración automática {#automatic-migration}

La migración de 0.6.x a 0.7.x ocurre automáticamente:

1. **Creación de backup**: Se crea automáticamente un Backup de base de datos
2. **Actualización de esquema**: El esquema de base de datos se actualiza automáticamente
3. **Migración de datos**: Los datos existentes se preservan y se migran
4. **Verificación**: La migración se verifica y se registra

### Pasos manuales requeridos {#manual-steps-required}

- **Actualizar Integraciones externas**: Actualice cualquier herramienta externa que utilice la API
- **Probar funcionalidad**: Verifique que Todos los características funcionen correctamente
- **Actualizar documentación**: Actualice cualquier documentación Personalizada

## Cambios de configuración {#configuration-changes}

### Nuevas opciones de configuración {#new-configuration-options}

```yaml
# Gestión de servidores {#server-management}
Servidores:
  defaultAlias: true
  showNotes: true
  sortBy: "name" # name, Estado, lastBackup

# Configuración de API {#api-settings}
api:
  Versión: "1.0"
  rateLimit: 100
  Tiempo de espera agotado: 30000

# Configuración de base de datos {#database-settings}
database:
  backupRetention: 7
  autoVacuum: true
  optimizeIntervalo: 24
```

### Opciones obsoletas {#deprecated-options}

- `machines` configuración (use `servers` en su lugar)
- Nombres de campo de API antiguos (use nuevos nombres de campo)

## Instalación {#installation}

### Docker Compose {#docker-compose}

```yaml

  duplistatus:
    image: wsjbr/duplistatus:0.7.x
    container_name: duplistatus
```

```bash
  --restart unless-stopped \
```

## Instrucciones de actualización {#upgrade-instructions}

### Desde Versión 0.6.x {#from-version-06x}

1. **Haga una copia de seguridad de sus datos** (Backup creado automáticamente durante la actualización)
2. **Actualizar imagen de Docker**: `docker pull wsjbr/duplistatus:0.7.x`
3. **Reiniciar contenedor**: `docker-compose down && docker-compose up -d`
4. **Monitorear Logs**: `docker logs -f duplistatus`
5. **Actualizar Integraciones**: Actualice cualquier herramienta externa que utilice la API
6. **Verificar actualización**: Verifique que Todos los características funcionen correctamente

### Desde Versión 0.5.x o anterior {#from-version-05x-or-earlier}

1. **Primero actualice a 0.6.x** (si está disponible)
2. **Luego actualice a 0.7.x** usando los pasos anteriores

## Problemas conocidos {#known-issues}

### Problemas de migración {#migration-issues}

- **Bases de datos grandes**: La migración puede tardar más para bases de datos grandes
- **Espacio en disco**: Asegúrese de tener suficiente espacio en disco para el backup de migración
- **Permisos**: Verifique los permisos de archivo para el acceso a la base de datos

### Compatibilidad de API {#api-compatibility}

- **Herramientas externas**: Las herramientas externas que utilizan la API necesitan actualizaciones
- **Nombres de campo**: Algunos nombres de campo han cambiado
- **Formato de respuesta**: Algunos formatos de respuesta han cambiado

### Soluciones alternativas {#workarounds}

- **Migración**: Asegúrese de tener suficiente espacio en disco y tiempo para la migración
- **API**: Actualice las Integraciones externas para usar el nuevo formato de API
- **Pruebas**: Pruebe Todos los características después de la actualización
