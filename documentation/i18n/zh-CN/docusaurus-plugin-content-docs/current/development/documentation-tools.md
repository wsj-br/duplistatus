# 文档工具 {#documentation-tools}

文档使用 [Docusaurus](https://docusaurus.io/) 构建，位于 `documentation` 文件夹中。文档托管在 [GitHub Pages](https://wsj-br.github.io/duplistatus/) 上，不再包含在 Docker 容器镜像中。

## 文件夹结构 {#folder-structure}

```
documentation/
├── docs/              # Documentation markdown files (English source)
│   ├── api-reference/
│   ├── development/
│   ├── installation/
│   ├── migration/
│   ├── release-notes/
│   └── user-guide/
├── i18n/              # Translations (auto-generated by translation workflow)
│   ├── de/            # German
│   ├── es/            # Spanish
│   ├── fr/            # French
│   ├── pt-BR/         # Brazilian Portuguese
│   └── zh-CN/         # Simplified Chinese
├── src/               # React components and pages
│   ├── components/    # Custom React components
│   ├── css/           # Custom styles
│   └── pages/         # Additional pages (e.g., 404)
├── static/            # Static assets (images, files)
├── docusaurus.config.ts  # Docusaurus configuration
├── sidebars.ts        # Sidebar navigation configuration
└── package.json       # Dependencies and scripts
```

### 国际化 (i18n) {#internationalization-i18n}

文档使用 Docusaurus 内置的 i18n 系统，默认语言为英语。翻译内容位于 `i18n/{locale}/docusaurus-plugin-content-docs/current/`，镜像 `docs/` 文件夹的结构。

- **源文件**：`docs/**/*.md`（英语）
- **翻译文件**：`i18n/{locale}/docusaurus-plugin-content-docs/current/**/*.md`
- **UI 翻译**：`i18n/{locale}/docusaurus-theme-classic/*.json` 及其他 JSON 文件
- **本地化截图**：`i18n/{locale}/docusaurus-plugin-content-docs/current/**/assets`，由仓库根目录的 `pnpm take-screenhots` 生成。

`pnpm write-translations` 命令将 UI 字符串（来自 Docusaurus 主题和自定义组件）提取到 JSON 翻译文件中。`pnpm translate` 脚本（在 `documentation/` 中运行，委托至仓库根目录）运行 **ai-i18n-tools**，根据 `ai-i18n-tools.config.json` 翻译 markdown、JSON 和 SVG。

:::important
仅编辑 `docs/` 中的文件和 `i18n/en/` 中的源 JSON 文件。`i18n/{other-locales}/` 中的翻译 markdown 文件为自动生成，不应手动编辑。
:::

### 支持的语言 {#supported-locales}

| Locale  | 语言             | 目录                                            |
|---------|------------------|-------------------------------------------------|
| `en`    | 英语（默认）     | `docs/`（源文件）                               |
| `de`    | 德语             | `i18n/de/docusaurus-plugin-content-docs/current/`    |
| `es`    | 西班牙语         | `i18n/es/docusaurus-plugin-content-docs/current/`    |
| `fr`    | 法语             | `i18n/fr/docusaurus-plugin-content-docs/current/`    |
| `pt-BR` | 巴西葡萄牙语     | `i18n/pt-BR/docusaurus-plugin-content-docs/current/` |
| `zh-CN` | 简体中文         | `i18n/zh-CN/docusaurus-plugin-content-docs/current/`   |


## 翻译文档 {#translate-the-documentation}

文档使用 AI 驱动的翻译系统，翻译内容（markdown 文件）和 UI 字符串（来自 Docusaurus 和自定义组件）。源内容为英语（`docs/`），翻译生成目标语言包括德语、法语、西班牙语、巴西葡萄牙语和简体中文。

### 翻译工作原理 {#how-translation-works}

1. **Docusaurus UI 字符串**：`pnpm write-translations` 将主题/自定义字符串提取到 `i18n/en/*.json`。
2. **AI 翻译**（OpenRouter；配置位于仓库根目录的 `ai-i18n-tools.config.json`）：在 `documentation/` 中，`pnpm translate` 运行根目录的 `i18n:translate` 脚本（UI 字符串、SVG 和 Docusaurus markdown/JSON），按配置写入 `documentation/i18n/` 和 `src/locales/`。
3. **构建**：`pnpm build` 在 `documentation/build/` 下为所有语言生成静态 HTML。

### 运行翻译 {#running-translation}

```bash
cd documentation
pnpm translate          # Same as repo root: i18n:translate (ui + svg + docs)
pnpm translate:docs
pnpm translate:svg
pnpm translate:ui
pnpm translate:status
```

CLI 标志由 **ai-i18n-tools** 定义；从仓库根目录运行 `pnpm exec ai-i18n-tools --help`，或参阅[翻译工作流](translation-workflow.md)。

### 手动翻译覆盖 {#manual-translation-overrides}

编辑 `documentation/glossary-user.csv`（可选：清除仓库根目录 `.translation-cache/` 下的过期条目），然后重新运行相应的 `pnpm translate:*` 命令。

## 常用命令 {#common-commands}

所有命令均应在 `documentation` 目录中运行：

### 开发 {#development}

启动指定语言的开发服务器（支持热重载）：

```bash
cd documentation
pnpm start:en    # English (default)
pnpm start:fr    # French
pnpm start:de    # German
pnpm start:es    # Spanish
pnpm start:pt-br # Brazilian Portuguese
```

站点可在 `http://localhost:3000`（或下一个可用端口）访问。

### 构建 {#build}

构建生产环境文档站点：

```bash
cd documentation
pnpm build
```

这会在 `documentation/build` 目录中生成静态 HTML 文件。

### 预览生产构建 {#serve-production-build}

在本地预览生产构建：

```bash
cd documentation
pnpm serve
```

这会从 `documentation/build` 目录提供已构建的站点。

### 其他实用命令 {#other-useful-commands}

- `pnpm clear` - 清除 Docusaurus 缓存
- `pnpm typecheck` - 运行 TypeScript 类型检查
- `pnpm write-heading-ids` - 使用 Docusaurus 规则将显式 `{#id}` 标题锚点写入 markdown（在 `documentation/` 中运行，以确保各语言翻译间链接稳定）

## 生成 README.md {#generating-readmemd}

项目的 `README.md` 文件自动从 `documentation/docs/intro.md` 生成，以保持 GitHub 仓库 README 与 Docusaurus 文档同步。

生成或更新 README.md 文件：

```bash
./scripts/generate-readme-from-intro.sh
```

此脚本会：
- 从 `package.json` 提取当前版本并添加版本徽章
- 从 `documentation/docs/intro.md` 复制内容
- 将 Docusaurus 提示块（note、tip、warning 等）转换为 GitHub 风格警报
- 将所有相对 Docusaurus 链接转换为绝对 GitHub 文档 URL（`https://wsj-br.github.io/duplistatus/...`）
- 将图片路径从 `/img/` 转换为 `documentation/static/img/`，以兼容 GitHub
- 移除迁移 IMPORTANT 块，并添加带 Docusaurus 文档链接的迁移信息部分
- 使用 `doctoc` 生成目录
- 生成 `README_dockerhub.md`，采用 Docker Hub 兼容格式（将图片和链接转换为绝对 URL，将 GitHub 警报转换为 emoji 格式）
- 从 `documentation/docs/release-notes/VERSION.md` 生成 GitHub 发布说明（`RELEASE_NOTES_github_VERSION.md`）（将链接和图片转换为绝对 URL）

## 更新 Docker Hub 的 README {#update-readme-for-docker-hub}

`generate-readme-from-intro.sh` 脚本会自动生成采用 Docker Hub 兼容格式的 `README_dockerhub.md`。它会：
- 将 `README.md` 复制为 `README_dockerhub.md`
- 将相对图片路径转换为绝对 GitHub raw URL
- 将相对文档链接转换为绝对 GitHub blob URL
- 将 GitHub 风格警报（`[!NOTE]`、`[!WARNING]` 等）转换为 emoji 格式，以更好地兼容 Docker Hub
- 确保所有图片和链接在 Docker Hub 上正常工作

## 生成 GitHub 发布说明 {#generate-github-release-notes}

运行 `generate-readme-from-intro.sh` 脚本时会自动生成 GitHub 发布说明。它会：
- 从 `documentation/docs/release-notes/VERSION.md` 读取发布说明（VERSION 从 `package.json` 提取）
- 将标题从 "# Version xxxx" 更改为 "# Release Notes - Version xxxxx"
- 将相对 markdown 链接转换为绝对 GitHub 文档 URL（`https://wsj-br.github.io/duplistatus/...`）
- 将图片路径转换为 GitHub raw URL（`https://raw.githubusercontent.com/wsj-br/duplistatus/main/documentation/static/img/...`），以便在发布描述中正确显示
- 处理带 `../` 前缀的相对路径
- 保持绝对 URL（http:// 和 https://）不变
- 在项目根目录创建 `RELEASE_NOTES_github_VERSION.md`

**示例：**
```bash
# This will generate both README.md and RELEASE_NOTES_github_VERSION.md
./scripts/generate-readme-from-intro.sh
```

生成的发布说明文件可直接复制粘贴到 GitHub 发布描述中。所有链接和图片在 GitHub 发布上下文中均可正常工作。


## 为文档截取屏幕截图 {#take-screenshots-for-documentation}

```bash
pnpm take-screenshots
```

或直接运行：`pnpm take-screenshots`（如需环境变量，使用 `--env-file=.env`）。

此脚本自动为文档目的截取应用程序屏幕截图。它会：
- 启动无头浏览器（Puppeteer）
- 以管理员和普通用户身份登录
- 导航至各页面（仪表板、服务器详情、设置等）
- 在不同视口尺寸下截取屏幕截图
- 将截图保存至 `documentation/static/assets/`（英语）或 `documentation/i18n/{locale}/docusaurus-plugin-content-docs/current/assets`（其他语言）

**要求：**
- 开发服务器必须在 `http://localhost:8666` 上运行
- 必须设置环境变量，将其添加到 `.env` 文件或导出：
  - `ADMIN_PASSWORD`：管理员账户密码
  - `USER_PASSWORD`：普通用户账户密码

**选项：** `--locale` 限制截图至一个或多个语言（逗号分隔）。若省略，则捕获所有语言。有效语言：`en`、`de`、`fr`、`es`、`pt-BR`、`zh-CN`。使用 `-h` 或 `--help` 打印用法。

**示例：**
```bash
export ADMIN_PASSWORD="your-admin-password"
export USER_PASSWORD="your-user-password"
pnpm take-screenshots
# All locales (default):
pnpm take-screenshots
# Single locale:
pnpm take-screenshots --locale en
# Multiple locales:
pnpm take-screenshots --locale en,de,pt-BR
```


## 部署文档 {#deploying-the-documentation}


要将文档部署到 GitHub Pages，需要生成 GitHub 个人访问令牌。前往 [GitHub Personal Access Tokens](https://github.com/settings/tokens) 创建具有 `repo` 范围的新令牌。

获得令牌后，将其存储在 Git 凭据存储中（例如使用 `git config credential.helper store` 或系统的凭据管理器）。

然后，在 `documentation` 目录中运行以下命令将文档部署到 GitHub Pages：

```bash
pnpm run deploy
```

这将构建文档并将其推送到仓库的 `gh-pages` 分支，文档可在 [https://wsj-br.github.io/duplistatus/](https://wsj-br.github.io/duplistatus/) 访问。

## 使用文档 {#working-with-documentation}

有关完整翻译工作流（术语表管理、AI 翻译、缓存管理），请参阅[翻译工作流](translation-workflow.md)。

### 源文件 {#source-files}

- **文档内容**：`documentation/docs/` 中的英语 markdown 文件
- **UI 翻译**：`documentation/i18n/en/` 中的英语 JSON 文件（由 `pnpm write-translations` 自动生成）
- **侧边栏导航**：`documentation/sidebars.ts`
- **Docusaurus 配置**：`documentation/docusaurus.config.ts`
- **自定义 React 组件**：`documentation/src/components/`
- **静态资源**：`documentation/static/`
- **主页**：`documentation/docs/intro.md`（用于生成 `README.md` 的源文件）

### 添加新组件 {#adding-new-components}

1. 在 `documentation/src/components/` 中创建 React 组件
2. 从 `documentation/src/theme/MDXComponents.js` 导出，使其在 MDX 中可用
3. 若组件包含可翻译的 UI 字符串，运行 `pnpm write-translations` 提取它们
4. 运行 `pnpm translate` 将新字符串翻译为所有语言

### 添加新文档页面 {#adding-new-documentation-pages}

1. 在 `documentation/docs/`（或子目录）中创建新的 `.md` 文件
2. 在 `documentation/sidebars.ts` 中将其添加到侧边栏
3. 运行 `pnpm write-translations` 更新翻译文件结构
4. 运行 `pnpm write-heading-ids` 生成标题 ID（锚点）
5. 运行 `pnpm translate` 将新页面翻译为所有语言
6. 构建并测试：`pnpm build`

### 静态资源 {#static-assets}

- **图片**：放在 `documentation/static/img/` 中，在 markdown 中使用 `/img/filename.png` 引用
- **下载/PDF**：放在 `documentation/static/` 中，使用 `/filename.pdf` 引用
- **各语言资源**：若资源需要特定语言（例如截图），放在 `documentation/i18n/{locale}/docusaurus-plugin-content-docs/current/assets/`

### 构建与测试 {#build--test}

```bash
cd documentation
pnpm build            # Builds all locales
pnpm serve            # Preview the built site locally
pnpm start:en         # Development server for English
pnpm start:pt-br      # Development server for Portuguese
```

请至少在默认英语语言和一个其他语言中测试更改，以确保翻译正确显示。
