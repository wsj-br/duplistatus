# 文档工具 {#documentation-tools}

文档使用 [Docusaurus](https://docusaurus.io/) 构建，位于 `documentation` 文件夹中。文档托管在 [GitHub Pages](https://wsj-br.github.io/duplistatus/) 上，不再包含在 Docker 容器镜像中。

## 文件夹结构 {#folder-structure}

```
documentation/
├── docs/              # Documentation markdown files (English source)
│   ├── api-reference/
│   ├── development/
│   ├── installation/
│   ├── migration/
│   ├── release-notes/
│   └── user-guide/
├── i18n/              # Translations (auto-generated by translation workflow)
│   ├── de/            # German
│   ├── es/            # Spanish
│   ├── fr/            # French
│   ├── pt-BR/         # Brazilian Portuguese
│   └── zh-CN/         # Simplified Chinese
├── src/               # React components and pages
│   ├── components/    # Custom React components
│   ├── css/           # Custom styles
│   └── pages/         # Additional pages (e.g., 404)
├── static/            # Static assets (images, files)
├── docusaurus.config.ts  # Docusaurus configuration
├── sidebars.ts        # Sidebar navigation configuration
└── package.json       # Dependencies and scripts
```

### 国际化 (i18n) {#internationalization-i18n}

文档使用 Docusaurus 内置的 i18n 系统，默认语言为英语。翻译内容位于 `i18n/{locale}/docusaurus-plugin-content-docs/current/`，其结构与 `docs/` 文件夹镜像一致。

- **源文件**: `docs/**/*.md` (英语)
- **翻译文件**: `i18n/{locale}/docusaurus-plugin-content-docs/current/**/*.md`
- **UI 翻译**: `i18n/{locale}/docusaurus-theme-classic/*.json` 和其他 JSON 文件
- **本地化截图**: `i18n/{locale}/docusaurus-plugin-content-docs/current/**/assets`，由根目录下的 `pnpm take-screenhots` 生成。

`pnpm write-translations` 命令将 UI 字符串（来自 Docusaurus 主题和自定义组件）提取到 JSON 翻译文件中。`pnpm translate` 脚本（来自 `documentation/`，委托给仓库根目录）运行 **ai-i18n-tools**，根据 `ai-i18n-tools.config.json` 翻译 markdown、JSON 和 SVG。

:::important
仅编辑 `docs/` 中的文件和 `i18n/en/` 中的源 JSON 文件。`i18n/{other-locales}/` 中的翻译后 markdown 文件是自动生成的，不应手动编辑。
:::

### 支持的区域设置 {#supported-locales}

| 区域设置 | 语言 | 目录 |
|---------|----------------------|------------------------------------------------------|
| `en` | English (默认) | `docs/` (源) |
| `de` | 德语 | `i18n/de/docusaurus-plugin-content-docs/current/` |
| `es` | 西班牙语 | `i18n/es/docusaurus-plugin-content-docs/current/` |
| `fr` | 法语 | `i18n/fr/docusaurus-plugin-content-docs/current/` |
| `pt-BR` | 巴西葡萄牙语 | `i18n/pt-BR/docusaurus-plugin-content-docs/current/` |
| `zh-CN` | 简体中文 | `i18n/zh-CN/docusaurus-plugin-content-docs/current/` |

## 翻译文档 {#translate-the-documentation}

文档使用 AI 驱动的翻译系统来翻译内容（markdown 文件）和 UI 字符串（来自 Docusaurus 和自定义组件）。源内容为英语 (`docs/`)，并为德语、法语、西班牙语、巴西葡萄牙语和简体中文生成翻译。

### 翻译工作原理 {#how-translation-works}

1. **Docusaurus UI 字符串**: `pnpm write-translations` 将主题/自定义字符串提取到 `i18n/en/*.json`。
2. **AI 翻译** (OpenRouter；配置位于仓库根目录的 `ai-i18n-tools.config.json`): 从 `documentation/` 开始，`pnpm translate` 运行根目录的 `i18n:translate` 脚本（UI 字符串、SVG 和 Docusaurus markdown/JSON），并根据配置将其翻译到 `documentation/i18n/` 和 `src/locales/`。
3. **构建**: `pnpm build` 为 `documentation/build/` 下的所有区域设置生成静态 HTML。

### 运行翻译 {#running-translation}

```bash
cd documentation
pnpm translate          # Same as repo root: i18n:translate (ui + svg + docs)
pnpm translate:docs
pnpm translate:svg
pnpm translate:ui
pnpm translate:status
```

CLI 标志由 **ai-i18n-tools** 定义；请在仓库根目录下运行 `pnpm exec ai-i18n-tools --help` 或参阅 [Translation Workflow](translation-workflow.md)。

### 手动翻译覆盖 {#manual-translation-overrides}

编辑 `documentation/glossary-user.csv`（并可选地清除仓库根目录下 `.translation-cache/` 中的过期条目），然后重新运行相关的 `pnpm translate:*` 命令。

## 常用命令 {#common-commands}

全部命令应在 `documentation` 目录下运行：

### 开发 {#development}

启动针对特定区域设置且支持热重载的开发服务器：

```bash
cd documentation
pnpm start:en    # English (default)
pnpm start:fr    # French
pnpm start:de    # German
pnpm start:es    # Spanish
pnpm start:pt-br # Brazilian Portuguese
```

站点将在 `http://localhost:3000`（或下一个可用端口）可用。

### 构建 {#build}

为生产环境构建文档站点：

```bash
cd documentation
pnpm build
```

这将在 `documentation/build` 目录下生成静态 HTML 文件。

### 托管生产构建版本 {#serve-production-build}

在本地预览生产构建版本：

```bash
cd documentation
pnpm serve
```

这将从 `documentation/build` 目录托管已构建的站点。

### 其他有用命令 {#other-useful-commands}

- `pnpm clear` - 清除 Docusaurus 缓存
- `pnpm typecheck` - 运行 TypeScript 类型检查
- `pnpm write-heading-ids` - 根据 Docusaurus 规则将显式的 `{#id}` 标题锚点写入 markdown（在 `documentation/` 中运行以确保翻译后的链接稳定）

## 生成 README.md {#generating-readmemd}

项目的 `README.md` 文件由 `documentation/docs/intro.md` 自动生成，以保持 GitHub 仓库的 README 与 Docusaurus 文档同步。

要生成或更新 README.md 文件：

```bash
./scripts/generate-readme-from-intro.sh
```

此脚本将：
- 从 `package.json` 提取当前版本并添加版本徽章
- 复制 `documentation/docs/intro.md` 中的内容
- 将 Docusaurus 警告（备注、提示、警告等）转换为 GitHub 风格的警报
- 将所有相对的 Docusaurus 链接转换为绝对的 GitHub 文档 URL（`https://wsj-br.github.io/duplistatus/...`）
- 将图片路径从 `/img/` 转换为 `documentation/static/img/`，以兼容 GitHub
- 移除迁移 IMPORTANT 块，并添加一个包含指向 Docusaurus 文档链接的“迁移信息”章节
- 使用 `doctoc` 生成目录
- 生成符合 Docker Hub 格式的 `README_dockerhub.md`（将图片和链接转换为绝对 URL，将 GitHub 警告转换为基于表情符号的格式）
- 从 `documentation/docs/release-notes/VERSION.md` 生成 GitHub 发行说明（`RELEASE_NOTES_github_VERSION.md`）（将链接和图片转换为绝对 URL）

## 为 Docker Hub 更新 README {#update-readme-for-docker-hub}

`generate-readme-from-intro.sh` 脚本会自动生成符合 Docker Hub 格式的 `README_dockerhub.md`。其功能包括：
- 将 `README.md` 复制到 `README_dockerhub.md`
- 将相对图片路径转换为绝对的 GitHub raw URL
- 将相对文档链接转换为绝对的 GitHub blob URL
- 将 GitHub 风格的警告（`[!NOTE]`、`[!WARNING]` 等）转换为基于表情符号的格式，以提高与 Docker Hub 的兼容性
- 确保所有图片和链接在 Docker Hub 上正确显示

## 生成 GitHub 发行说明 {#generate-github-release-notes}

`generate-readme-from-intro.sh` 脚本在运行时会自动生成 GitHub 发行说明。其功能包括：
- 从 `documentation/docs/release-notes/VERSION.md` 读取发行说明（其中 VERSION 从 `package.json` 提取）
- 将标题从 "# Version xxxx" 更改为 "# 发行说明 - Version xxxxx"
- 将相对 Markdown 链接转换为绝对的 GitHub 文档 URL（`https://wsj-br.github.io/duplistatus/...`）
- 将图片路径转换为 GitHub raw URL（`https://raw.githubusercontent.com/wsj-br/duplistatus/main/documentation/static/img/...`），以便在发行说明中正确显示
- 处理带有 `../` 前缀的相对路径
- 保持绝对 URL（http:// 和 https://）不变
- 在项目根目录下创建 `RELEASE_NOTES_github_VERSION.md`

**示例：**

```bash
# This will generate both README.md and RELEASE_NOTES_github_VERSION.md
./scripts/generate-readme-from-intro.sh
```

生成的发行说明文件可直接复制粘贴到 GitHub 发行版描述中。所有链接和图片在 GitHub 发行版环境中均可正常显示。

## 为文档拍摄截图 {#take-screenshots-for-documentation}

```bash
pnpm take-screenshots
```

或直接运行：`pnpm take-screenshots`（如需设置环境变量，请使用 `--env-file=.env`）。

该脚本会自动为应用程序拍摄截图，用于文档编写。其功能包括：
- 启动无头浏览器（Puppeteer）
- 以管理员和普通用户身份登录
- 导航至各个页面（仪表盘、服务器详情、设置等）
- 在不同视口尺寸下拍摄截图
- 将截图保存到 `documentation/static/assets/`（英文）或 `documentation/i18n/{locale}/docusaurus-plugin-content-docs/current/assets`（其他语言环境）

**要求：**
- 开发服务器必须在 `http://localhost:8666` 上运行
- 必须设置环境变量，请将以下内容添加到您的 `.env` 文件中或通过 export 导出：
  - `ADMIN_PASSWORD`：管理员账户的密码
  - `USER_PASSWORD`：普通用户账户的密码

**选项：** `--locale` 限制截图仅针对一个或多个语言环境（逗号分隔）。如果省略，则捕获所有语言环境。有效语言环境：`en`、`de`、`fr`、`es`、`pt-BR`、`zh-CN`。使用 `-h` 或 `--help` 打印用法。

**示例：**

```bash
export ADMIN_PASSWORD="your-admin-password"
export USER_PASSWORD="your-user-password"
pnpm take-screenshots
# All locales (default):
pnpm take-screenshots
# Single locale:
pnpm take-screenshots --locale en
# Multiple locales:
pnpm take-screenshots --locale en,de,pt-BR
```

## 部署文档 {#deploying-the-documentation}

要将文档部署到 GitHub Pages，您需要生成一个 GitHub 个人访问令牌。请前往 [GitHub 个人访问令牌](https://github.com/settings/tokens) 并创建一个具有 `repo` 范围的新令牌。

当你获得令牌后，将其存储在 Git 凭据存储中（例如使用 `git config credential.helper store` 或你系统的凭据管理器）。

然后，要将文档部署到 GitHub Pages，请从 `documentation` 目录运行以下命令：

```bash
pnpm run deploy
```

这将构建文档并将其推送到仓库的 `gh-pages` 分支，文档将在 [https://wsj-br.github.io/duplistatus/](https://wsj-br.github.io/duplistatus/) 上可用。

## 处理文档 {#working-with-documentation}

关于完整的翻译工作流程（术语表管理、AI翻译、缓存管理），请参见[翻译工作流程](translation-workflow.md)。

### 源文件 {#source-files}

- **文档内容**：`documentation/docs/` 中的英文 markdown 文件
- **UI 翻译**：`documentation/i18n/en/` 中的英文 JSON 文件（由 `pnpm write-translations` 自动生成）
- **侧边栏导航**：`documentation/sidebars.ts`
- **Docusaurus 配置**：`documentation/docusaurus.config.ts`
- **自定义 React 组件**：`documentation/src/components/`
- **静态资源**：`documentation/static/`
- **主页**：`documentation/docs/intro.md`（生成 `README.md` 的源文件）

### 添加新组件 {#adding-new-components}

1. 在 `documentation/src/components/` 中创建你的 React 组件
2. 从 `documentation/src/theme/MDXComponents.js` 导出，使其在 MDX 中可用
3. 如果组件包含可翻译的 UI 字符串，运行 `pnpm write-translations` 提取它们
4. 运行 `pnpm translate` 将新字符串翻译到所有语言环境

### 添加新文档页面 {#adding-new-documentation-pages}

1. 在 `documentation/docs/` 中创建新的 `.md` 文件（或子目录）
2. 在 `documentation/sidebars.ts` 中将其添加到侧边栏
3. 运行 `pnpm write-translations` 更新翻译文件结构
4. 运行 `pnpm write-heading-ids` 生成标题 ID（锚点）
5. 运行 `pnpm translate` 将新页面翻译到所有语言环境
6. 构建并测试：`pnpm build`

### 静态资源 {#static-assets}

- **图像**：放置在 `documentation/static/img/` 中，在 markdown 中使用 `/img/filename.png` 引用
- **下载/PDF**：放置在 `documentation/static/` 中，使用 `/filename.pdf` 引用
- **特定语言环境的资源**：如果资源需要特定语言环境（如屏幕截图），放置在 `documentation/i18n/{locale}/docusaurus-plugin-content-docs/current/assets/` 中

### 构建和测试 {#build--test}

```bash
cd documentation
pnpm build            # Builds all locales
pnpm serve            # Preview the built site locally
pnpm start:en         # Development server for English
pnpm start:pt-br      # Development server for Portuguese
```

始终在至少默认的英文语言环境和另一个语言环境中测试你的更改，以确保翻译正确显示。
