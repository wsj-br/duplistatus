# 版本 0.8.x {#version-08x}

## 概览 {#overview}

版本 0.8.x 是一个重大更新，引入了显著的安全增强、改进的服务器管理能力以及支持 SMTP 邮件的全面通知系统。此版本专注于加强安全性、提升用户体验并提供更灵活的备份监控选项。

## 新功能 {#new-features}

### 服务器管理增强 {#server-management-enhancements}

- 在收集备份时**自动持久化服务器 URL 和密码**
  - 用户可以通过两种方法更新 URL 和密码：
     - 使用更新后的值重新收集备份
     - 在 [设置 → 服务器设置](../user-guide/settings/server-settings.md) 中修改设置

### 增强的备份监控 {#enhanced-backup-monitoring}

- **新增 `Backup monitoring` 配置选项卡**，具有与 Duplicati 服务器兼容的间隔设置
    - 支持自定义间隔（例如 "1D12h"）
    - 在收集备份日志期间，从 Duplicati 配置中自动更新逾期间隔
    - **建议：** 在 Duplicati 服务器中更改备份作业间隔后运行收集，以同步 duplistatus 配置
    - 通知模板现在使用 `{backup_interval}` 而不是 `{backup_interval_value}` 和 `{backup_interval_type}`。

### NTFY 设备配置 {#ntfy-device-configuration}

- **生成二维码**，用于自动配置设备以接收来自 **duplistatus** 的通知
   - 右键单击应用程序工具栏中的 `View NTFY Messages` 按钮
   - [设置 → NTFY 设置](../user-guide/settings/ntfy-settings.md) 中的“配置设备”按钮

### 改进的备份收集 {#improved-backup-collection}

- **自动 HTTP/HTTPS 检测**，连接尝试顺序为：HTTPS → 带自签名证书的 HTTPS → HTTP 回退
- `Collect Backup Logs` 界面中的 **服务器选择下拉菜单**，用于直接进行特定服务器的收集
- 主机名现在接受 **多个地址**（以逗号分隔），且必须共享相同的密码和端口
- 在 `Dashboard (table)`、[设置 → 备份监控](../user-guide/settings/backup-monitoring-settings.md) 和 [设置 → 服务器](../user-guide/settings/server-settings.md) 中添加了 **单服务器备份收集按钮**
- 为所有具有有效配置（URL 和密码）的服务器提供 **批量收集功能**
   - [设置 → 备份监控](../user-guide/settings/backup-monitoring-settings.md) 和 [设置 → 服务器](../user-guide/settings/server-settings.md) 中的 `Collect All` 按钮
   - 应用程序工具栏中 `Collect Backup Logs` 按钮的右键上下文菜单

### 增强的通知系统 {#enhanced-notification-system}

- **SMTP 邮件通知支持：**
    - 在 [设置 → 邮件设置](../user-guide/settings/email-settings.md) 中配置 SMTP 服务器
    - 为每个备份作业配置 NTFY 和/或邮件通知
    - 使用现有的 [设置 → 通知模板](../user-guide/settings/notification-templates.md) 的 HTML 格式模板
- **单项备份作业通知配置：**
    - 在 [设置 → 备份通知](../user-guide/settings/backup-notifications-settings.md) 中设置个人通知偏好
    - 当 NTFY 或邮件未正确配置时显示视觉指示（灰色图标）

- **区域设置支持：**
    - 创建了新的环境变量 `LANG` 以设置应用程序的区域设置
    - 现在通知和日志中的日期和数字格式可预测

## 🔒 安全增强 {#-security-enhancements}

### CSRF 保护 {#csrf-protection}

- 具有强大会话管理的 **基于会话的身份验证**
- 对所有更改状态的 API 请求强制执行 **CSRF 令牌验证**
- **会话在 24 小时后自动过期**；CSRF 令牌每 30 分钟刷新一次
- 在保持与外部 API 兼容的同时，确保防御跨站请求伪造

### 最小化明文密码 {#plaintext-password-minimization}

- **密码只能通过用户界面设置**；没有 API 端点会暴露存储的密码
- 在整个系统中 **最小化明文密码的操作**

### 敏感数据的高级加密 {#advanced-cryptography-for-sensitive-data}

- 使用 AES-256-GCM 进行 **敏感数据加密**（例如：密码、SMTP 凭据）
- **主密钥自动生成**并安全地存储在 `.duplistatus.key` 中
- 使用 **迭代 100,000 次的 PBKDF2**进行密钥派生以增强安全性
- **验证身份验证标签**并在使用后安全地清除内存
- **主密钥文件权限**严格设置为 0400 以实现最大程度的保护

## 🚀 迁移注意事项 {#-migration-notes}

### 从 版本 0.7.x 迁移 {#from-version-07x}

此版本包含重大安全性增强和新功能。从 版本 0.7.x 升级时：

1. **自动数据库迁移：** 应用程序将自动迁移您的数据库架构以支持新功能
2. **主密钥生成：** 将生成一个新的主密钥用于敏感数据加密
3. **会话管理：** 现有会话将被失效，并建立新的受 CSRF 保护的会话
4. **配置更新：** 部分配置键已更新以支持新功能
5. **模板变更：** 变量 `{backup_interval_value}` 和 `{backup_interval_type}` 已被 `{backup_interval}` 替换。默认 模板 将自动调整，但自定义 模板 将不会被迁移。请 检查 您的 模板 。

### 安全考量 {#security-considerations}

- **主密钥文件：** 确保 `.duplistatus.key` 文件已正确备份并安全存储
- **文件权限：** 出于安全考虑，主密钥文件将具有受限权限 (0400)
- **密码 加密：** 现有 密码 将使用新的加密系统进行加密
