
# 版本 0.8.x {#version-08x}

## 概览 {#overview}

版本 0.8.x 是一次重大更新，引入显著的安全增强、改进的服务器管理功能，以及支持 SMTP 电子邮件的综合通知系统。本版本专注于加强安全、提升用户体验，并提供更灵活的备份监控选项。

## 新功能 {#new-features}

### 服务器管理增强 {#server-management-enhancements}

- **采集备份时自动持久化服务器 URL 和密码**
  - 用户可通过两种方式更新 URL 和密码：
     - 使用更新后的值重新采集备份
     - 在[设置 → 服务器设置](../user-guide/settings/server-settings.md)中修改设置

### 增强的备份监控 {#enhanced-backup-monitoring}

- **新的「Backup monitoring」配置选项卡**，支持 Duplicati 服务器兼容的间隔设置
    - 支持自定义间隔（例如「1D12h」）
    - 采集备份日志时从 Duplicati 配置自动更新逾期间隔
    - **建议：** 在 Duplicati 服务器中更改备份作业间隔后运行采集，以同步 duplistatus 配置
    - 通知模板现使用 `{backup_interval}` 替代 `{backup_interval_value}` 和 `{backup_interval_type}`。

### NTFY 设备配置 {#ntfy-device-configuration}

- **QR 码生成**，用于自动配置设备以接收来自 **duplistatus** 的通知
   - 在应用程序工具栏中右键点击 `View NTFY Messages` 按钮
   - [设置 → NTFY 设置](../user-guide/settings/ntfy-settings.md) 中的「Configure Device」按钮

### 改进的备份采集 {#improved-backup-collection}

- **自动 HTTP/HTTPS 检测**，连接尝试顺序：HTTPS → 自签名 HTTPS → HTTP 回退
- **服务器选择下拉框**，在「Collect Backup Logs」界面中直接按服务器采集
- **主机名支持多个地址**（逗号分隔），须共享相同密码和端口
- **单服务器备份采集按钮**已添加到「Dashboard (table)」、[设置 → 备份监控](../user-guide/settings/backup-monitoring-settings.md) 和 [设置 → 服务器](../user-guide/settings/server-settings.md)
- **批量采集功能**，适用于所有配置有效（URL 和密码）的服务器
   - [设置 → 备份监控](../user-guide/settings/backup-monitoring-settings.md) 和 [设置 → 服务器](../user-guide/settings/server-settings.md) 中的「Collect All」按钮
   - 应用程序工具栏「Collect Backup Logs」按钮的右键上下文菜单

### 增强的通知系统 {#enhanced-notification-system}

- **SMTP 电子邮件通知支持：**
    - 在[设置 → 电子邮件设置](../user-guide/settings/email-settings.md)中配置 SMTP 服务器
    - 按备份作业配置 NTFY 和/或电子邮件通知
    - 使用现有[设置 → 通知模板](../user-guide/settings/notification-templates.md)的 HTML 格式模板
- **按备份作业的通知配置：**
    - 在[设置 → 备份通知](../user-guide/settings/backup-notifications-settings.md)中单独设置通知偏好
    - NTFY 或电子邮件未正确配置时显示视觉指示（灰色图标）

- **区域设置支持：**
    - 新增环境变量 `LANG` 以在应用程序中设置区域
    - 通知和日志中的日期和数字格式现已可预测

## 🔒 安全增强 {#-security-enhancements}

### CSRF 保护 {#csrf-protection}

- **基于会话的身份验证**，具有健壮的会话管理
- **CSRF 令牌验证**，对所有状态变更 API 请求强制生效
- **会话 24 小时后自动过期**；CSRF 令牌每 30 分钟刷新
- 在保持与外部 API 兼容的同时防范跨站请求伪造

### 明文密码最小化 {#plaintext-password-minimization}

- **密码仅可通过用户界面设置**；无 API 端点暴露存储的密码
- **在整个系统中最小化明文密码操作**

### 敏感数据的高级加密 {#advanced-cryptography-for-sensitive-data}

- **敏感数据加密**（例如密码、SMTP 凭据）使用 AES-256-GCM
- **主密钥自动生成**并安全存储在 `.duplistatus.key`
- **PBKDF2 100,000 次迭代**用于密钥派生以加强安全
- **验证认证标签**，使用后安全清除内存
- **主密钥文件权限**严格设为 0400 以最大程度保护

## 🚀 迁移说明 {#-migration-notes}

### 从 0.7.x 升级 {#from-version-07x}

本版本包含显著的安全增强和新功能。从 0.7.x 升级时：

1. **自动数据库迁移：** 应用程序将自动迁移数据库架构以支持新功能
2. **主密钥生成：** 将为敏感数据加密生成新主密钥
3. **会话管理：** 现有会话将失效，建立新的 CSRF 保护会话
4. **配置更新：** 部分配置键已更新以支持新功能
5. **模板变更：** 变量 `{backup_interval_value}` 和 `{backup_interval_type}` 已替换为 `{backup_interval}`。默认模板会自动调整，但自定义模板不会迁移。请检查您的模板。

### 安全注意事项 {#security-considerations}

- **主密钥文件：** 确保 `.duplistatus.key` 文件已正确备份和保护
- **文件权限：** 主密钥文件将具有受限权限（0400）以确保安全
- **密码加密：** 现有密码将使用新加密系统进行加密
