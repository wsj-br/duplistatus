# 版本 1.3.1 {#version-131}

## 概览 {#overview}

版本 1.3.1 为应用程序界面引入了完整的多语言支持，通过对额外端点进行必需的身份验证来加强 API 安全性，并修复了与主密钥检测、区域设置格式化和通知设置相关的几个错误。此版本专注于国际化改进和安全加固。

## 新功能 {#new-features}

### 多语言界面支持 {#multi-language-interface-support}

- **应用程序标题中的语言选择器**：在用户菜单中的下拉菜单中选择您首选的界面语言。支持的语言：英语、德语、法语、西班牙语和葡萄牙语（巴西）。您的选择将被保存并在会话间保持不变。
- **区域设置感知导航**：所有页面和链接现在支持带有语言前缀的 URL，以便在您选择的语言中进行一致的导航。

### 通知模板国际化 {#notification-templates-internationalization}

- **模板语言选择器**：通知模板现在支持多种语言，并提供专门的语言选择器。所有 5 种支持的语言都有预翻译的默认模板。
- **重置选项**："将此模板重置为默认值"和"全部重置为默认值"按钮，带有确认对话框，方便在更改语言时管理模板。
- **区域设置感知的通知内容**：通知中的日期、时间、数字和备份状态值现在根据模板语言进行格式化。

### 日期和数字本地化 {#date-and-number-localization}

- **区域设置感知的日期和时间格式**：整个应用程序中的日期和时间（仪表板、服务器详情、图表、工具提示）现在以适合您区域设置的格式显示（例如，欧洲区域设置为 DD/MM/YYYY，英语区域设置为 MM/DD/YYYY）。
- **区域设置感知的数字格式**：数字、文件大小和统计数据使用适合区域设置的格式（例如，德语为 1.234,56，英语为 1,234.56）。

### 右至左语言支持准备 {#rtl-support-preparation}

- **右至左语言的基础架构**：已添加基础架构以支持未来的右至左语言（阿拉伯语、希伯来语）。当前语言仍保持从左到右。

## 安全性 {#security}

### API 身份验证 {#api-authentication}

- **额外端点的必需身份验证**：以下端点现在需要有效的会话和 CSRF 令牌（之前是可选身份验证）：
  - `/api/backups/collect`
  - `/api/backups/sync-schedule`
  - `/api/cron-config`（POST）
  - `/api/notifications/check-overdue`
  - `/api/notifications/clear-overdue-timestamps`
- **依赖项安全更新**：升级了易受攻击的依赖项（fast-xml-parser、brace-expansion）以解决已知的安全问题。

---

## 错误修复 {#bug-fixes}

- **主密钥更改检测**：修复了在一起恢复数据库和 `.duplistatus.key` 文件时不正确的"密钥已更改"检测（相同的密钥）。系统现在使用密钥指纹进行可靠检测，而不是使用可能误解数据损坏为密钥更改的解密检查。
- **详情页面初始加载**：服务器详情页面现在在首次加载时正确显示逾期消息和数据，而不是仅在第一次自动刷新后显示。

---

## 改进 {#improvements}

### 国际化一致性 {#internationalisation-consistency}

- **一致的标签**：时间单位、间隔、状态标签和常用术语现在在整个应用程序中保持一致。
- **区域设置感知的工作日顺序**：备份监控中的工作日显示遵循您的语言/区域设置。可以在显示设置中覆盖。

### 认证 API 错误和成功代码（国际化）{#auth-api-error-and-success-codes-i18n}

- **稳定的错误和成功代码**：认证端点（`/api/auth/login`、`/api/auth/logout`、`/api/auth/me`、`/api/auth/change-password`、`/api/auth/password-policy`）现在除了返回英文的 `error` 或 `message` 字段外，还会返回一个 `errorCode`（成功时还会返回一个 `successCode`）。客户端使用这些代码来显示用户语言的本地化消息。
- **登录页面**：所有登录错误消息（例如“用户名和密码为必填项”、“用户名或密码无效”、“账户已锁定”）现在在五个语言环境中登录页面均已翻译。
- **更改密码弹窗**：更改密码 API 的所有错误和成功消息现在在更改密码弹窗中均已翻译。
- **API 参考文档**：[认证与安全](../api-reference/authentication-security.md#auth-api-error-and-success-codes-i18n) API 文档列出了所有错误和成功代码，并说明了如何将其用于国际化（i18n）。

### 用户界面改进 {#ui-improvements}

- **服务器筛选器宽度**：备份通知表单中的服务器筛选器宽度已增加（从 260px 增至 360px），以提升可用性。
- **批量选择**：改进批量选择界面，支持备份数量的正确复数形式。
- **帮助工具提示**：改进帮助工具提示文本，增加页面名称上下文。

### 文档 {#documentation}

- **配置页面重命名**：时区和语言配置页面已从 `configure-tz-lang` 重命名为 `configure-tz`。
- **通知模板指南**：更新用户指南，记录新的模板语言选择器、重置按钮及语言行为。
- **多语言文档**：用户指南现在说明如何通过用户菜单中的语言选择器更改界面语言。

## 迁移说明 {#migration-notes}

### 从版本 1.3.0 升级 {#from-version-130}

升级到版本 1.3.1 时：

1. **API 认证**：调用备份采集、同步计划、cron 配置或通知检查/清除端点的脚本或工具现在必须包含有效的会话认证和 CSRF 令牌。

2. **无需数据库迁移**：此版本无需任何数据库模式更改。

3. **语言环境支持**：环境变量 `LANG` 不再使用。您可以在 `Settings > Templates` 页面上定义通知的语言。

## 更新日志 {#changelog}

### 版本 1.3.1 更新内容 {#version-131-changes}

- **新增**：支持多语言界面，应用页眉中提供语言选择器（en、de、fr、es、pt-BR）

- **新增**：为未来支持从右到左书写的语言（RTL）做好准备
- **变更**：备份采集、同步计划、cron 配置和通知端点现在需要 API 认证
- **变更**：文档页面从 configure-tz-lang 重命名为 configure-tz
- **变更**：用户指南已更新，包含通知模板和多语言支持说明
- **修复**：修复在使用相同主密钥恢复数据库时错误触发主密钥变更检测的问题
- **修复**：修复恢复后出现“数据库磁盘映像损坏”错误的问题

- **修复**：详情页首次加载时现在能正确显示逾期消息和数据
- **修复**：本地化文档的截图已更新以匹配翻译内容
- **修复**：登录和认证 API 错误消息现已翻译（认证端点返回 errorCode/successCode；登录页面和更改密码弹窗将代码映射为本地化字符串）
- **改进**：提升国际化一致性（应用中的标签和术语）
- **改进**：帮助按钮现在以当前界面语言打开文档。
- **改进**：用户界面改进（服务器筛选器宽度、批量选择复数形式、帮助工具提示）
- **安全**：升级存在漏洞的依赖项（fast-xml-parser、brace-expansion）
