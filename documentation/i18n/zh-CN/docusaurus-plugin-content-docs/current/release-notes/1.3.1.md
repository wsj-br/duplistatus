# 版本 1.3.1 {#version-131}

## 概览 {#overview}

版本 1.3.1 为应用程序界面引入完整多语言支持，通过额外端点的必需身份验证加强 API 安全，并修复与主密钥检测、区域格式和通知设置相关的多项错误。本版本专注于国际化改进和安全加固。

## 新功能 {#new-features}

### 多语言界面支持 {#multi-language-interface-support}

- **应用程序标题栏语言选择器**：从用户菜单下拉框中选择首选界面语言。支持语言：English、Deutsch、Français、Español 和 Português (BR)。您的选择会保存并在会话间持久化。
- **区域感知导航**：所有页面和链接现支持带区域前缀的 URL，在所选语言中保持一致的导航。

### 通知模板国际化 {#notification-templates-internationalization}

- **模板语言选择器**：通知模板现支持多种语言，具有专用语言选择器。所有 5 种支持语言均有预翻译的默认模板。
- **重置选项**：「Reset this template to default」和「Reset all to default」按钮，带确认对话框，便于更改语言时管理模板。
- **区域感知的通知内容**：通知中的日期、时间、数字和备份状态值现根据模板语言格式化。

### 日期和数字本地化 {#date-and-number-localization}

- **区域感知的日期和时间格式化**：整个应用程序（仪表板、服务器详情、图表、工具提示）中的日期和时间现以适合您区域的格式显示（例如欧洲区域为 DD/MM/YYYY，英语为 MM/DD/YYYY）。
- **区域感知的数字格式化**：数字、文件大小和统计使用适合区域的格式（例如德语为 1.234,56，英语为 1,234.56）。

### RTL 支持准备 {#rtl-support-preparation}

- **从右到左语言的基础**：已添加基础设施以支持未来的 RTL 语言（阿拉伯语、希伯来语）。当前语言仍为从左到右。

## 安全 {#security}

### API 身份验证 {#api-authentication}

- **额外端点需要身份验证**：以下端点现需要有效会话和 CSRF 令牌（先前为可选身份验证）：
  - `/api/backups/collect`
  - `/api/backups/sync-schedule`
  - `/api/cron-config` (POST)
  - `/api/notifications/check-overdue`
  - `/api/notifications/clear-overdue-timestamps`
- **依赖安全更新**：升级存在漏洞的依赖（fast-xml-parser、brace-expansion）以解决已知安全问题。

---

## 错误修复 {#bug-fixes}

- **主密钥变更检测**：修复同时恢复数据库和 `.duplistatus.key` 文件（相同密钥）时错误的「密钥已更改」检测。系统现使用密钥指纹进行可靠检测，而非可能将数据损坏误解为密钥变更的基于解密的检查。
- **详情页初始加载**：服务器详情页现首次加载时正确显示逾期消息和数据，而非仅在首次自动刷新后。

## 改进 {#improvements}

### 国际化一致性 {#internationalisation-consistency}

- **一致的标签**：时间单位、间隔、状态标签和常用术语现贯穿应用程序一致显示。
- **区域感知的星期排序**：备份监控中的星期显示尊重您的语言/区域。可在显示设置中覆盖。

### Auth API 错误和成功代码（i18n） {#auth-api-error-and-success-codes-i18n}

- **稳定的错误和成功代码**：Auth 端点（`/api/auth/login`、`/api/auth/logout`、`/api/auth/me`、`/api/auth/change-password`、`/api/auth/password-policy`）现除英文 `error` 或 `message` 字段外还返回 `errorCode`（成功时返回 `successCode`）。客户端使用这些代码以用户语言显示本地化消息。
- **登录页面**：所有登录错误消息（例如「Username and password are required」、「Invalid username or password」、「Account locked」）现已在登录页面为所有五种区域翻译。
- **更改密码模态框**：更改密码模态框中所有更改密码 API 错误和成功消息现已翻译。
- **API 参考**：[Authentication & Security](../api-reference/authentication-security.md#auth-api-error-and-success-codes-i18n) API 文档列出所有错误和成功代码并说明如何用于 i18n。

### UI 改进 {#ui-improvements}

- **服务器筛选宽度**：备份通知表单中服务器筛选宽度增加（260px → 360px）以提升可用性。
- **批量选择**：增强批量选择 UI，对备份计数提供正确的复数形式支持。
- **帮助工具提示**：改进帮助工具提示文本，含页面名称上下文。

### 文档 {#documentation}

- **配置页面重命名**：时区和语言配置页面已从 `configure-tz-lang` 重命名为 `configure-tz`。
- **通知模板指南**：更新用户指南以记录新模板语言选择器、重置按钮和语言行为。
- **多语言文档**：用户指南现记录如何通过用户菜单中的语言选择器更改界面语言。

## 迁移说明 {#migration-notes}

### 从 1.3.0 升级 {#from-version-130}

升级到 1.3.1 时：

1. **API 身份验证**：调用 backups collect、sync-schedule、cron-config 或 notification check/clear 端点的脚本或工具现必须包含有效会话身份验证和 CSRF 令牌。

2. **无需数据库迁移**：本版本不需要任何数据库架构变更。

3. **区域支持**：环境变量 `LANG` 不再使用。您可在 `Settings > Templates` 页面定义通知语言。
