# 版本 1.3.0 {#version-130}

## 概览 {#overview}

版本 1.3.0 引入了可配置的密码策略、供管理员使用的新应用程序日志查看器、基于区域设置的星期显示，以及多项用户体验改进。此版本重点增强管理工具并改进国际化支持。

## 新功能 {#new-features}

### 可配置的密码策略 {#configurable-password-policy}

- **密码策略环境变量**：通过环境变量配置密码验证要求：
  - `PWD_ENFORCE`：设置为 `false` 以禁用密码复杂性要求（大写字母、小写字母、数字）。默认强制执行所有规则。禁用后，仅强制执行最小长度。
  - `PWD_MIN_LEN`：设置密码最小长度（字符数）。默认为 8 个字符。
  - 默认密码检查（防止使用默认管理员密码）始终生效，不受 `PWD_ENFORCE` 设置影响。

### 应用程序日志查看器 {#application-logs-viewer}

- **新的仅限管理员访问的日志查看器**：可直接从“设置”页面查看和下载应用程序日志：
  - 访问合并的 `application.log` 文件及轮转后的日志版本
  - 可配置的行数选择器（100、500、1000、5000、10000 行）
  - 实时搜索过滤器，支持不区分大小写的文本匹配
  - 自动滚动开关，用于实时日志监控并自动轮询
  - 复制到剪贴板，并将过滤后的日志导出为文本文件
  - 文件元数据显示行数、文件大小和最后修改时间戳
  - 容器启动时进行日志轮转（可通过 `LOG_ROTATION_VERSIONS` 配置，默认值：5）

### 上下文感知用户指南 {#context-aware-user-guide}

- **动态帮助按钮**：应用页头中的“用户指南”按钮现在会打开与当前页面相关的文档。工具提示将更新为显示“`Page Name` 的帮助”，以指示将打开的文档。

### 服务器配置增强 {#server-configuration-enhancement}

- **快速访问 Duplicati UI**：右键单击服务器配置按钮或服务器选择弹出框中的任何服务器项，将在新标签页中打开 Duplicati 传统 UI（`/ngax`）。

## 🔧 改进 {#-improvements}

### 基于区域设置的星期显示 {#locale-aware-weekday-display}

- **国际化星期排序**：备份监控表单中的星期显示现在会遵循浏览器的区域设置。星期按区域设置正确排序（英国/欧洲为星期一至星期日，美国/巴西为星期日至星期六）。

### 增强的 SMTP 错误处理 {#enhanced-smtp-error-handling}

- **改进的错误消息**：SMTP 错误消息现在提供更准确的指导：
  - 认证错误明确指出用户名/密码问题
  - SSL/TLS 版本错误被正确识别
  - 仅在真正相关时才显示连接类型建议

### 提示通知行为 {#toast-notification-behaviour}

- **持久性错误通知**：错误提示现在将持续显示，直到手动关闭，确保您不会错过重要错误消息。成功和信息提示将在 3 秒后自动消失。

### 通知设置表格 {#notification-settings-table}

- **简化布局**：通知设置表格已简化，合并了“服务器 / 备份”列，并使用上下文图标指示附加目的地配置状态。

## 🚀 迁移注意事项 {#-migration-notes}

### 自版本 1.2.x 起 {#from-version-12x}

升级到版本 1.3.0 时：

1. **密码策略**：新密码策略默认强制执行所有复杂度要求。如果您需要更简单的密码策略，请配置 `PWD_ENFORCE` 和 `PWD_MIN_LEN` 环境变量。

2. **应用程序日志**：所有容器输出（服务器和 cron 服务）现在都整合到一个 `data/logs/application.log` 文件中。

3. **日志轮转**：日志轮转可通过 `LOG_ROTATION_VERSIONS` 环境变量进行配置（默认：5 个版本）。

4. **无需数据库迁移**：本次发布不需要任何数据库模式更改。

## API 端点 {#api-endpoints}

本版本引入了以下 API 端点：

### 密码策略端点 {#password-policy-endpoint}

- `GET /api/auth/password-policy` - 返回当前密码策略配置

### 应用程序日志端点 {#application-logs-endpoints}

- `GET /api/application-logs` - 查看日志，支持文件选择和 tail 功能
- `GET /api/application-logs/export` - 将过滤后的日志导出为文本文件

### 已移除的端点 {#removed-endpoints}

本版本中移除了以下未使用的 API 端点：

- `DELETE /api/backups/[backupId]` - 仅用于开发的端点，从未被使用
- `GET /api/audit-log/stats` - 审计日志查看器未调用该端点

- **移除**：移除了未被应用程序调用的未使用 API 端点 (`DELETE /api/backups/[backupId]` 和 `GET /api/audit-log/stats`)
