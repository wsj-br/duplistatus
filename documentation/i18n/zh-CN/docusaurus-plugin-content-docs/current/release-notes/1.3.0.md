# 版本 1.3.0 {#version-130}

## 概览 {#overview}

版本 1.3.0 引入可配置密码策略、面向管理员的新应用程序日志查看器、区域感知的星期显示以及多项用户体验改进。本版本专注于增强管理工具和改进国际化支持。

## 新功能 {#new-features}

### 可配置密码策略 {#configurable-password-policy}

- **密码策略环境变量**：通过环境变量配置密码验证要求：
  - `PWD_ENFORCE`：设为 `false` 可禁用密码复杂度要求（大写、小写、数字）。默认强制所有规则。禁用时仅强制最小长度。
  - `PWD_MIN_LEN`：设置最小密码长度（字符数）。默认 8 个字符。
  - 默认密码检查（防止使用默认管理员密码）无论 `PWD_ENFORCE` 设置如何始终强制。

### 应用程序日志查看器 {#application-logs-viewer}

- **新的仅管理员日志查看器**：直接从设置页面查看和下载应用程序日志：
  - 访问合并的 `application.log` 文件和轮换日志版本
  - 可配置行数选择器（100、500、1000、5000、10000 行）
  - 实时搜索筛选，不区分大小写文本匹配
  - 自动滚动切换，用于实时日志监控和自动轮询
  - 复制到剪贴板和将筛选日志导出为文本文件
  - 文件元数据显示行数、文件大小和最后修改时间戳
  - 容器启动时日志轮换（通过 `LOG_ROTATION_VERSIONS` 配置，默认：5）

### 上下文感知用户指南 {#context-aware-user-guide}

- **动态帮助按钮**：应用程序标题栏中的用户指南按钮现打开与当前页面相关的文档。工具提示更新为显示「Help for `Page Name`」，指示将打开哪份文档。

### 服务器配置增强 {#server-configuration-enhancement}

- **快速访问 Duplicati UI**：右键点击服务器配置按钮或服务器选择弹出框中的任何服务器项，在新标签页中打开 Duplicati 旧版 UI（`/ngax`）。

## 🔧 改进 {#-improvements}

### 区域感知的星期显示 {#locale-aware-weekday-display}

- **国际化星期排序**：备份监控表单中的星期显示现尊重浏览器区域设置。星期根据区域以正确顺序显示（英国/欧洲为周一至周日，美国/巴西为周日至周六）。

### 增强的 SMTP 错误处理 {#enhanced-smtp-error-handling}

- **改进的错误消息**：SMTP 错误消息现提供更准确的指导：
  - 身份验证错误清楚指示用户名/密码问题
  - SSL/TLS 版本错误被正确识别
  - 连接类型建议仅在真正相关时出现

### Toast 通知行为 {#toast-notification-behaviour}

- **持久错误通知**：错误 toast 现保持显示直到手动关闭，确保不会错过重要错误消息。成功和信息 toast 3 秒后自动消失。

### 通知设置表 {#notification-settings-table}

- **简化布局**：通知设置表已精简，合并「Server / Backup」列，上下文图标指示附加目标配置状态。

## 🚀 迁移说明 {#-migration-notes}

### 从 1.2.x 升级 {#from-version-12x}

升级到 1.3.0 时：

1. **密码策略**：新密码策略默认强制所有复杂度要求。若需要更简单的密码策略，请配置 `PWD_ENFORCE` 和 `PWD_MIN_LEN` 环境变量。

2. **应用程序日志**：所有容器输出（服务器和 cron 服务）现合并到单个 `data/logs/application.log` 文件。

3. **日志轮换**：日志轮换通过 `LOG_ROTATION_VERSIONS` 环境变量配置（默认：5 个版本）。

4. **无需数据库迁移**：本版本不需要任何数据库架构变更。

## API 端点 {#api-endpoints}

本版本引入以下 API 端点：

### 密码策略端点 {#password-policy-endpoint}

- `GET /api/auth/password-policy` - 返回当前密码策略配置

### 应用程序日志端点 {#application-logs-endpoints}

- `GET /api/application-logs` - 查看日志，支持文件选择和 tail 功能
- `GET /api/application-logs/export` - 将筛选日志导出为文本文件

### 已移除端点 {#removed-endpoints}

本版本移除以下未使用的 API 端点：

- `DELETE /api/backups/[backupId]` - 仅开发用端点，从未使用
- `GET /api/audit-log/stats` - 审计日志查看器未调用此端点
