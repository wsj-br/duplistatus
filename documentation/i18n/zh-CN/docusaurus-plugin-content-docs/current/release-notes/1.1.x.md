# 版本 1.1.x {#version-11x}

## 概览 {#overview}

版本 1.1.x 是一次次要版本，解决逾期备份检测的关键问题，改进 Podman 容器兼容性，并引入容器架构改进。本版本专注于修复逾期备份检测准确性，以及解决 Podman 环境中的网络问题。

## 新功能 {#new-features}

### 用户界面增强 {#user-interface-enhancements}

- **自动刷新按钮右键菜单** — 在应用程序工具栏中右键点击自动刷新按钮可打开显示设置页面，快速访问显示偏好
- **改进的采集备份模态框** — 选择服务器时，密码字段标签现明确显示「(only fill if password changed)」，更清楚何时需要填写该字段

### 文档改进 {#documentation-improvements}

- **文档托管迁移** — 文档已从 Docker 容器镜像迁移至 GitHub Pages：https://wsj-br.github.io/duplistatus/user-guide/overview，便于访问和更新
- 应用程序标题栏中的用户指南链接现指向外部文档站点

---

## 🐛 错误修复 {#-bug-fixes}

### 关键：逾期备份检测 — 完全重新实现 {#critical-overdue-backup-detection---complete-reimplementation}

- **修复逾期备份检测** — 使用新算法完全重新实现逾期备份检测系统，正确跟踪预期备份日期：
  - 算法确保手动备份不会改变计划备份时间（例如晚上 10 点的手动备份不会改变下午 2 点的计划）
  - 修复备份监控设置中「Next Run」列显示错误日期
  - 通过 `/api/upload` 或 `/api/backups/collect` 接收备份后立即更新备份设置
  - 对于未与 Duplicati 同步的新服务器，使用首次备份的时间作为计划参考
  - 更多详情见 [Overdue Detection Algorithm](https://github.com/wsj-br/duplistatus/blob/main/dev/OVERDUE_DETECTION_ALGORITHM.md) 文档

### Podman 容器修复 {#podman-container-fixes}

- **修复 Podman Pod 网络** — 解决在 Podman Pod 中运行时无法从外部访问应用程序的问题。服务器现可在 Podman Pod 和独立容器模式下正确绑定到所有接口（`0.0.0.0:9666`）
- **修复 Tailscale 主机名的 DNS 解析** — 解决 Podman 容器中无法连接 Tailscale 主机名或自定义 DNS 服务器的 DNS 解析问题。安装文档已更新说明。

### 电子邮件配置修复 {#email-configuration-fixes}

- **修复 SMTP 空用户名/密码处理** — 解决清除电子邮件配置中的用户名或密码字段后仍在数据库中存储加密值的问题。空字段现正确存储为空字符串并可正确清除

### 系统改进 {#system-improvements}

- **减少控制台日志** — AuditLogger 现仅将失败的登录尝试记录到控制台；所有其他审计条目仅写入数据库

## 🚀 迁移说明 {#-migration-notes}

### 从 1.0.x 升级 {#from-version-10x}

升级到 1.1.0 时：

1. **文档位置变更** — 文档现托管在 GitHub Pages 上，不再包含在 Docker 容器中。应用程序标题栏中的用户指南链接现指向外部文档站点

2. **Podman 容器配置** — 若使用 Podman Pod，请使用 `--dns` 和 `--dns-search` 标志在 Pod 级别设置 DNS 配置。应用程序现可在 Podman Pod 和独立容器中正常工作

3. **逾期备份检测** — 逾期备份检测系统已使用新算法完全重新实现，保留计划时间。现有备份监控配置将继续工作，但您可能在备份监控设置中看到更准确的「Next Run」日期。新算法确保手动备份不会偏移计划备份时间

4. **无需数据库迁移** — 本版本不需要任何数据库架构变更
