# 版本 1.1.x {#version-11x}

## 概览 {#overview}

版本 1.1.x 是一个次要版本，解决了逾期备份检测的关键问题，提高了 Podman 容器的兼容性，并引入了容器架构的改进。此版本重点在于修复逾期备份检测的准确性以及解决 Podman 环境中的网络问题。

## 新功能 {#new-features}

### 用户界面增强 {#user-interface-enhancements}

- **自动刷新按钮的右键上下文菜单** - 右键单击应用程序工具栏中的自动刷新按钮可打开“显示设置”页面，以便快速访问显示首选项
- **改进的采集备份模态窗口** - 选中服务器时，密码字段标签现在会清晰地显示“（仅在密码更改时填写）”，使该字段何时需要填写更加明确

### 文档改进 {#documentation-improvements}

- **文档托管迁移** - 文档已从 Docker 容器镜像迁移至 GitHub Pages (https://wsj-br.github.io/duplistatus/user-guide/overview)，以提高可访问性并简化更新
- 应用页眉中的用户指南链接现在指向外部文档站点

---

## 🐛 错误修复 {#-bug-fixes}

### 关键：逾期备份检测 - 完全重新实现 {#critical-overdue-backup-detection---complete-reimplementation}

- **修复逾期备份检测** - 完全重新实现了逾期备份检测系统，采用新算法以正确跟踪预期备份日期：
  - 该算法确保手动备份不会更改计划备份时间（例如，晚上 10 点的手动备份不会更改下午 2 点的计划时间）
  - 修复了备份监控设置中“下次运行”列显示日期错误的问题
  - 在通过 `/api/upload` 或 `/api/backups/collect` 接收备份后，备份设置现在会立即更新
  - 对于没有 Duplicati 同步的新服务器，将使用首次备份的时间点作为计划参考
  - 欲了解更多详情，请参阅 [逾期检测算法](https://github.com/wsj-br/duplistatus/blob/main/dev/OVERDUE_DETECTION_ALGORITHM.md) 文档

### Podman 容器修复 {#podman-container-fixes}

- **修复 Podman pod 网络** - 解决了在 Podman pod 中运行时无法从外部访问应用程序的问题。现在服务器在 Podman pod 和独立容器模式下均能正确绑定到所有接口 (`0.0.0.0:9666`)
- **修复 Tailscale 主机名的 DNS 解析** - 解决了 Podman 容器中导致无法连接到 Tailscale 主机名或自定义 DNS 服务器的 DNS 解析问题。已在安装文档中更新相关说明。

### 邮件配置修复 {#email-configuration-fixes}

- **修复 SMTP 空用户名/密码处理** - 解决了在邮件配置中清除用户名或密码字段后，数据库中仍存储加密值的问题。空字段现在将正确存储为空字符串，并可以被正确清除

### 系统改进 {#system-improvements}

- **减少控制台日志** - AuditLogger 现在仅将失败的登录尝试记录到控制台；所有其他审计条目仅写入数据库

## 🚀 迁移注意事项 {#-migration-notes}

### 从版本 1.0.x 升级 {#from-version-10x}

升级到版本 1.1.0 时：

1. **文档位置变更** - 文档现在托管在 GitHub Pages 上，而不再包含在 Docker 容器中。应用页眉中的用户指南链接现在指向外部文档站点

2. **Podman 容器配置** - 如果使用 Podman pod，请确保使用 `--dns` 和 `--dns-search` 标志在 pod 级别设置 DNS 配置。应用程序现在在 Podman pod 和独立容器中均能正常工作

3. **逾期备份检测** - 逾期备份检测系统已使用新算法完全重新实现，该算法可保留计划的时间点。现有的备份监控配置将继续有效，但您可能会在备份监控设置中注意到更准确的“下次运行”日期。新算法确保手动备份不会偏移计划备份时间

4. **无需数据库迁移** - 此版本不需要任何数据库架构更改
