# 版本 1.4.1 {#version-141}

## 概览 {#overview}

版本 1.4.1 通过内联图表控件、增强的筛选功能、系统主题支持和区域格式选项，显著改善了仪表板体验。本版本还修复了影响图表渲染、备份详情加载和服务器连接检测的若干错误，并解决了依赖项中的安全漏洞。

---

## 新功能 {#new-features}

### 内联图表控件 {#inline-chart-controls}

- **图表时间范围选择器**：图表面板标题中的股票图风格药丸按钮（`1W | 2W | 1M | 3M`），无需进入显示设置即可快速切换时间范围。与显示设置共享状态。
- **图表样式切换（issue #54）**：直接从图表面板标题切换折线图和柱状图的切换按钮。持久化并在所有面板间同步。
- **简化的时间范围**：从 7 个选项减少到 4 个聚焦范围（1 周、2 周、1 个月、3 个月），默认 1 个月。

### 显示和格式选项 {#display-and-formatting-options}

- **系统主题支持**：显示设置现包含「系统（跟随 OS）」选项，自动匹配操作系统的浅色/深色偏好。
- **格式区域覆盖（issue #59）**：用户可在设置 > 显示设置中独立于 UI 语言选择格式区域（支持 416 个区域）。日期、时间和数字格式遵循所选格式区域，并提供实时预览。

### 服务器筛选 {#server-filtering}

- **仪表板服务器筛选**：搜索字段移至应用标题栏，与自动刷新和工具栏操作一起。控件为空时仅显示搜索图标；悬停/点击展开文本字段。
- **改进的筛选匹配（issue #53）**：筛选现匹配服务器 id、URL 和任何备份任务名称。修复了筛选更改时概览卡片网格不更新的问题。
- **设置筛选**：在设置 > 备份监控和设置 > 服务器中的服务器列表添加了筛选输入。

## 改进 {#improvements}

### 图表数据处理 {#chart-data-handling}

- **滚动时间窗口**：时间范围现使用滚动天数窗口：1W = 最近 7 天，2W = 最近 14 天，1M = 最近 30 天，3M = 最近 90 天（此前基于日历）。
- **数据合并**：所有图表数据点在分桶前按日历日合并。同一天的多次备份适当聚合。
- **限制数据点**：图表现强制最多 30 个数据点，以提高性能和可读性。
- **工具提示简化**：工具提示现仅显示日期（例如 "5/12/26"），而非日期和时间。

### 备份调度 {#backup-scheduling}

- **基于 UTC 的计算**：下次预期备份和逾期检测现以 UTC 推进间隔和星期几检查，避免主机时区跨越 DST 时的一小时 UTC 漂移。
- **增加逾期容差**：新安装和默认值现使用 2 小时而非 1 小时的逾期容差。

### 连接和可靠性 {#connectivity-and-reliability}

- **连接丢失检测**：添加 `/api/ping` 端点和基于标题栏的 30 秒连接探测。刷新和自动刷新失败现打开连接丢失模态框，不暴露原始错误文本。
- **UI 偏好持久化**：修复了导致用户 UI 偏好（概览侧边面板、图表时间范围、图表样式）在页面刷新时不持久化的竞态条件。

### 文档 {#documentation}

- **Duplicati 远程访问说明（issue #40）**：文档现澄清 Duplicati 远程访问为可选，仅当需要指向 Duplicati UI 的直接链接时才必需。

## 错误修复 {#bug-fixes}

- **概览图表面板忽略时间范围筛选**：修复了面板无论所选时间范围如何都显示所有数据点的问题。面板现从 API 获取带正确时间范围参数的数据。
- **备份详情 OOM 崩溃（issue #62）**：通过优化查询和 UI 渲染防止 `JavaScript heap out of memory`。备份日志现按需获取而非一次全部加载，最多 1000 行。
- **备份版本解析（issue #65）**：`convertTimestampToISO()` 现处理 Duplicati 在不同系统区域设置中使用的所有常见区域特定时间戳格式。

## 安全 {#security}

### 依赖项漏洞修复 {#dependency-vulnerability-fixes}

- **npm audit 漏洞**：解决了 7 个漏洞（1 个高危，6 个中危）：
  - 将 `next` 更新至 `16.3.0-canary.19` 以修复 `postcss@8.4.31`（XSS）
  - 添加 `.pnpmfile.cjs` 强制 `serialize-javascript@^7.0.5`（RCE、DoS）
  - 将 `mermaid` override 更新至 `>=11.14.1`（4 个中危漏洞）
  - 添加显式 `postcss@^8.5.14` 和 `serialize-javascript@^7.0.5` 依赖

- **pnpm overrides**：为传递依赖 `uuid >=14.0.0` 和 `postcss >=8.5.10` 添加根 `pnpm.overrides`。

- **高危漏洞**：通过 pnpm overrides 修复 `minimatch >=3.1.4` 和 `serialize-javascript >=7.0.3` 的 4 个高危漏洞。

## 迁移说明 {#migration-notes}

### 从版本 1.3.2 {#from-version-132}

升级到 1.4.1 时：

1. **无需数据库迁移**：本版本不需要任何数据库架构更改。
2. **新默认逾期容差**：新安装默认 2 小时逾期容差而非 1 小时。现有设置保持不变。
3. **图表偏好**：图表时间范围和样式偏好现以不同方式存储，升级后将重置为默认值（1 个月、折线图）。
4. **依赖项**：运行 `pnpm install` 以应用更新的 lockfile 和 overrides。
