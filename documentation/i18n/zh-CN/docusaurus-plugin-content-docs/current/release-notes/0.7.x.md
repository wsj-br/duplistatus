

# 版本 0.7.x {#version-07x}

## 概览 {#overview}

版本 0.7.x 是一次重大版本，对数据库架构、API 结构和用户界面进行显著改进。本版本包含破坏性变更，需要更新外部集成。

## 破坏性变更 {#breaking-changes}

### 数据库架构变更 {#database-schema-changes}
- **表重命名**：`machines` 表重命名为 `servers`
- **新字段**：新增 `alias`、`notes`、`created_at` 和 `updated_at` 字段
- **API 响应变更**：更新字段名以保持一致

### 不向后兼容的 API 变更 {#backward-incompatible-api-changes}
- **字段重命名**：`totalMachines` → `totalServers`
- **对象重命名**：API 响应中 `machine` → `server`
- **字段重命名**：`backup_types_count` → `backupJobsCount`

:::warning
**使用 API 的外部集成必须更新**才能与 0.7.x 版本配合工作。详情见[不向后兼容的 API 变更](../migration/api-changes)指南。
:::

## 配置变更 {#configuration-changes}

### 新配置选项 {#new-configuration-options}
```yaml
# Server management
servers:
  defaultAlias: true
  showNotes: true
  sortBy: "name" # name, status, lastBackup

# API settings
api:
  version: "1.0"
  rateLimit: 100
  timeout: 30000

# Database settings
database:
  backupRetention: 7
  autoVacuum: true
  optimizeInterval: 24
```

### 已弃用选项 {#deprecated-options}
- `machines` 配置（改用 `servers`）
- 旧 API 字段名（使用新字段名）

## 安装 {#installation}

### Docker Compose {#docker-compose}
```yaml
services:
  duplistatus:
    image: wsjbr/duplistatus:0.7.x
    container_name: duplistatus
    restart: unless-stopped
    ports:
      - "9666:9666"
    volumes:
      - duplistatus_data:/app/data
    environment:
      - TZ=UTC
```

### Docker CLI {#docker-cli}
```bash
docker run -d \
  --name duplistatus \
  --restart unless-stopped \
  -p 9666:9666 \
  -v duplistatus_data:/app/data \
  wsjbr/duplistatus:0.7.x
```

## 升级说明 {#upgrade-instructions}

### 从 0.6.x 升级 {#from-version-06x}
1. **备份数据**（升级期间自动创建备份）
2. **更新 Docker 镜像**：`docker pull wsjbr/duplistatus:0.7.x`
3. **重启容器**：`docker-compose down && docker-compose up -d`
4. **监控日志**：`docker logs -f duplistatus`
5. **更新集成**：更新使用 API 的任何外部工具
6. **验证升级**：检查所有功能正常工作

### 从 0.5.x 或更早版本升级 {#from-version-05x-or-earlier}
1. **先升级到 0.6.x**（如可用）
2. **然后使用上述步骤升级到 0.7.x**
