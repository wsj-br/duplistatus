# 版本 1.2.1 {#version-121}

## 概览 {#overview}

版本 1.2.1 是一个次要版本，引入了对备份通知管理的重大增强，提高了审计日志能力，修复了边缘情况下的下次运行/逾期检测，并增加了数据库备份和恢复功能。此版本重点在于提升通知设置的用户体验。

## 新功能 {#new-features}

### 增强的备份通知管理 {#enhanced-backup-notifications-management}

- **服务器级默认值和继承系统**：在服务器级别配置默认的额外目标（通知事件、邮件和 NTFY 主题），所有备份将自动继承这些设置。备份可以在需要时覆盖这些默认值，并有清晰的视觉指示符显示继承状态。
- **改进的视觉层级**：备份现在按服务器分组，并具有明显的服务器标题行。继承的值用链接图标清晰标记，覆盖的值可以轻松恢复为继承状态。
- **批量操作**：“同步到全部”按钮可清除所有备份的覆盖设置，使所有备份继承自服务器默认值。“全部清除”按钮可清除所有额外目标，同时保持继承结构。
- **测试功能**：在备份通知设置页面直接添加了测试邮件和测试通知按钮。测试按钮同时适用于单个备份设置和批量编辑操作，允许您在应用更改前验证通知传递情况。
- **QR 码支持**：为 NTFY 主题添加了 QR 码按钮，以便轻松地与移动设备共享通知主题。

### 通知模板增强 {#notification-template-enhancements}

- **新 `{log_text}` 变量**：在通知模板中添加了 `{log_text}` 变量，可访问日志文本消息以用于通知消息。该变量包含警告和错误的完整合并文本，格式为每行一项的纯文本。
- **自动模板迁移**：系统包含一种延迟升级机制，在升级后首次读取旧的默认模板时，会自动检测并更新。自定义模板将保持不变。

### 数据库管理 {#database-management}

- **数据库备份功能**：管理员现在可以从数据库维护设置页面创建整个数据库的备份。支持二进制 (.db) 和 SQL 文本 (.sql) 格式，以提供灵活性。
- **数据库恢复功能**：使用之前创建的备份文件恢复数据库，具备自动创建安全备份、完整性验证以及失败时回滚的功能。为了安全起见，恢复后所有会话将被清除。

### 主密钥检测 {#master-key-detection}

- **自动主密钥更改检测**：登录后，系统会自动检查 `.duplistatus.key` 文件是否发生更改。如果检测到更改，所有加密密码将被清除，并弹出模态窗口通知用户需要重新配置密码。这可以防止从备份恢复或迁移到新系统时出现错误。

### 备份监控改进 {#backup-monitoring-improvements}

- **增强的 CSV 导出**：在备份监控设置中通过“下载 CSV”按钮生成的 CSV 文件中增加了“上次备份时间戳 (DB)”列。这允许用户将数据库中实际的上次备份时间戳与配置的“上次备份”时间戳进行比较。
- **下次运行工具提示**：将鼠标悬停在每行中的“下次运行”值上，现在会显示一个工具提示，显示来自数据库的上次备份时间戳，其格式与“下次运行”的显示方式相同。

### 审计日志增强 {#audit-logging-enhancements}

- **单个通知审计条目**：现在每次发送的通知和邮件都会生成一条独立的审计日志条目，包含通道、配置和结果（成功或失败）的详细信息。这包括测试通知，从而提供更好的通知投递可追溯性。
- **增强的失败日志记录**：所有通知失败的审计日志操作现在都包裹在 try-catch 块中，以防止审计日志错误中断通知流程。在记录邮件失败时，如果 SMTP 配置未立即可用，系统会尝试获取该配置。
- **改进的配置更新审计日志**：`email_config_updated` 和 `ntfy_config_updated` 的审计日志条目现在仅在配置发生实际更改时记录。当检测到更改时，审计日志会包含每个修改字段的旧值和新值，便于追踪具体变更内容。

### 文档改进 {#documentation-improvements}

- 文档样式已更新，以与应用程序的外观保持一致。侧边栏也经过修订，以提高可见性并使导航更便捷。

## 🐛 错误修复 {#-bug-fixes}

- **修复审计日志时间戳**：审计日志查看器中的时间戳现在会正确显示为用户浏览器所在时区，而非 GMT。SQLite 时间戳（以 UTC 存储）现在会正确解析为 UTC，并转换为浏览器本地时区进行显示。
- **修正“下次运行”计算和逾期检测**：“备份监控”设置中的“下次运行”列现在能准确计算并显示下次预期备份日期，综合考虑了备份计划和上次备份时间戳。这适用于计划运行和手动运行的备份任务，从而实现更可靠的逾期检测。

## 🚀 迁移注意事项 {#-migration-notes}

### 从版本 1.1.x 升级 {#from-version-11x}

升级到版本 1.2.1 时：

1. **通知模板**：系统在升级后首次访问时，会自动更新默认通知模板，以包含新的 `{log_text}` 变量。自定义模板将保持不变。如果您希望在自己的模板中使用新的 `{log_text}` 变量，则需要手动更新模板或恢复默认模板。

2. **主密钥变更**：如果您从备份中恢复或迁移到具有不同 `.duplistatus.key` 文件的新系统，系统将自动检测此情况，并提示您重新配置加密密码（SMTP 密码和 Duplicati 服务器密码）。

3. **数据库备份/恢复**：数据库维护设置页面中新增了数据库备份和恢复功能。建议在升级前创建备份，尽管本次发布无需数据库模式更改。

4. **无需数据库迁移**：本次发布不需要任何数据库模式更改。

## API 端点 {#api-endpoints}

本版本引入了以下 API 端点。

### 认证与安全端点 {#authentication--security-endpoints}

- `GET /api/auth/admin-must-change-password` - 检查管理员用户是否必须更改密码

### 数据库管理端点 {#database-management-endpoints}

- `GET /api/database/backup` - 创建数据库备份（二进制或 SQL 格式）
- `POST /api/database/restore` - 从备份文件恢复数据库

### 备份管理端点 {#backup-management-endpoints}

- `GET /api/backups/last-timestamps` - 获取所有服务器-备份组合的上次备份时间戳
