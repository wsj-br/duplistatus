# 版本 1.2.1 {#version-121}

## 概览 {#overview}

版本 1.2.1 是一次次要版本，对备份通知管理进行重大增强，改进审计日志功能，修复边缘情况下的 next run/逾期检测，并新增数据库备份和恢复功能。本版本专注于改进通知设置的用户体验。

## 新功能 {#new-features}

### 增强的备份通知管理 {#enhanced-backup-notifications-management}

- **服务器级默认值和继承系统**：在服务器级别配置默认附加目标（通知事件、电子邮件和 NTFY 主题），所有备份自动继承。备份可按需覆盖默认值，继承状态有清晰的视觉指示。
- **改进的视觉层次**：备份现按服务器分组，具有 distinct 服务器标题行。继承值以链接图标标记，覆盖值可轻松恢复为继承。
- **批量操作**：「Sync to All」按钮清除所有备份覆盖，使所有备份继承服务器默认值。「Clear All」按钮清除所有附加目标，同时保持继承结构。
- **测试功能**：在备份通知设置页面直接添加测试邮件和测试通知按钮。单个备份设置和批量编辑操作均可用测试按钮，在应用更改前验证通知投递。
- **QR 码支持**：为 NTFY 主题添加 QR 码按钮，便于与移动设备共享通知主题。

### 通知模板增强 {#notification-template-enhancements}

- **新 `{log_text}` 变量**：在通知模板中添加 `{log_text}` 变量，提供日志文本消息供通知内容使用。变量包含警告和错误的完整文本，格式为纯文本，每行一项。
- **自动模板迁移**：系统包含惰性升级机制，升级后首次读取时自动检测并更新旧默认模板。自定义模板保持不变。

### 数据库管理 {#database-management}

- **数据库备份功能**：管理员现可从数据库维护设置页面创建整个数据库的备份。支持二进制（.db）和 SQL 文本（.sql）格式。
- **数据库恢复功能**：从先前创建的备份文件恢复数据库，自动创建安全备份、完整性验证和失败时回滚。恢复后所有会话清除以确保安全。

### 主密钥检测 {#master-key-detection}

- **自动主密钥变更检测**：登录后，系统自动检查 `.duplistatus.key` 文件是否已更改。若检测到，所有加密密码将被清除，模态弹窗通知用户需要重新配置密码。这可防止从备份恢复或迁移到新系统时出现错误。

### 备份监控改进 {#backup-monitoring-improvements}

- **增强的 CSV 导出**：备份监控设置中「Download CSV」按钮生成的 CSV 文件新增「Last Backup Timestamp (DB)」列。用户可比较数据库中的实际最后备份时间戳与配置的「Last Backup」时间戳。
- **Next Run 工具提示**：悬停在每行的「Next Run」值上现显示工具提示，展示数据库中的最后备份时间戳，格式与「Next Run」显示相同。

### 审计日志增强 {#audit-logging-enhancements}

- **单独的通知审计条目**：每条发送的通知和邮件现生成单独的审计日志条目，包含渠道、配置和结果（成功或失败）详情。包括测试通知，提供更好的通知投递可追溯性。
- **健壮的失败日志**：所有通知失败的审计日志操作现包装在 try-catch 块中，防止审计日志错误中断通知流程。记录邮件失败时，若 SMTP 配置未立即可用，系统会尝试检索。
- **改进的配置更新审计日志**：`email_config_updated` 和 `ntfy_config_updated` 审计日志条目现仅在配置实际变更时记录。检测到变更时，审计日志包含每个修改字段的旧值和新值，便于跟踪具体变更。

### 文档改进 {#documentation-improvements}

- 文档样式已更新以与应用程序外观一致。侧边栏也已修订以提高可见性并简化导航。

## 🐛 错误修复 {#-bug-fixes}

- **修复审计日志时间戳**：审计日志查看器中的时间戳现正确显示为用户浏览器时区而非 GMT。SQLite 时间戳（以 UTC 存储）现正确解析为 UTC 并转换为浏览器本地时区显示。
- **修正「Next Run」计算和逾期检测**：备份监控设置中的「Next Run」列现准确计算并显示下次预期备份日期，同时考虑备份计划和最后备份时间戳。适用于计划和手动运行的备份作业，逾期检测更可靠。

## 🚀 迁移说明 {#-migration-notes}

### 从 1.1.x 升级 {#from-version-11x}

升级到 1.2.1 时：

1. **通知模板**：系统将在升级后首次访问时自动更新默认通知模板以包含新 `{log_text}` 变量。自定义模板保持不变。若要在自己的模板中使用新 `{log_text}` 变量，需手动更新或恢复默认模板。

2. **主密钥变更**：若从备份恢复或迁移到具有不同 `.duplistatus.key` 文件的新系统，系统将自动检测并提示重新配置加密密码（SMTP 密码和 Duplicati 服务器密码）。

3. **数据库备份/恢复**：数据库维护设置页面提供新的数据库备份和恢复功能。建议升级前创建备份，但本版本不需要数据库架构变更。

4. **无需数据库迁移**：本版本不需要任何数据库架构变更。

## API 端点 {#api-endpoints}

本版本引入以下 API 端点。

### 身份验证与安全端点 {#authentication--security-endpoints}

- `GET /api/auth/admin-must-change-password` - 检查管理员是否必须更改密码

### 数据库管理端点 {#database-management-endpoints}

- `GET /api/database/backup` - 创建数据库备份（二进制或 SQL 格式）
- `POST /api/database/restore` - 从备份文件恢复数据库

### 备份管理端点 {#backup-management-endpoints}

- `GET /api/backups/last-timestamps` - 获取所有服务器-备份组合的最后备份时间戳
