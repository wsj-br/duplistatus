# 版本 1.4.2 {#version-142}

## 概览 {#overview}

版本 1.4.2 是一个专注于稳定性和可靠性的维护版本。减少了仪表板加载时的冗余身份验证请求，改进了“合并重复服务器”工具以合并重复的 Duplicati 备份 ID，并移除了仪表板中的重复行。此外，它还将服务器合并记录在审计日志中，并将 Next.js 固定在稳定版本。

---

## Bug 修复 {#bug-fixes}

- **仪表板上的重复条目**：当多个记录共享同一个最近备份日期时，仪表板不再为同一个备份名称显示多行。
- **合并重复服务器**：合并服务器现在还会合并在合并后服务器上共享相同备份名称的重复 Duplicati 备份 ID，并保留最近备份的 ID 以保持历史记录的一致性。

---

## 改进 {#improvements}

### 性能 {#performance}

- **减少加载时的身份验证请求**：现在仅获取一次已登录用户并在页面（页眉、密码保护、主题/配置提供程序和仪表板组件）中共享，而不是由每个组件单独请求，从而减少了仪表板打开时的冗余网络调用。

### 审计 {#auditing}

- **审计日志记录服务器合并**：合并服务器现在将作为 `servers_merged` 条目记录在审计日志中，以提高可追溯性。

### 界面 {#interface}

- **合并重复服务器布局**：在 设置 > 数据库维护 > 合并重复服务器 中，“目标服务器（最新）”和“旧服务器 ID”标签现在渲染为正式的表头单元格（支持主题），而非药丸/按钮形状，使外观更清晰、更一致。

---

## 已变更 {#changed}

- **Next.js 稳定版本**：Next.js 现在固定在稳定 `16.2.9` 版本（此前为 canary 构建），以提高可靠性。

---

## 迁移说明 {#migration-notes}

### 从版本 1.4.1 升级 {#from-version-141}

升级到版本 1.4.2 时：

1. **无需数据库迁移**：此版本不需要任何数据库架构更改。
2. **Docker 用户**：请拉取最新镜像；启动崩溃修复已包含在更新的容器构建中。

## 更新日志 {#changelog}

### 版本 1.4.2 变更 {#version-142-changes}

- **修复**：修复了由于缺少 `@swc/helpers` ESM 辅助文件导致的 Docker 独立容器启动崩溃问题。
- **修复**：修复了多个记录共享相同最新备份日期时，仪表板出现重复行的问题。
- **修复**：服务器合并现在会合并共享相同备份名称的重复 Duplicati 备份 ID。
- **变更**：已登录用户通过共享提供程序获取一次，减少了加载时的冗余 `/api/auth/me` 请求。
- **变更**：服务器合并操作现在记录在审计日志中为 `servers_merged`。
- **变更**：合并重复服务器的标签渲染为表头单元格，而非药丸/按钮形状。
- **变更**：Next.js 固定在稳定 `16.2.9` 版本。
