
# 版本 1.0.x {#version-10x}

## 概览 {#overview}

版本 1.0.x 是一次功能版本，对电子邮件配置、服务器管理功能和数据库维护工具进行重大增强。本版本专注于提高电子邮件通知可靠性、扩展 SMTP 连接选项，并提供更好的服务器和数据库完整性管理工具。

## 新功能 {#new-features}

### 服务器管理增强 {#server-management-enhancements}

- **服务器密码更改功能** — 用户现可直接在服务器详情页点击钥匙图标更改服务器密码
- **重复服务器检测与合并** — 数据库维护表单中的新功能，检测重复服务器并合并为单一服务器条目

### 审计日志增强 {#audit-logging-enhancements}

- **备份上传审计日志** — 通过 `/api/upload` 端点的所有备份日志上传现自动记录在审计日志中。每条记录包含备份详情（服务器 ID、服务器名称、备份名称、状态、日期、持续时间、文件计数、大小、警告和错误）以及客户端 IP 地址和用户代理信息，用于跟踪和安全

### 电子邮件配置增强 {#email-configuration-enhancements}

- **配置不完整提示** — 电子邮件设置未正确配置时显示视觉告警
- **服务状态指示器** — 电子邮件服务未配置时，通知复选框显示「(disabled)」标签
- **可自定义发件人信息**：
  - `Sender Name` 字段，自定义电子邮件通知中显示的发件人名称
  - `From Address` 字段，指定显示为发件人的电子邮件地址
  - 注意：部分电子邮件提供商会用 `SMTP Server Username` 覆盖 `From Address`
- **SMTP 身份验证开关** — 启用或禁用 SMTP 身份验证，支持不需要身份验证的服务器
- **电子邮件格式验证** — 验证收件人和发件人地址字段，确保格式正确
- **增强的测试邮件** — 测试邮件现显示完整连接详情，包括 SMTP 服务器主机名、端口、连接类型、身份验证状态、用户名（如适用）、收件人电子邮件、发件人地址、发件人名称和测试时间戳

### SMTP 连接类型支持 {#smtp-connection-type-support}

- **多种连接类型** — 支持三种连接类型：
  - `Plain SMTP`（无加密），端口 25
  - `STARTTLS`（升级到 TLS），端口 587
  - `Direct SSL/TLS`（从一开始就加密），端口 465
- **默认连接类型** — 新配置默认改为 `Direct SSL/TLS`
- **连接特定要求** — `Plain SMTP` 连接需要 `From Address` 字段，以确保正确的发件人识别和 RFC 5322 合规

## 🐛 错误修复 {#-bug-fixes}

### 电子邮件系统修复 {#email-system-fixes}

- **修复缺失 `From` 头** — 通过确保所有邮件包含正确的 `From` 头解决 RFC 5322 合规错误
- **修复 STARTTLS 电子邮件加密** — 修正 `STARTTLS` 连接（端口 587）的加密处理
- **修复 Plain SMTP 连接** — 解决导致 `Plain SMTP` 连接失败的 TLS 协商错误
- **修复未保存配置丢失** — 更改密码时防止未保存的电子邮件配置被清除
- **修复过时的 SMTP 配置** — 解决测试邮件端点使用过时 SMTP 配置的问题

### 用户界面修复 {#user-interface-fixes}

- **修复密码可见性切换** — 修正密码更改表单中查看密码功能的行为；点击眼睛图标现同时显示两个密码字段

### 系统修复 {#system-fixes}

- **修复 Docker entrypoint 日志** — 解决 Docker entrypoint 脚本丢弃日志消息的问题
- **修复 Next Run 日期计算** — 采集备份日志后自动重新计算 `Next Run` 日期，确保日志采集后日期正确更新

## 🚀 迁移说明 {#-migration-notes}

### 从 0.9.x 升级 {#from-version-09x}

本版本包含电子邮件配置和服务器管理的增强。从 0.9.x 升级时：

1. **无需数据库迁移** — 本版本不需要任何数据库架构变更
2. **电子邮件配置更新** — 现有电子邮件配置将继续工作，但您可能希望审查并更新设置以利用新功能：
   - 考虑设置自定义 `Sender Name` 以便更好地识别邮件
   - 验证 `From Address` 设置正确，尤其在使用 `Plain SMTP` 连接时
   - 使用增强的测试邮件功能测试电子邮件配置
3. **连接类型默认值** — 新电子邮件配置默认使用 `Direct SSL/TLS` 连接类型
4. **服务器密码管理** — 现可直接在服务器详情页更改服务器密码，无需重新采集备份

### 电子邮件配置建议 {#email-configuration-recommendations}

- **生产环境**：使用 `STARTTLS` 或 `Direct SSL/TLS` 连接以确保安全邮件投递
- **本地测试**：`Plain SMTP` 连接可用于受信任的本地网络
- **From Address 要求**：使用 `Plain SMTP` 或禁用身份验证时，确保正确配置 `From Address` 字段
- **测试配置**：始终使用「Send Test Email」按钮验证电子邮件设置

## API 端点 {#api-endpoints}

### 电子邮件配置端点 {#email-configuration-endpoints}

- `GET /api/email/config` - 获取电子邮件配置
- `POST /api/email/config` - 更新电子邮件配置
- `POST /api/email/test` - 发送测试邮件
- `DELETE /api/email/config` - 删除电子邮件配置

### 服务器管理端点 {#server-management-endpoints}

- `PATCH /api/servers/{id}/password` - 更新服务器密码
- `POST /api/database/maintenance/merge-duplicate-servers` - 合并重复服务器
