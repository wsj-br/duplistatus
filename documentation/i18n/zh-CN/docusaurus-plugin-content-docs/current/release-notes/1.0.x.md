# 版本 1.0.x {#version-10x}

## 概览 {#overview}

版本 1.0.x 是一个功能更新版本，显著增强了邮件配置、服务器管理功能以及数据库维护工具。此版本专注于提高邮件通知的可靠性，扩展 SMTP 连接选项，并提供更完善的服务器和数据库完整性管理工具。

## 新功能 {#new-features}

### 服务器管理增强 {#server-management-enhancements}

- **服务器密码更改功能** - 用户现在可以通过点击服务器详情页面上的钥匙图标，直接更改服务器密码
- **重复服务器检测与合并** - 数据库维护表单中新增功能，用于检测重复服务器并将它们合并为单个服务器条目

### 审计日志增强 {#audit-logging-enhancements}

- **备份上传审计日志** - 通过 `/api/upload` 端点上传的所有备份日志现在都会自动记录到审计日志中。每条记录包含备份详情（服务器 ID、服务器名称、备份名称、状态、日期、持续时间、文件数量、大小、警告和错误），以及客户端 IP 地址和 User Agent 信息，用于追踪和安全目的

### 邮件配置增强 {#email-configuration-enhancements}

- **配置不完整提示** - 当邮件设置未正确配置时，显示视觉提醒
- **服务状态指示器** - 当邮件服务未配置时，在通知复选框旁显示“(已禁用)”标签
- **可自定义的发件人信息**:
  - `Sender Name` 字段用于自定义邮件通知中显示的发件人名称
  - `From Address` 字段用于指定邮件通知中显示的发件人邮箱地址
  - 注意：某些邮件服务提供商会用 `SMTP Server Username` 覆盖 `From Address`
- **SMTP 认证切换** - 可选择启用或禁用 SMTP 认证，支持无需认证的服务器
- **邮件格式验证** - 对收件人和发件人地址字段进行验证，确保符合正确的邮件地址格式
- **增强的测试邮件** - 测试邮件现在显示完整的连接详情，包括 SMTP 服务器主机名、端口、连接类型、认证状态、用户名（如适用）、收件人电子邮件、发件地址、发件人名称以及测试时间戳

### SMTP 连接类型支持 {#smtp-connection-type-support}

- **多种连接类型** - 支持三种连接类型：
  - `Plain SMTP`（无加密）使用端口 25
  - `STARTTLS`（升级至 TLS）使用端口 587
  - `Direct SSL/TLS`（从开始即加密）使用端口 465
- **默认连接类型** - 新配置的默认连接类型更改为 `Direct SSL/TLS`
- **连接特定要求** - `Plain SMTP` 连接需要填写 `From Address` 字段，以确保正确的邮件发件人标识并符合 RFC 5322 规范

## 🐛 错误修复 {#-bug-fixes}

### 邮件系统修复 {#email-system-fixes}

- **修复缺失的 `From` 头部** - 通过确保所有邮件消息包含正确的 `From` 头部，解决了 RFC 5322 合规性错误
- **修复 STARTTLS 邮件加密** - 修正了 `STARTTLS` 连接（端口 587）的加密处理
- **修复纯 SMTP 连接** - 解决了导致 `Plain SMTP` 连接失败的 TLS 协商错误
- **修复未保存配置丢失问题** - 防止在更改密码时清除未保存的邮件配置
- **修复陈旧的 SMTP 配置** - 解决了测试邮件端点使用过时 SMTP 配置的问题

### 用户界面修复 {#user-interface-fixes}

- **修复密码可见性切换功能** - 修正了密码更改表单中“查看密码”功能的行为；点击眼睛图标后，两个密码字段的密码将同时显示

### 系统修复 {#system-fixes}

- **修复 Docker 入口脚本日志记录** - 解决了 Docker 入口脚本丢失日志消息的问题
- **修复下次运行日期计算** - 在收集备份日志后，自动重新计算 `Next Run` 日期，确保日志收集后日期正确更新

## 🚀 迁移注意事项 {#-migration-notes}

### 从版本 0.9.x 升级 {#from-version-09x}

此版本包含对邮件配置和服务器管理的增强。从 0.9.x 版本升级时：

1. **无需数据库迁移** - 此版本不需要任何数据库模式更改
2. **邮件配置更新** - 现有的邮件配置将继续正常工作，但建议您查看并更新设置以利用新功能：
   - 考虑设置自定义的 `Sender Name` 以更好地识别邮件
   - 确认您的 `From Address` 设置正确，尤其是在使用 `Plain SMTP` 连接时
   - 使用增强的测试邮件功能测试您的邮件配置
3. **连接类型默认值** - 新的邮件配置将默认使用 `Direct SSL/TLS` 连接类型
4. **服务器密码管理** - 现在您可以直接从服务器详情页面更改服务器密码，而无需重新采集备份

### 邮件配置建议 {#email-configuration-recommendations}

- **生产环境使用**：使用 `STARTTLS` 或 `Direct SSL/TLS` 连接以实现安全的邮件投递
- **本地测试使用**：`Plain SMTP` 连接可用于受信任的本地网络
- **发件地址要求**：当使用 `Plain SMTP` 或禁用身份验证时，请确保正确配置 `From Address` 字段
- **测试您的配置**：始终使用“发送测试邮件”按钮验证您的邮件设置是否正常工作

## API 端点 {#api-endpoints}

### 邮件配置端点 {#email-configuration-endpoints}

- `GET /api/email/config` - 获取邮件配置
- `POST /api/email/config` - 更新邮件配置
- `POST /api/email/test` - 发送测试邮件
- `DELETE /api/email/config` - 删除邮件配置

### 服务器管理端点 {#server-management-endpoints}

- `PATCH /api/servers/{id}/password` - 更新服务器密码
- `POST /api/database/maintenance/merge-duplicate-servers` - 合并重复服务器
