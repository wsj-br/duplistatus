
# 版本 0.9.x {#version-09x}

## 概览 {#overview}

版本 0.9.x 是一次重大版本，引入全面的**用户访问控制与安全系统**。本版本将 **duplistatus** 从开放访问应用程序转变为安全的多用户系统，具有基于角色的访问控制、全面审计日志和企业级安全功能。所有现有功能现均受身份验证保护，确保仅授权用户可访问和管理备份监控。

## 破坏性变更 {#breaking-changes}

### 需要身份验证 {#authentication-required}

- **所有页面现需身份验证** — 用户必须登录才能访问应用程序
- **自动创建默认管理员账户**：
  - 用户名：`admin`
  - 密码：`Duplistatus09`（首次登录时必须更改）
- **现有会话失效** — 升级后所有用户必须重新登录
- **API 端点现需身份验证** — 外部集成必须包含会话 cookie 或使用已认证请求

:::warning
**升级到 0.9.x 后所有用户必须登录。** 默认管理员凭据为 admin/Duplistatus09。首次登录后请立即更改密码。
:::

### 数据库架构迁移 {#database-schema-migration}

- **现有安装自动从 v3.1 迁移到 v4.0**
- **新增表**：`users`、`sessions`、`audit_log`
- **迁移前自动创建数据库备份**
- **新安装**直接使用架构 v4.0（无需迁移）


## 新功能 {#new-features}

### 用户访问控制与安全系统 {#user-access-control--security-system}

#### 身份验证系统 {#authentication-system}

- **用户登录/登出功能**，具有安全会话管理
- **基于密码的身份验证**，使用 bcrypt 哈希（成本因子 12）
- **账户锁定机制**（5 次失败尝试，锁定 15 分钟）
- **首次登录强制更改密码**
- **密码策略强制**：
  - 最少 8 个字符
  - 至少一个大写字母
  - 至少一个小写字母
  - 至少一个数字
  - 特殊字符可选
- **「记住我」功能**，用户名持久化在 localStorage
- **基于会话的身份验证**，数据库支持的会话（替代内存存储）
- **CSRF 保护**与身份验证集成
- **HTTP-only cookie**用于会话安全
- **IP 地址和用户代理跟踪**用于安全监控

#### 用户管理（仅管理员） {#user-management-admin-only}

- **用户创建**，自动生成临时密码
- **用户列表**，支持搜索、分页和筛选
- **用户编辑**（用户名、管理员状态、密码更改要求）
- **密码重置功能**，显示临时密码
- **用户删除**，带保护（防止删除最后一个管理员）
- **基于角色的访问控制**（管理员/用户角色）
- **状态指示器**（活跃、锁定、必须更改密码）
- **最后登录跟踪**和显示

#### 审计日志系统 {#audit-logging-system}

- **全面审计跟踪**，记录所有系统变更和用户操作
- **审计日志查看器**，高级筛选：
  - 日期范围筛选
  - 用户筛选
  - 操作筛选
  - 类别筛选
  - 状态筛选
- **导出功能**（CSV 和 JSON 格式）
- **审计日志统计**和分析
- **可配置保留期**（30–365 天，默认：90 天）
- **自动清理 cron 作业**（每天 UTC 2:00 运行）
- **手动清理 API**，支持 dry-run
- **审计日志记录**：
  - 身份验证事件（登录、登出、密码更改、账户锁定）
  - 用户管理操作（创建、更新、删除、密码重置）
  - 配置变更（电子邮件、NTFY、模板、逾期待忍、备份设置）
  - 备份操作（采集、删除、清理）
  - 服务器管理（添加、更新、删除）
  - 系统操作（迁移、清理、通知）

### 设置页面重新设计 {#settings-page-redesign}

- **现代侧边栏导航**，可折叠侧边栏
- **用户特定侧边栏状态持久化**（每用户保存折叠/展开偏好）
- **分组设置部分**：
  - **Notifications**：Backup Notifications、Backup monitoring、Templates
  - **Integrations**：NTFY、Email
  - **System**：Servers、Users（仅管理员）、Audit Log
- **粘性侧边栏**，滚动时保持可见
- **响应式设计**，优化间距
- **设置图标和「System Settings」标题**在侧边栏标题
- **返回按钮**集成到应用程序标题栏

### 用户界面增强 {#user-interface-enhancements}

- **独立登录页面**，现代设计
- **登录表单「记住我」复选框**，用户名持久化
- **登录和密码更改表单的显示/隐藏密码按钮**
- **更改密码模态框**，实时验证清单
- **应用程序标题栏中的用户指示器和登出按钮**
- **基于角色的 UI 可见性**（管理员专用功能对普通用户隐藏）
- **状态徽章和指示器**贯穿 UI
- **用户特定偏好**存储在 localStorage（例如侧边栏折叠状态）

### 安全功能 {#security-features}

- **密码哈希**使用 bcrypt（行业标准）
- **审计日志中敏感数据脱敏**（密码、令牌、密钥永不记录）
- **登录尝试速率限制**
- **会话过期**（24 小时）
- **CSRF 令牌验证**用于所有状态变更操作
- **管理员恢复 CLI 工具**用于锁定时的密码重置

### 开发者工具 {#developer-tools}

- **管理员恢复 CLI 脚本**（`admin-recovery`），便于在 Docker 容器中执行
- **本地和 Docker 中均可工作**，自动路径检测
- **全面的 TypeScript 接口**（无 `any` 类型）
- **身份验证中间件**用于路由保护
- **AuditLogger 工具类**，便捷方法

## 🚀 迁移说明 {#-migration-notes}

### 从 0.8.x 升级 {#from-version-08x}

本版本引入身份验证，要求所有用户登录。从 0.8.x 升级时：

1. **自动数据库迁移**：应用程序将自动将数据库架构从 v3.1 迁移到 v4.0
2. **数据库备份**：迁移前自动创建备份
3. **默认管理员账户**：创建默认管理员账户：
   - 用户名：`admin`
   - 密码：`Duplistatus09`
   - **首次登录必须更改此密码**
4. **会话失效**：所有现有会话失效
5. **需要用户登录**：升级后所有用户必须登录
6. **API 身份验证**：外部集成必须更新以包含身份验证

### 安全注意事项 {#security-considerations}

- **更改默认密码**：首次登录后立即更改默认管理员密码
- **创建用户账户**：为每个需要访问的人员创建独立账户
- **审查审计日志**：定期检查审计日志以进行安全监控
- **配置保留**：根据合规要求设置适当的审计日志保留期
- **备份主密钥**：若从 0.8.x 升级，确保 `.duplistatus.key` 文件已备份

### 首次登录步骤 {#first-login-steps}

1. **登录**默认凭据（`admin` / `Duplistatus09`）
2. **更改密码**（首次登录时提示，强制）
3. **创建用户账户**供其他用户使用（Settings → Users）
4. **配置审计日志保留**（如需要）（Settings → Audit Log）
5. **审查审计日志**以验证系统正常工作

## API 端点 {#api-endpoints}

### 身份验证端点 {#authentication-endpoints}

- `POST /api/auth/login` - 用户身份验证
- `POST /api/auth/logout` - 会话终止
- `GET /api/auth/me` - 当前用户信息
- `POST /api/auth/change-password` - 密码更改

### 用户管理端点（仅管理员） {#user-management-endpoints-admin-only}

- `GET /api/users` - 列出用户
- `POST /api/users` - 创建用户
- `PATCH /api/users/{id}` - 更新用户
- `DELETE /api/users/{id}` - 删除用户

### 审计日志端点 {#audit-log-endpoints}

- `GET /api/audit-log` - 查询审计日志
- `GET /api/audit-log/download` - 导出审计日志
- `GET /api/audit-log/stats` - 审计统计
- `POST /api/audit-log/cleanup` - 手动清理（仅管理员）
- `GET /api/audit-log/retention` - 获取保留设置
- `PATCH /api/audit-log/retention` - 更新保留（仅管理员）

:::note
所有 API 端点现需身份验证。身份验证详情见 [API 参考](../api-reference/authentication-security.md)。
:::
