# 版本 0.9.x {#version-09x}

## 概览 {#overview}

版本 0.9.x 是一个重大版本，引入了全面的 **用户访问控制与安全系统**。此版本将 **duplistatus** 从一个开放访问的应用程序转变为一个具有基于角色的访问控制、全面审计日志和企业级安全功能的安全多用户系统。所有现有功能现在都受到身份验证的保护，确保只有授权用户才能访问和管理备份监控。

## 重大变更 {#breaking-changes}

### 需要身份验证 {#authentication-required}

- **所有页面现在都需要身份验证** - 用户必须登录才能访问应用程序
- **默认管理员账户** 将自动创建：
  - 用户名：`admin`
  - 密码：`Duplistatus09`（必须在首次登录时更改）
- **现有会话将失效** - 所有用户在升级后必须重新登录
- **API 端点现在需要身份验证** - 外部集成必须包含会话 Cookie 或使用经过身份验证的请求

:::warning
**所有用户在升级到版本 0.9.x 后必须登录。** 默认管理员凭据为 admin/Duplistatus09。请在首次登录后立即更改密码。
:::

### 数据库架构迁移 {#database-schema-migration}

- **现有安装从 v3.1 到 v4.0 的自动迁移**
- **新增表**：`users`, `sessions`, `audit_log`
- **迁移前自动创建数据库备份**
- **新安装** 直接从架构 v4.0 开始（无需迁移）

## 新功能 {#new-features}

### 用户访问控制与安全系统 {#user-access-control--security-system}

#### 身份验证系统 {#authentication-system}

- **用户登录/退出登录功能**，具有安全的会话管理
- **基于密码的身份验证**，采用 bcrypt 哈希（成本因子 12）
- **账户锁定机制**（5 次失败尝试，锁定 15 分钟）
- **首次登录时强制更改密码**
- **密码策略强制执行**：
  - 至少 8 个字符
  - 至少一个大写字母
  - 至少一个小写字母
  - 至少一个数字
  - 特殊字符可选
- **“记住我”功能**，用户名持久化存储在 localStorage 中
- **基于会话的身份验证**，采用数据库支持的会话（取代内存存储）
- **CSRF 保护** 与身份验证集成
- **HTTP-only Cookie** 用于会话安全
- **IP 地址和 User Agent 跟踪** 用于安全监控

#### 用户管理（仅限管理员） {#user-management-admin-only}

- **用户创建**，具有自动临时密码生成功能
- **用户列表**，支持搜索、分页和过滤
- **用户编辑**（用户名、管理员状态、密码更改要求）
- **密码重置功能**，可显示临时密码
- **用户删除** 及其保护措施（防止删除最后一个管理员）
- **基于角色的访问控制**（管理员/用户角色）
- **状态指示器**（激活、已锁定、必须更改密码）
- **上次登录跟踪**与显示

#### 审计日志系统 {#audit-logging-system}

- **针对所有系统更改和用户操作的全面审计轨迹**
- **审计日志查看器**，具有高级过滤功能：
  - 日期范围过滤
  - 用户过滤
  - 操作过滤
  - 类别过滤
  - 状态过滤
- **导出功能**（CSV 和 JSON 格式）
- **审计日志统计**和分析
- **可配置的保留期限**（30-365 天，默认：90 天）
- **自动清理 cron 任务**（每天 UTC 时间凌晨 2 点运行）
- **手动清理 API**，支持干跑（dry-run）模式
- **以下内容的审计日志记录**：
  - 身份验证事件（登录、退出登录、更改密码、账户锁定）
  - 用户管理操作（创建、更新、删除、密码重置）
  - 配置更改（邮件、NTFY、模板、逾期容差、备份设置）
  - 备份操作（收集、删除、清理）
  - 服务器管理（添加、更新、删除）
  - 系统操作（迁移、清理、通知）

### 设置页重新设计 {#settings-page-redesign}

- **现代化的侧边栏导航**，支持侧边栏折叠
- **用户特定的侧边栏状态持久化**（按用户保存折叠/展开偏好）
- **分组设置部分**：
  - **通知**：备份通知、备份监控、模板
  - **集成**：NTFY、邮件
  - **系统**：服务器、用户（仅限管理员）、审计日志
- **粘性侧边栏**，在滚动时保持可见
- **响应式设计**，优化了间距
- **侧边栏页眉中的设置图标和“系统设置”标题**
- **集成到应用页眉中的返回按钮**

### 用户界面增强 {#user-interface-enhancements}

- **独立登录页面**，采用现代设计
- **登录表单中的“记住我”复选框**，用于用户名持久化
- **登录和更改密码表单中的显示/隐藏密码按钮**
- **更改密码模态框**，带有实时验证清单
- **应用页眉中的用户指示器和退出登录按钮**
- **基于角色的 UI 可见性**（普通用户无法看到仅限管理员的功能）
- **整个 UI 中的状态徽章和指示器**
- **存储在 localStorage 中的用户特定偏好**（例如，侧边栏折叠状态）

### 安全特性 {#security-features}

- **使用 bcrypt 进行密码哈希**（行业标准）
- **审计日志中的敏感数据脱敏**（绝不记录密码、令牌、密钥）
- **登录尝试的速率限制**
- **会话过期**（24 小时）
- **所有状态变更操作的 CSRF 令牌验证**
- **管理员恢复 CLI 工具**，用于在被锁定时重置密码

### 开发人员工具 {#developer-tools}

- **管理员恢复 CLI 脚本** (`admin-recovery`)，方便在 Docker 容器中执行
- **同时支持本地和 Docker 运行**，具有自动路径检测功能
- **全面的 TypeScript 接口**（无 `any` 类型）
- **用于路由保护的身份验证中间件**
- **带有便捷方法的审计日志记录实用类**

## 🚀 迁移注意事项 {#-migration-notes}

### 从版本 0.8.x 升级 {#from-version-08x}

此版本引入了身份验证，并要求所有用户登录。从版本 0.8.x 升级时：

1. **自动数据库迁移**：应用程序将自动将您的数据库架构从 v3.1 迁移到 v4.0
2. **数据库备份**：在迁移前会自动创建备份
3. **默认管理员账户**：将创建一个默认管理员账户：
   - 用户名：`admin`
   - 密码：`Duplistatus09`
   - **您必须在首次登录时更改此密码**
4. **会话失效**：所有现有会话均失效
5. **需要用户登录**：所有用户在升级后必须登录
6. **API 身份验证**：必须更新外部集成以包含身份验证

### 安全考量 {#security-considerations}

- **更改默认密码**：在首次登录后立即更改管理员默认密码
- **创建用户账户**：为每个需要访问的人创建单独的用户账户
- **审查审计日志**：定期检查审计日志以进行安全监控
- **配置保留期**：根据合规要求设置适当的审计日志保留期
- **备份主密钥**：如果从0.8.x版本升级，请确保备份 `.duplistatus.key` 文件

### 首次登录步骤 {#first-login-steps}

1. **登录**使用默认凭据（`admin` / `Duplistatus09`）
2. **更改密码**（在提示时更改，首次登录强制更改）
3. **创建用户账户**给其他用户（设置 → 用户）
4. **配置审计日志保留期**（如果需要）（设置 → 审计日志）
5. **审查审计日志**以验证系统正常工作

## API 端点 {#api-endpoints}

### 身份验证端点 {#authentication-endpoints}

- `POST /api/auth/login` - 用户身份验证
- `POST /api/auth/logout` - 会话终止
- `GET /api/auth/me` - 当前用户信息
- `POST /api/auth/change-password` - 密码更改

### 用户管理端点（仅管理员） {#user-management-endpoints-admin-only}

- `GET /api/users` - 列出用户
- `POST /api/users` - 创建用户
- `PATCH /api/users/{id}` - 更新用户
- `DELETE /api/users/{id}` - 删除用户

### 审计日志端点 {#audit-log-endpoints}

- `GET /api/audit-log` - 查询审计日志
- `GET /api/audit-log/download` - 导出审计日志
- `GET /api/audit-log/stats` - 审计统计
- `POST /api/audit-log/cleanup` - 手动清理（仅管理员）
- `GET /api/audit-log/retention` - 获取保留设置
- `PATCH /api/audit-log/retention` - 更新保留期（仅管理员）

:::note
所有 API 端点现在都需要身份验证。有关身份验证的详细信息，请参见 [API 参考](../api-reference/authentication-security.md)。
:::
