# 版本 1.3.2 {#version-132}

## 概览 {#overview}

版本 1.3.2 是一次维护版本，解决依赖中的安全漏洞，修复 pnpm install 警告（peer webpack 和已弃用子依赖），解决应用程序和文档中的 lint 和构建警告，并更新文档以反映当前 Docker 脚本名称和包版本。

## 安全 {#security}

### 依赖漏洞修复 {#dependency-vulnerability-fixes}

- **Dependabot 依赖更新（文档）**：合并两个 Dependabot PR 以解决文档工作区中的漏洞：
  - `/documentation` 中的 **ajv**：从 8.17.1 更新到 8.18.0，从 6.12.6 更新到 6.14.0。
  - `/documentation` 中的 **fast-xml-parser**：从 5.3.5 更新到 5.3.8。
  两者均通过 /documentation 的 dependabot 多更新合并。

- **高严重性漏洞的 pnpm overrides**：添加 pnpm overrides 修复四个高严重性问题：
  - **minimatch** (>=3.1.4)：修复 documentation > @docusaurus/core > serve-handler > minimatch 中的 GHSA-3ppc-4f35-3m26（重复通配符 ReDoS）、GHSA-7r86-cg39-jmmj（组合回溯）和 GHSA-23c5-xmqv-rm74（嵌套 extglobs 回溯）。
  - **serialize-javascript** (>=7.0.3)：修复 webpack > terser-webpack-plugin > serialize-javascript 中的 GHSA-5c6j-r48x-rmvq（通过 RegExp.flags 和 Date.prototype.toISOString() 的 RCE）。

---

## 错误修复 {#bug-fixes}

- **pnpm install 警告**：解决三个安装时警告：(1) `next-intlayer` > `@intlayer/webpack` 的未满足 peer `webpack@5.104.1`（项目有 webpack 5.105.3），通过在根 `package.json` 中添加 `pnpm.peerDependencyRules.allowedVersions.webpack: "5"` 解决。(2) 已弃用子依赖 `prebuild-install@7.1.3`（来自 better-sqlite3）和 `whatwg-encoding@3.1.1`（来自 documentation > docusaurus-search-local > cheerio），通过为两者添加 `pnpm.allowedDeprecatedVersions` 使安装无警告运行。
- **应用程序日志查看器（exhaustive-deps）**：解决自动滚动效果的 react-hooks/exhaustive-deps 警告。`logData` 有意从依赖数组中省略，使滚动仅在自动滚动或所选文件变更时发生，而非每次轮询；新行滚动在 `loadLogs` 中处理。添加了带简短注释的 eslint-disable-next-line。
- **构建警告 vscode-languageserver-types（文档）**：通过 `documentation/docusaurus.config.ts` 中的内联插件添加 `ignoreWarnings`，抑制文档（Docusaurus）构建中传递依赖 `vscode-languageserver-types`（来自 intlayer-editor）的 Webpack「Critical dependency: require function is used in a way in which dependencies cannot be statically extracted」警告。警告仅在 docs 构建期间出现，不在主应用程序中。

## 文档 {#documentation}

- **Docker 脚本名称**：文档现使用当前 npm/pnpm 脚本名称：`docker-up` → `docker:up`、`docker-down` → `docker:down`、`docker-clean` → `docker:clean`、`docker-devel` → `docker:devel`。已在 AGENTS.md 和文档（setup.md、devel.md、release-management.md）中更新。

## 迁移说明 {#migration-notes}

### 从 1.3.1 升级 {#from-version-131}

升级到 1.3.2 时：

1. **无需数据库迁移**：本版本不需要任何数据库架构变更。
2. **Docker 命令**：若使用文档化的 pnpm 脚本，请使用 `pnpm docker:up`、`pnpm docker:down`、`pnpm docker:clean` 和 `pnpm docker:devel`，而非先前的 `docker-up`、`docker-down` 等。
3. **依赖**：运行 `pnpm install` 以应用更新的 lockfile 和 overrides；典型安装无需手动更改依赖。
