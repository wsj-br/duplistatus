# 版本 1.3.2 {#version-132}

## 概览 {#overview}

版本 1.3.2 是一个维护性版本，修复了依赖项中的安全漏洞，解决了 pnpm install 警告（对等依赖 webpack 和已弃用的子依赖项），修复了应用和文档中的 lint 与构建警告，并更新了文档以反映当前的 Docker 脚本名称和包版本。

## 安全性 {#security}

### 依赖项漏洞修复 {#dependency-vulnerability-fixes}

- **Dependabot 依赖项更新（文档）**：合并了两个 Dependabot PR 以解决文档工作区中的漏洞：
  - **ajv** 在 `/documentation` 中：从 8.17.1 更新至 8.18.0，以及从 6.12.6 更新至 6.14.0。
  - **fast-xml-parser** 在 `/documentation` 中：从 5.3.5 更新至 5.3.8。
  两者均通过 /documentation 的 dependabot 多更新合并。

- **pnpm 高危漏洞覆盖**：添加 pnpm 覆盖以修复四个高危问题：
  - **minimatch** (>=3.1.4)：修复 GHSA-3ppc-4f35-3m26（通过重复通配符的 ReDoS）、GHSA-7r86-cg39-jmmj（组合回溯）和 GHSA-23c5-xmqv-rm74（嵌套 extglobs 回溯）漏洞，路径为 documentation > @docusaurus/core > serve-handler > minimatch。
  - **serialize-javascript** (>=7.0.3)：修复 GHSA-5c6j-r48x-rmvq（通过 RegExp.flags 和 Date.prototype.toISOString() 的 RCE）漏洞，路径为 webpack > terser-webpack-plugin > serialize-javascript。

## Bug 修复 {#bug-fixes}

- **pnpm install 警告**：解决了三个安装时警告：(1) 由于项目中已包含 webpack 5.105.3，通过在根目录 `package.json` 中添加 `pnpm.peerDependencyRules.allowedVersions.webpack: "5"` 解决了来自 `next-intlayer` > `@intlayer/webpack` 的未满足对等依赖 `webpack@5.104.1` 问题。(2) 通过为 better-sqlite3 中的 `prebuild-install@7.1.3` 和 documentation > docusaurus-search-local > cheerio 中的 `whatwg-encoding@3.1.1` 添加 `pnpm.allowedDeprecatedVersions`，解决了已弃用的子依赖项问题，使安装过程无警告运行。
- **应用程序日志查看器（exhaustive-deps）**：解决了自动滚动效果的 react-hooks/exhaustive-deps 警告。`logData` 被故意从依赖数组中省略，以确保滚动仅在自动滚动或所选文件更改时触发，而不是每次轮询时都触发；新行滚动由 `loadLogs` 处理。添加了 eslint-disable-next-line 并附有简短注释。
- **构建警告 vscode-languageserver-types（文档）**：通过在 `documentation/docusaurus.config.ts` 中添加内联插件，为文档（Docusaurus）构建中的传递依赖 `vscode-languageserver-types`（来自 intlayer-editor）添加 `ignoreWarnings`，从而抑制了 Webpack 的“关键依赖：require 函数的使用方式导致依赖无法静态提取”警告。该警告仅在文档构建期间出现，主应用构建中不会出现。

- **Docker 脚本名称**：文档现在使用当前的 npm/pnpm 脚本名称：`docker-up` → `docker:up`，`docker-down` → `docker:down`，`docker-clean` → `docker:clean`，`docker-devel` → `docker:devel`。已在 AGENTS.md 和文档（setup.md、devel.md、release-management.md）中更新。

## 迁移说明 {#migration-notes}

### 从版本 1.3.1 升级 {#from-version-131}

升级到版本 1.3.2 时：

1. **无需数据库迁移**：此版本无需任何数据库模式更改。
2. **Docker 命令**：如果您使用文档中描述的 pnpm 脚本，请使用 `pnpm docker:up`、`pnpm docker:down`、`pnpm docker:clean` 和 `pnpm docker:devel`，而不是之前的 `docker-up`、`docker-down` 等。
3. **依赖项**：运行 `pnpm install` 以应用更新后的 lockfile 和覆盖项；典型安装无需手动更改依赖项。
