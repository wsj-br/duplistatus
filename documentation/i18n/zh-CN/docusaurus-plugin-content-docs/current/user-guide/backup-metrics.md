# 备份指标 {#backup-metrics}

仪表板（表格视图）和服务器详情页面均显示随时间变化的备份指标图表。

- **仪表板**：图表显示 **duplistatus** 数据库中记录的备份总数。如果您使用卡片布局，可以选择一台服务器以查看其综合指标（当侧边栏显示指标时）。
- **服务器详情**页面：图表显示所选服务器（针对其全部备份）或单个特定备份的指标。

![Backup Metrics](../assets/screen-metrics.png)

## 内联图表控件 {#inline-chart-controls}

图表面板标题栏直接提供快速访问控件，无需导航至显示设置即可轻松配置：

### 时间范围选择器 {#time-range-selector}

图表标题栏中出现药丸形按钮，用于快速选择时间范围：**1W | 2W | 1M | 3M**

- **1W**：上次 7 天（滚动窗口）
- **2W**：上次 14 天（滚动窗口）
- **1M**：上次 30 天（滚动窗口，默认）
- **3M**：上次 90 天（滚动窗口）

此处所做的更改将与您的显示设置同步，因此您的偏好将在页面刷新后被记住。

### 图表样式切换 {#chart-style-toggle}

图表标题栏中的切换按钮允许您在以下选项之间切换：

- **平滑曲线**：以平滑曲线连接显示数据点
- **柱状图**：将每个时间段的数据显示为离散的柱条

两种模式均使用时间桶聚合以实现最佳显示。柱状图模式中的空白时段不渲染柱条。您的偏好将在页面刷新后持久化，并与显示设置同步。

## 图表数据整合 {#chart-data-consolidation}

当同一天发生多次备份时，**duplistatus** 在将数据显示在图表上之前会对其进行整合：

- **SUM**：用于累积指标（持续时间、文件数、文件大小、已上传大小）
- **LAST**：用于存储大小（当天的最新值）
- **MAX**：用于可用版本（当天的最高计数）

这种整合发生在应用时间桶之前，从而确保聚合指标的准确性。例如，5/12/26 的两次备份将在图表上产生一个整合后的数据点。

## 指标定义 {#metric-definitions}

- **已上传大小**：每天从 Duplicati 服务器上传/传输到目的地（本地存储、FTP、云提供商...）的数据总量。
- **持续时间**：每天接收到的所有备份的总持续时间，格式为 HH:MM。
- **文件数**：每天接收到的所有备份的文件数计数之和。
- **文件大小**：每天接收到的所有备份由 Duplicati 服务器报告的文件大小之和。
- **存储大小**：每天由 Duplicati 服务器报告的备份目的地已用存储大小之和。
- **可用版本**：每天所有备份的全部可用版本之和。

:::note
您可以使用 [显示设置](settings/display-settings.md) 控件来配置图表的时间范围。
:::
