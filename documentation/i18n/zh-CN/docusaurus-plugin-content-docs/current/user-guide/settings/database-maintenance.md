# 数据库维护 {#database-maintenance}

通过数据库维护操作管理您的备份数据并优化性能。

![Database maintenance](../../assets/screen-settings-database-maintenance.png)

<br/>

## 数据库备份 {#database-backup}

为安全保存或迁移目的创建整个数据库的备份。

1.  导航至[设置 → 数据库维护](database-maintenance.md)。
2.  在**数据库备份**部分，选择备份格式：
    - **数据库文件 (.db)**：二进制格式 - 最快的备份，精确保留所有数据库结构
    - **SQL 转储 (.sql)**：文本格式 - 人类可读的 SQL 语句，可在恢复前编辑
3.  点击 <IconButton icon="lucide:download" label="下载备份" />。
4.  备份文件将以带时间戳的文件名下载到您的计算机。

**备份格式：**

- **.db 格式**：推荐用于常规备份。使用 SQLite 的备份 API 创建数据库文件的精确副本，即使在数据库使用中也能确保一致性。
- **.sql 格式**：适用于迁移、检查或需要在恢复前编辑数据的情况。包含重建数据库所需的所有 SQL 语句。

**最佳实践：**

- 在主要操作（清理、合并等）前创建常规备份
- 将备份存储在与应用程序分开的安全位置
- 定期测试恢复程序以确保备份有效

<br/>

## 数据库恢复 {#database-restore}

从之前创建的备份文件恢复数据库。

1.  导航至[设置 → 数据库维护](database-maintenance.md)。
2.  在**数据库恢复**部分，点击文件输入并选择备份文件：
    - 支持的格式：`.db`、`.sql`、`.sqlite`、`.sqlite3`
    - 最大文件大小：100MB
3.  点击 <IconButton icon="lucide:upload" label="恢复数据库" />。
4.  在对话框中确认操作。

**恢复过程：**

- 在恢复前自动创建当前数据库的安全备份
- 当前数据库被备份文件替换
- 出于安全原因清除所有会话（用户必须重新登录）
- 恢复后验证数据库完整性
- 清除所有缓存以确保数据为最新

**恢复格式：**

- **.db 文件**：直接替换数据库文件。最快的恢复方法。
- **.sql 文件**：执行 SQL 语句以重建数据库。允许选择性恢复（如果需要）。

:::warning
恢复数据库将**替换所有当前数据**。此操作无法撤销。
系统将自动创建安全备份，但建议在恢复前创建自己的备份。
 
**重要：** 恢复后，出于安全原因将清除所有用户会话。您需要重新登录。
:::

**故障排除：**

- 如果恢复失败，将自动从安全备份中恢复原始数据库
- 确保备份文件未损坏且符合预期格式
- 对于大型数据库，恢复过程可能需要几分钟

<br/>

:::note
此说明适用于以下所有维护功能：仪表板、详情页和图表上的所有统计数据均使用 **duplistatus** 数据库中的数据计算。删除旧信息将影响这些计算。

## 数据清理周期 {#data-cleanup-period}

删除过时的备份记录，以释放存储空间并提高系统性能。

1.  导航至 [设置 → 数据库维护](database-maintenance.md)。
2.  选择保留周期：
    - **6 个月**：保留最近 6 个月的记录。
    - **1 年**：保留最近 1 年的记录。
    - **2 年**：保留最近 2 年的记录（默认）。
    - **删除全部数据**：删除所有备份记录和服务器。
3.  点击 <IconButton icon="lucide:trash-2" label="清除旧记录" />。
4.  在对话框中确认操作。

**清理效果：**

- 删除早于所选周期的备份记录
- 更新所有相关的统计数据和指标

:::warning

选择“删除全部数据”选项将从系统中**永久删除所有备份记录和配置设置**。

强烈建议在执行此操作之前创建数据库备份。

:::

<br/>

## 删除备份任务数据 {#delete-backup-job-data}

删除特定备份任务（类型）的数据。

1.  导航至 [设置 → 数据库维护](database-maintenance.md)。
2.  从下拉列表中选择一个备份任务。
    - 备份将按服务器别名或名称排序，然后按备份名称排序。
3.  点击 <IconButton icon="lucide:folder-open" label="删除备份任务" />。
4.  在对话框中确认操作。

**删除效果：**

- 永久删除与此备份任务/服务器相关的所有数据。
- 清理相关的配置设置。
- 相应地更新仪表板统计数据。

<br/>

## 删除服务器数据 {#delete-server-data}

删除特定服务器及其所有相关的备份数据。

1.  导航至 [设置 → 数据库维护](database-maintenance.md)。
2.  从下拉列表中选择一台服务器。
3.  点击 <IconButton icon="lucide:server" label="删除服务器数据" />。
4.  在对话框中确认操作。

**删除效果：**

- 永久删除所选服务器及其所有备份记录
- 清理相关的配置设置
- 相应地更新仪表板统计数据

<br/>

## 合并重复服务器 {#merge-duplicate-servers}

检测并合并名称相同但 ID 不同的重复服务器。使用此功能可将它们整合为单个服务器条目。

这种情况可能会在升级或重新安装后 Duplicati 的 `machine-id` 发生变化时出现。仅在存在重复服务器时才会显示重复服务器。如果未检测到重复项，该部分将显示一条消息，表明所有服务器名称均唯一。

1.  导航至 [设置 → 数据库维护](database-maintenance.md)。
2.  如果检测到重复服务器，将出现 **合并重复服务器** 部分。
3.  检查重复服务器组列表：
    - 每个组显示名称相同但 ID 不同的服务器
    - **目标服务器**（按创建日期排序最新）将被高亮显示
    - 将被合并的 **旧服务器 ID** 将单独列出
4.  勾选每个组旁边的复选框，选择您想要合并的服务器组。
5.  点击 <IconButton icon="lucide:git-merge" label="合并所选服务器" />。
6.  在对话框中确认该操作。

**合并过程：**

- 所有旧服务器 ID 均合并到目标服务器（按创建日期排序最新）
- 所有备份记录和配置均转移到目标服务器
- 相同备份名称的重复 `backup_id` 值将合并为单个 ID（以最近的备份行为准）
- 旧服务器条目将被删除
- 仪表板统计数据将自动更新

:::info[IMPORTANT]
此操作无法撤销。建议在确认前进行数据库备份。
:::
