# 备份监控 {#backup-monitoring}

![Backup alerts](../../assets/screen-settings-monitoring.png)

## 服务器筛选 {#server-filtering}

可以使用筛选字段对本页面的服务器列表进行筛选。

**筛选匹配项：**
- 服务器 ID
- 服务器 URL
- 备份任务名称

在管理多个系统时，这可以方便地在监控设置中快速定位特定的服务器或备份。

## 配置单个备份监控设置 {#configure-per-backup-monitoring-settings}

- **服务器名称**：要监控逾期备份的服务器名称。
   - 点击 <SvgIcon svgFilename="duplicati_logo.svg" height="18"/> 以打开 Duplicati 服务器的 Web 界面
   - 点击 <IIcon2 icon="lucide:download" height="18"/> 以从该服务器采集备份日志。
- **备份名称**：要监控逾期备份的备份名称。
- **下次运行**：下次计划的备份时间。如果计划在未来运行，则显示为绿色；如果已逾期，则显示为红色。将鼠标悬停在“下次运行”值上，会显示一个工具提示，其中包含来自数据库的上次备份时间戳，格式为完整日期/时间和相对时间。
- **备份监控**：为此备份启用或禁用备份监控。
- **预期备份间隔**：预期的备份间隔。
- **单位**：预期间隔的单位。
- **允许的日期**：备份允许的星期。

如果服务器名称旁边的图标显示为灰色，则表示该服务器未在 [设置 → 服务器设置](/user-guide/settings/server-settings) 中配置。

:::note
当你从 Duplicati 服务器采集备份日志时，**duplistatus** 会自动更新备份监控间隔和配置。
:::

:::tip
为了获得最佳效果，请在 Duplicati 服务器中更改备份作业间隔配置后采集备份日志。这可确保 **duplistatus** 与当前配置保持同步。
:::

## 全局配置 {#global-configurations}

这些设置适用于所有备份：

| 设置 | 描述 |
|:--------------------------------|:----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **备份容差** | 在标记为逾期之前，在预期备份时间基础上增加的宽限期（允许的额外时间）。默认值为 **1 小时**。 |
| **备份监控间隔** | 系统检查逾期备份的频率。默认为**5 分钟**。                                                                                                                                                                                                                                                            |
| **通知频率** | 发送逾期通知的频率：<br/> **一次`: Send **just one** notification when the backup becomes overdue. <br/> `每天`: Send **daily** notifications while overdue (default). <br/> `每周`: Send **weekly** notifications while overdue. <br/> `每月**：在逾期期间发送**每月**通知。 |

## 可用操作 {#available-actions}

| 按钮 | 描述 |
|:--------------------------------------------------------------------|:--------------------------------------------------------------------------------------------------------------------------------------|
| <IconButton label="保存备份监控设置" /> | 保存设置，清除任何已禁用备份的定时器，并运行逾期检查。 |
| <IconButton icon="lucide:import" label="全部采集 (#)"/>          | 从所有配置的服务器采集备份日志，括号内为要采集的服务器数量。                                   |
| <IconButton icon="lucide:download" label="下载 CSV"/>           | 下载一个 CSV 文件，其中包含所有备份监控设置以及数据库中的“上次备份时间戳 (DB)”。               |
| <IconButton icon="lucide:refresh-cw" label="立即检查"/>            | 立即运行逾期备份检查。这在更改配置后非常有用。它还会触发“下次运行”的重新计算。 |
| <IconButton icon="lucide:timer-reset" label="重置通知"/> | 重置全部备份的上次发送的逾期通知。                                                                            |
