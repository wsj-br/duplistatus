# 审计日志 {#audit-logs}

审计日志提供了 **duplistatus** 中所有系统更改和用户操作的完整记录。这有助于出于安全和故障排除目的跟踪配置更改、用户活动和系统操作。

![Audit Log](../../assets/screen-settings-audit.png)

## 审计日志查看器 {#audit-log-viewer}

审计日志查看器按时间顺序显示所有记录的事件，包含以下信息：

- **时间戳**：事件发生的时间
- **用户**：执行操作的用户名（自动化操作显示为“系统”)
- **操作**：执行的具体操作
- **类别**：操作的类别（身份验证、用户管理、配置、备份操作、服务器管理、系统操作）
- **状态**：操作成功或失败
- **目标**：受影响的对象（如果适用）
- **详情**：关于操作的附加信息

### 查看日志详情 {#viewing-log-details}

点击任何日志条目旁边的 <IconButton icon="lucide:eye" /> 眼睛图标即可查看详细信息，包括：
- 完整时间戳
- 用户信息
- 完整的操作详情（例如：更改的字段、统计等）
- IP 地址和 User Agent
- 错误消息（如果操作失败）

### 导出审计日志 {#exporting-audit-logs}

您可以将过滤后的审计日志导出为两种格式：

| 按钮 | 描述 |
|:------|:-----------|
| <IconButton icon="lucide:download" label="CSV"/> | 将日志导出为 CSV 文件，以便进行电子表格分析 |
| <IconButton icon="lucide:download" label="JSON"/> | 将日志导出为 JSON 文件，以便进行程序化分析 |

:::note
导出内容仅包含基于当前活动过滤器可见的日志。要导出所有日志，请先全部清除过滤器。
:::
