# Version 1.4.2 {#version-142}

## Overview {#overview}

Version 1.4.2 is a maintenance release focused on stability and reliability. Reduces redundant authentication requests when the dashboard loads, improves the "Merge Duplicate Servers" tool so it also consolidates duplicate Duplicati backup IDs, and removes duplicate rows from the dashboard. It also records server merges in the audit log and pins Next.js to a stable release.

---

## Bug fixes {#bug-fixes}

- **Duplicate entries on the dashboard**: The dashboard no longer shows multiple rows for the same backup name when several records share the same most-recent backup date.
- **Merging duplicate servers**: Merging servers now also consolidates duplicate Duplicati backup IDs that share the same backup name on the merged server, keeping the ID from the most recent backup so history stays consistent.

---

## Improvements {#improvements}

### Performance {#performance}

- **Fewer authentication requests on load**: The signed-in user is now fetched once and shared across the page (header, password guard, theme/configuration providers, and dashboard widgets) instead of each component requesting it separately, reducing redundant network calls when the dashboard opens.

### Auditing {#auditing}

- **Server merges recorded in the audit log**: Merging servers is now recorded as a `servers_merged` entry in the audit log for better traceability.

### Interface {#interface}

- **Merge Duplicate Servers layout**: In Settings > Database Maintenance > Merge Duplicate Servers, the "Target Server (newest)" and "Old Server ID" labels now render as proper table-header cells (theme-aware) instead of pill/button shapes, for a clearer, more consistent look.

---

## Changed {#changed}

- **Next.js stable release**: Next.js is now pinned to the stable `16.2.9` release (previously a canary build) for improved reliability.

---

## Migration notes {#migration-notes}

### From version 1.4.1 {#from-version-141}

When upgrading to version 1.4.2:

1. **No database migration required**: This release does not require any database schema changes.
2. **Docker users**: Pull the latest image; the startup crash fix is included in the updated container build.

## Changelog {#changelog}

### Version 1.4.2 changes {#version-142-changes}

- **Fixed**: Docker standalone container startup crash caused by a missing `@swc/helpers` ESM helper file.
- **Fixed**: Duplicate dashboard rows when several records shared the same latest backup date.
- **Fixed**: Server merge now consolidates duplicate Duplicati backup IDs sharing the same backup name.
- **Changed**: Signed-in user is fetched once via a shared provider, reducing redundant `/api/auth/me` requests on load.
- **Changed**: Server merge operations are now recorded in the audit log as `servers_merged`.
- **Changed**: Merge Duplicate Servers labels render as table-header cells instead of pill/button shapes.
- **Changed**: Next.js pinned to the stable `16.2.9` release.
