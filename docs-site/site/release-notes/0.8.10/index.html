    Release Notes 0.8.10¶

    Release notes for duplistatus version 0.8.10.

    Overview¶

    Version 0.8.10 introduces enhanced overdue backup monitoring, improved notification system, and better performance optimizations.

    New Features¶

    Enhanced Overdue Backup Monitoring¶

    • Smart Detection: Intelligent overdue backup detection with configurable grace periods
    • Escalation System: Automatic notification escalation for persistent overdue backups
    • Recovery Tracking: Track when overdue backups are resolved
    • Customizable Thresholds: Per-backup job overdue tolerance settings

    Improved Notification System¶

    • Enhanced Templates: More customizable notification message templates
    • Multi-Channel Support: Better support for multiple notification channels
    • Delivery Tracking: Track notification delivery status
    • Retry Logic: Automatic retry for failed notifications

    Performance Improvements¶

    • Database Optimization: Improved database query performance
    • Memory Management: Better memory usage and garbage collection
    • Caching: Enhanced caching for frequently accessed data
    • API Response Times: Faster API response times

    Enhancements¶

    User Interface¶

    • Better Dashboard: Improved dashboard layout and responsiveness
    • Enhanced Charts: Better chart visualization and interactivity
    • Mobile Support: Improved mobile device compatibility
    • Accessibility: Better accessibility features

    API Improvements¶

    • New Endpoints: Additional API endpoints for better integration
    • Enhanced Responses: More detailed API responses
    • Better Error Handling: Improved error messages and codes
    • Rate Limiting: Basic rate limiting for API protection

    Configuration Management¶

    • Unified Settings: Centralized configuration management
    • Import/Export: Configuration backup and restore
    • Validation: Better configuration validation
    • Defaults: Improved default settings

    Bug Fixes¶

    Configuration Changes¶

    New Configuration Options¶

    # Enhanced overdue monitoring
overdue:
  defaultGracePeriod: 3600
  escalation:
    enabled: true
    levels:
      - delay: 3600
        action: warning
      - delay: 7200
        action: critical

# Improved notifications
notifications:
  templates:
    success: "Backup {{backupName}} completed successfully"
    error: "Backup {{backupName}} failed: {{errorMessage}}"
  delivery:
    retryAttempts: 3
    retryDelay: 300

    Deprecated Options¶

    • notification.oldTemplate (use notification.templates instead)
    • overdue.simpleCheck (use overdue.escalation instead)

    Installation¶

    Docker Compose¶

    services:
  duplistatus:
    image: wsjbr/duplistatus:0.8.10
    container_name: duplistatus
    restart: unless-stopped
    ports:
      - "9666:9666"
    volumes:
      - duplistatus_data:/app/data
    environment:
      - TZ=UTC

    Docker CLI¶

    docker run -d \
  --name duplistatus \
  --restart unless-stopped \
  -p 9666:9666 \
  -v duplistatus_data:/app/data \
  wsjbr/duplistatus:0.8.10

    Upgrade Instructions¶

    From Version 0.7.x¶

    1. Backup your data (automatic backup created during upgrade)
    2. Update Docker image: docker pull wsjbr/duplistatus:0.8.10
    3. Restart container: docker-compose down && docker-compose up -d
    4. Monitor logs: docker logs -f duplistatus
    5. Verify upgrade: Check that all features work correctly

    From Version 0.6.x or Earlier¶

    1. First upgrade to 0.7.x (see Version 0.7 Migration Guide)
    2. Then upgrade to 0.8.10 using the steps above

    Known Issues¶

    Minor Issues¶

    • Charts: Some chart animations may be slow on older devices
    • Notifications: Email notifications may take longer on first send
    • Database: Large databases may take longer to migrate

    Workarounds¶

    • Charts: Disable animations in settings
    • Notifications: Test email configuration after upgrade
    • Database: Ensure sufficient disk space for migration
