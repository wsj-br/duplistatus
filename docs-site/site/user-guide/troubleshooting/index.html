    Troubleshooting¶

    Common issues and solutions for duplistatus installation, configuration, and operation.

    Common Issues¶

    Installation Issues¶

    Container Won't Start¶

    Symptoms: Container fails to start or exits immediately

    Possible Causes: - Port 9666 already in use - Insufficient system resources - Docker configuration issues - Image corruption

    Solutions: 1. Check port availability: 

    netstat -tulpn | grep 9666
    2. Check container logs: 
    docker logs duplistatus
    3. Verify system resources: 
    docker system df
free -h
    4. Recreate container: 
    docker-compose down
docker-compose up -d

    Can't Access Web Interface¶

    Symptoms: Browser shows connection error or timeout

    Possible Causes: - Firewall blocking port 9666 - Container not running - Wrong IP address or port - Network configuration issues

    Solutions: 1. Verify container status: 

    docker ps | grep duplistatus
    2. Check port mapping: 
    docker port duplistatus
    3. Test local connectivity: 
    curl http://localhost:9666
    4. Check firewall rules: 
    sudo ufw status
sudo iptables -L

    Configuration Issues¶

    Duplicati Server Not Appearing¶

    Symptoms: Duplicati server doesn't show up in dashboard

    Possible Causes: - Duplicati server not configured to send reports - Network connectivity issues - Firewall blocking communication - Wrong URL configuration

    Solutions: 1. Verify Duplicati configuration: - Check send-http-url setting - Verify send-http-result-output-format is set to Json - Ensure send-http-log-level is set to Information

    1. Test connectivity: 

      curl -X POST http://your-duplistatus-server:9666/api/upload \
  -H "Content-Type: application/json" \
  -d '{"test": "connection"}'

    2. Check network connectivity: 

      ping your-duplistatus-server
telnet your-duplistatus-server 9666

    3. Review duplistatus logs: 

      docker logs duplistatus | grep -i error

    Missing Backup Data¶

    Symptoms: Dashboard shows servers but no backup information

    Possible Causes: - Backup jobs not running - Duplicati server configuration issues - Data collection problems - Database issues

    Solutions: 1. Check backup job status: - Verify backup jobs are scheduled and running - Check Duplicati server logs - Ensure backup jobs are configured correctly

    1. Manual log collection:
    2. Go to Settings Database Maintenance Collect Logs
    3. Select servers and start collection

    4. Monitor collection progress

    5. Verify database: 

      docker exec -it duplistatus sqlite3 /app/data/backups.db ".tables"

    6. Check data collection logs: 

      docker logs duplistatus | grep -i "collect\|upload"

    Performance Issues¶

    Slow Dashboard Loading¶

    Symptoms: Dashboard takes a long time to load

    Possible Causes: - Large database size - Insufficient system resources - Network latency - Database optimization issues

    Solutions: 1. Check database size: 

    docker exec -it duplistatus ls -lh /app/data/backups.db

    1. Monitor system resources: 

      docker stats duplistatus

    2. Optimize database:

    3. Go to Settings Database Maintenance
    4. Clean old backup logs

    5. Remove unnecessary data

    6. Check network performance: 

      ping your-duplistatus-server
traceroute your-duplistatus-server

    High Memory Usage¶

    Symptoms: Container using excessive memory

    Possible Causes: - Memory leaks - Large dataset processing - Inefficient queries - Resource limits

    Solutions: 1. Monitor memory usage: 

    docker stats duplistatus

    1. Check for memory leaks: 

      docker logs duplistatus | grep -i "memory\|leak"

    2. Optimize data processing:

    3. Reduce data collection frequency
    4. Clean old data regularly

    5. Optimize database queries

    6. Adjust resource limits: 

      services:
  duplistatus:
    # ... other configuration
    deploy:
      resources:
        limits:
          memory: 512M
        reservations:
          memory: 256M

    Notification Issues¶

    Notifications Not Sending¶

    Symptoms: No notifications received despite configuration

    Possible Causes: - Incorrect notification configuration - Network connectivity issues - Service authentication problems - Rate limiting

    Solutions: 1. Test notification configuration: - Go to Settings Notifications - Use "Test Notifications" feature - Check notification logs

    1. Verify NTFY configuration:
    2. Check NTFY server URL
    3. Verify topic name

    4. Test NTFY connectivity

    5. Check email configuration:

    6. Verify SMTP settings
    7. Test email connectivity

    8. Check spam folders

    9. Review notification logs: 

      docker logs duplistatus | grep -i "notification\|ntfy\|email"

    Too Many Notifications¶

    Symptoms: Receiving excessive notifications

    Possible Causes: - Overly sensitive notification settings - Multiple notification channels - False positive alerts - Notification loops

    Solutions: 1. Adjust notification sensitivity: - Increase grace periods - Reduce notification frequency - Configure quiet hours

    1. Review notification settings:
    2. Check per-backup notification settings
    3. Verify global notification configuration

    4. Disable unnecessary notifications

    5. Implement notification filtering:

    6. Use notification templates
    7. Configure notification rules
    8. Set up notification priorities

    Diagnostic Tools¶

    Log Analysis¶

    Container Logs¶

    # View all logs
docker logs duplistatus

# Follow logs in real-time
docker logs -f duplistatus

# Filter logs by keyword
docker logs duplistatus | grep -i "error\|warning\|failed"

# View logs with timestamps
docker logs -t duplistatus

    Application Logs¶

    # Access container shell
docker exec -it duplistatus /bin/bash

# View application logs
tail -f /app/logs/app.log

# Check error logs
grep -i "error" /app/logs/app.log

    Database Diagnostics¶

    Database Health Check¶

    # Access database
docker exec -it duplistatus sqlite3 /app/data/backups.db

# Check database integrity
PRAGMA integrity_check;

# View database schema
.schema

# Check table sizes
SELECT name, COUNT(*) FROM sqlite_master WHERE type='table';

    Performance Analysis¶

    # Check slow queries
EXPLAIN QUERY PLAN SELECT * FROM backups WHERE server_id = 1;

# Analyze database statistics
ANALYZE;

# Check index usage
PRAGMA index_list(backups);

    Connectivity Testing¶

    # Test local connectivity
curl -v http://localhost:9666

# Test remote connectivity
curl -v http://your-server:9666

# Test API endpoints
curl -X GET http://localhost:9666/api/servers

    Port and Service Testing¶

    # Check if port is open
nmap -p 9666 your-server

# Test service response

# Check service status
systemctl status docker

    Recovery Procedures¶

    Data Recovery¶

    Database Backup¶

    # Create database backup
docker exec duplistatus cp /app/data/backups.db /app/data/backups.db.backup

# Copy backup to host
docker cp duplistatus:/app/data/backups.db.backup ./backups.db.backup

    Database Restore¶

    # Stop container
docker-compose down

# Restore database
docker cp ./backups.db.backup duplistatus:/app/data/backups.db

# Start container
docker-compose up -d

    Service Recovery¶

    Container Recovery¶

    # Restart container
docker-compose restart duplistatus

# Recreate container
docker-compose down
docker-compose up -d

# Clean restart
docker-compose down
docker system prune -f
docker-compose up -d

    Data Recovery¶

    # Check volume status
docker volume ls | grep duplistatus

# Inspect volume
docker volume inspect duplistatus_data

# Recover from volume
docker run --rm -v duplistatus_data:/data -v $(pwd):/backup alpine tar czf /backup/duplistatus-data.tar.gz -C /data .
